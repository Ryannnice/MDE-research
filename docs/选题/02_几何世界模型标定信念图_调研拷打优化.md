## 0. 结论先行

原始 I2 的直觉是对的: 几何基础模型的点图、深度和相机输出不能被下游当作确定事实。问题是, 2024-2026 的近作已经把很多表层 claim 做掉了:

- DUSt3R/MASt3R 已经输出 pointmap 和 confidence, 并把 confidence 用于 alignment/fusion。
- Trust3R 已经把 feed-forward pointmap 的 heuristic confidence 推进到 evidential uncertainty, 并报告 risk-coverage、sparsification、VGGT 迁移、Tricky24 透明/反光可靠性和 MASt3R-SLAM 权重替换。
- UGOT 已经明确做了 depth uncertainty + sparse-view 3DGS depth supervision。
- GaussianLSS/alpha-OCC 已经说明 BEV/OCC 下游会直接消费 depth uncertainty。
- BayesCap 已经覆盖了 "冻结 deterministic model + 轻量 calibrated uncertainty" 的通用套路。

因此当前版本不能再主张:

```text
我们首次给几何基础模型输出加 uncertainty/belief map。
```

也不建议主张:

```text
我们首次让下游 3D 系统按不确定性消费几何先验。
```

更稳的优化版应改成:

```text
给已有几何基础模型的 pointmap/confidence/uncertainty 输出加一个 consumer-calibrated belief adapter,
把模型内部可靠性分数转成下游 sparse-view reconstruction 可消费、可校准、可审计的软几何约束,
并证明这种校准过的消费策略比 deterministic prior、原始 confidence、Trust3R-style uncertainty readout 和 UGOT-style depth uncertainty weighting 更少传播错误几何。
```

推荐新工作名:

```text
Consumer-Calibrated PointMap Priors for Sparse-View Reconstruction
```

更审稿友好的中文表述:

```text
面向稀疏视角重建的消费端标定点图先验
```

当前最小可救命题:

```text
已有 pointmap/geometry foundation model 的置信图或预测方差, 即使能排序错误区域,
也未必是下游可直接消费的权重。我们学习/后处理一个单调、可校准的 belief adapter,
使其在 held-out calibration split 上满足几何残差覆盖或风险控制,
再把校准半径转换为 3DGS/NeRF 的软几何正则权重。
```

## 4. 最严厉的 reviewer 会怎么打

### Field expert

最强正面: 几何基础模型正在从输出 depth 转向 pointmap/camera/tracks, 下游确实需要知道哪些几何能信。  
拒稿级问题: Trust3R 已经把 pointmap uncertainty 做成 probabilistic evidential head, 并直接批评 DUSt3R/MASt3R confidence 是 heuristic。若本文还是训练 sigma head, novelty 基本崩。  
修复条件: 论文核心从 "预测 uncertainty" 改成 "校准已有 uncertainty 供下游消费", 并把 Trust3R 作为 baseline/输入源之一。

### Method expert

最强正面: 消费端 calibration 是合理机制, 因为 downstream weight 需要的是 risk radius, 不是模型自称的方差。  
拒稿级问题: 如果只用 NLL/heteroscedastic loss, 不能保证 coverage; 如果只做 per-pixel sigma, 不能处理多视图相关误差和空间相关错误。  
修复条件: 给出明确 calibration protocol: held-out split、group/slice calibration、monotone map、coverage curve、risk-controlled weight, 并承认只保证 marginal/group 层面, 不声称 epistemic truth。

### Experiment expert

最强正面: ECE/AUSE/coverage + 3DGS downstream 是可以闭环验证的。  
拒稿级问题: 只比 vanilla 3DGS 或 deterministic depth prior 没意义; UGOT、DNGaussian、DepthSplat、MASt3R confidence、Trust3R 必须进入比较或被明确标注为不可复现/协议不兼容。  
修复条件: 先做 L0/L1: frozen mean, 同一 depth/pointmap, 只换 reliability-to-weight policy; 证明收益来自 calibration 而非 mean refinement 或调参。

### AC / venue expert

最强正面: 这是 CVPR/ICCV 喜欢的 "强模型 + 可解释失败模式 + 下游闭环" 题。  
拒稿级问题: 标题若写 world model, 实验只做 sparse-view 3DGS, 会被认为 overclaim。  
修复条件: 标题和摘要用 sparse-view reconstruction/pointmap priors; world model 只在 introduction 的更大背景中出现。

### Skeptical prior-art expert

最强 reject: "Trust3R already does evidential pointmap uncertainty and downstream weighting; UGOT already does uncertainty-guided depth prior in sparse-view 3DGS. This paper just combines the two."  
最小反驳: "Trust3R optimizes model-side uncertainty quality; UGOT uses depth uncertainty in 3DGS. We study a missing interface problem: how to calibrate heterogeneous geometry reliability scores into consumer-side risk radii/weights under failure slices and distribution shift, and whether this prevents downstream error propagation."  
反驳成立条件: 实验必须显示 uncalibrated Trust3R/MASt3R/DUSt3R confidence 在某些 slices 或 downstream weight 下会 miscalibrate, 而 adapter 改善 coverage 和 NVS artifacts。

## 5. 优化后的研究卡

| 字段 | 优化版 |
|---|---|
| Task | 把几何基础模型输出的 pointmap/depth/confidence/uncertainty 转换为下游 sparse-view 3D reconstruction 可消费的校准软约束 |
| Gap | 现有模型端 confidence 或 uncertainty 多用于 ranking/filtering; 下游消费时缺少 calibrated risk radius 和 failure-slice coverage 检查 |
| Root challenge | 几何错误不是独立像素噪声; 透明/反光、遮挡边界、动态区、远距离会产生相关错误, 在 3DGS/NeRF 优化中被放大 |
| Core insight | 下游需要的不是"哪个像素 uncertainty 大", 而是"把该几何作为约束时应该给多大半径/权重才不过度信任" |
| Mechanism | 冻结 geometry mean 和原始 score, 在 calibration split 上学习 monotonic/conformal belief adapter, 输出 risk radius 或 calibrated sigma, 再转成 soft prior weight |
| Contribution type | 消费端几何接口 + 校准协议 + 下游闭环诊断 |
| Expected evidence | calibrated coverage、risk-coverage/AUSE、failure-slice coverage、3DGS/NVS 指标与几何伪影减少 |
| Non-goal | 不主张新 pointmap backbone; 不主张首次 uncertainty; 不主张通用 world model |
| Main threat | Trust3R/UGOT 组合会让贡献显得薄 |
| Best venue fit | CVPR/ICCV/ECCV; 若 calibration/risk-control 理论足够强, 可考虑 ICLR/NeurIPS |

## 6. 方法重构

### 6.1 输入与输出

给定几何基础模型:

```math
(\mu_X(p), s_0(p), h(p)) = F_\theta(I_{1:N})
```

其中:

- `mu_X(p)`: pointmap/depth-unprojected point。
- `s_0(p)`: 原始可靠性分数, 例如 DUSt3R/MASt3R confidence、VGGT confidence、Trust3R uncertainty trace、BayesCap sigma、gradient uncertainty。
- `h(p)`: 可选特征, 首版尽量不用, 避免变成新模型训练论文。

目标不是重新预测 `mu_X`, 而是学习一个消费端校准器:

```math
r_\alpha(p) = C_\psi(s_0(p), z(p))
```

`r_alpha(p)` 是几何残差半径或风险尺度, 目标为:

```math
P(e_X(p) \le r_\alpha(p)) \approx \alpha
```

其中 `z(p)` 是少量分组变量, 如 depth bin、edge/boundary、view overlap、transparent/specular mask、far-range bin。首版只允许低维 `z`, 防止训练一个难解释的大 head。

### 6.2 校准器选择

按复杂度从低到高:

| 版本 | 机制 | 为什么要做 |
|---|---|---|
| V0: global temperature / scale | `r = T * s_0` 或 `r = a s_0 + b` | 证明原始 score 可被简单校准 |
| V1: isotonic / monotonic bin calibrator | 按 score 分桶映射 residual quantile | 非参数、单调、审稿容易理解 |
| V2: group conformal calibrator | 对 depth-bin / failure-slice 分组做 coverage | 防止只在平均像素上校准, slice 上失效 |
| V3: consumer-risk calibrator | 直接校准下游 residual 或 NVS artifact proxy | 最贴合主张, 但实现最难 |

建议首版主方法用 V1+V2, V3 作为分析或 appendix。

### 6.3 下游消费方式

将 calibrated radius 转为软约束权重:

```math
w(p) = \operatorname{clip}\left(\frac{1}{r_\alpha(p)^2 + \epsilon}, w_{\min}, w_{\max}\right)
```

在 3DGS/NeRF 中使用:

```math
\mathcal{L}_{prior}
=
\sum_p w(p)\,
\rho(\hat{G}(p) - \mu_G(p))
```

关键设计点:

- 不删除高风险区域, 只把硬约束变软。
- 所有方法共用同一 `mu_G`, 只换 `w(p)`, 隔离 belief 的作用。
- 同时报告 deterministic prior、raw confidence prior、uncalibrated uncertainty prior、calibrated prior。

### 6.4 与 Trust3R 的明确差异

| 维度 | Trust3R | 优化版 I2 |
|---|---|---|
| 主问题 | 模型端 pointmap uncertainty 如何预测得更好 | 已有 reliability score 如何被下游可靠消费 |
| 方法核心 | Evidential UQ head + NIW/Student-t distribution | Post-hoc / lightweight consumer calibration adapter |
| 主要指标 | AURC/AUSE/NLL、selective pointmaps、SLAM weighting | Coverage/risk calibration + sparse-view 3DGS/NVS error propagation |
| 对 backbone | 训练/附加 UQ head | 可把 DUSt3R/MASt3R/VGGT/Trust3R score 都作为输入 |
| 防守句 | "better uncertainty estimates" | "better calibrated consumption of geometry priors" |

### 6.5 与 UGOT 的明确差异

| 维度 | UGOT | 优化版 I2 |
|---|---|---|
| 先验类型 | monocular depth prior + uncertainty | pointmap/depth/camera geometry foundation prior |
| 机制 | uncertainty-guided OT for depth supervision | calibration-to-weight interface, 可替换不同 UQ source |
| novelty | 优化 3DGS depth supervision | 评测并校准几何先验的消费风险 |
| 必比项 | UGOT-style uncertainty weighting | 必须作为 3DGS baseline |

## 7. 最小实验闭环

### L0: belief 是否真的校准几何误差

| 项 | 设计 |
|---|---|
| Claim | adapter 后的 `r_alpha` 比 raw confidence/uncertainty 更接近目标 coverage |
| 数据 | ScanNet/ScanNet++, TUM RGB-D, ETH3D, KITTI, DTU; 具体按可跑代码和 GT 可得性定 |
| 输入源 | DUSt3R/MASt3R confidence, VGGT confidence, Trust3R uncertainty, BayesCap, GrUMoDepth/gradient |
| 指标 | ECE / adaptive calibration error, coverage curve, NLL/CRPS if density exists, AURC/AUSE, Spearman |
| 必须切片 | depth bins, edges, low texture, far range, overlap count, transparent/specular masks |
| 状态 | 待跑 |

Go gate:

```text
G0: calibrated adapter 在 held-out split 上 coverage error 明显低于 raw score,
且 AUSE/risk-ranking 不显著变差。
```

### L1: 下游 3DGS/NeRF 是否少传播错误

| 项 | 设计 |
|---|---|
| Claim | calibrated soft prior 比 deterministic/raw-uncertainty prior 更少产生局部几何伪影 |
| 任务 | sparse-view NVS / reconstruction |
| 数据 | LLFF, DTU, ScanNet, RealEstate10K 或 Mip-NeRF360 子集 |
| Baselines | no prior, deterministic depth/pointmap prior, raw confidence weighting, UGOT-style weighting, DNGaussian/SparseGS/FSGS/DepthSplat where protocol compatible, ours calibrated |
| 指标 | PSNR, SSIM, LPIPS, depth/normal error, rendered-depth error, floaters/artifact mask, Chamfer if GT available |
| 控制 | 同一 `mu_G`, 同一训练步数, 同一 3DGS hyperparams; 只换 weight policy |
| 状态 | 待跑 |

Go gate:

```text
G1: calibrated prior 在至少两个数据集或一个数据集的多个 sparse-view setting 中优于 raw confidence/uncalibrated uncertainty,
并在 failure slice 可视化中减少错误几何传播。
```

### L2: 机制消融

| 消融 | 必须回答 |
|---|---|
| raw score vs temperature vs isotonic vs group conformal | 收益来自校准还是任意重映射 |
| MASt3R confidence vs Trust3R uncertainty trace | 强 UQ 输入是否仍需消费端校准 |
| same ranking, different calibration | 排序好不等于权重好 |
| calibration split size | 需要多少标定数据才有用 |
| group calibration | failure slice 是否被平均 coverage 掩盖 |
| deterministic mean frozen | 收益不是 mean 变准 |
| oracle error weight | 下游任务理论可提升空间 |

Go gate:

```text
G2: 去掉 calibration 或只用 raw score 时, coverage 与下游结果下降;
oracle weight 显示还有合理上界, 不是任务本身不需要 uncertainty。
```

### L3: 高风险切片

| 切片 | 数据/标注来源 | 指标 |
|---|---|---|
| 透明/反光 | Booster/Tricky24, LayeredDepth | slice coverage, high-error AUROC, NVS artifact |
| 遮挡边界 | depth discontinuity / mask dilation | boundary ECE, edge artifact |
| 远距离 | KITTI/ETH3D depth bins | long-range NLL/coverage |
| 低纹理 | image gradient / entropy bins | uncertainty lift, reconstruction blur |
| 动态物体 | D2USt3R/MonST3R 相关数据或 dynamic masks | 不作为首版主线, 先列为 appendix |

Go gate:

```text
G3: 至少两个 failure slice 上, group calibration 比 global calibration 更稳。
```

## 8. 不能踩的坑

| 坑 | 为什么致命 | 修正 |
|---|---|---|
| 继续叫 "geometry world model" | 实验只做 reconstruction 会被认为 overclaim | 标题收窄为 pointmap prior / sparse-view reconstruction |
| 训练新 sigma head 当主贡献 | Trust3R/BayesCap/Kendall-Gal 已覆盖 | 主贡献改成 consumer calibration; sigma head 只做 baseline |
| 只报告 ECE/NLL/AUSE | 下游价值没有证明 | 主表必须有 3DGS/NVS 闭环 |
| 只比 vanilla 3DGS | baseline 太弱 | 加 UGOT/DNGaussian/DepthSplat/raw confidence/Trust3R |
| 把 conformal 写成强 guarantee | dense pixels 相关、分布 shift、conditional coverage 都有坑 | 只说 marginal/group coverage; 明确 calibration split 和限制 |
| 声称 epistemic uncertainty | 单次 score/adapter 无法可靠区分 epistemic | 写 reliability/risk score, 不写 epistemic truth |
| calibration 与 test 混用 | 直接破坏可信度 | train / calibration / test 三分 |
| 下游调参不公平 | weight policy 很容易靠超参赢 | 固定训练预算, 报告 sensitivity |

## 9. 修正后的 claim 版本

### 强 claim, 需要 L0-L2 全过

```text
We show that geometry foundation model confidence and uncertainty maps are not automatically consumable by downstream sparse-view reconstruction systems. A lightweight consumer-side calibration adapter converts heterogeneous pointmap reliability scores into calibrated geometric risk radii, enabling soft geometry priors that reduce error propagation in sparse-view Gaussian Splatting.
```

### 中等 claim, 若只过 L0+L1

```text
We provide a calibrated interface for consuming pointmap priors in sparse-view reconstruction, showing that post-hoc calibration improves coverage and makes uncertainty-weighted geometry regularization more stable than raw confidence or uncalibrated uncertainty.
```

### 保守 claim, 若下游提升不稳定

```text
We audit the calibration gap between geometry foundation model reliability scores and downstream geometry consumption, and provide a benchmark protocol plus failure-slice analysis for pointmap belief maps.
```

若 L0 不过:

```text
停止 I2 主线。改成 failure detector / benchmark audit, 不再主张 calibrated belief map。
```

若 L1 不过:

```text
停止 "downstream interface" 主张。只能写 uncertainty calibration diagnostic, 不适合作为 CCF-A 主攻。
```

## 10. 推荐执行路线

1. L0 最小复现: 固定一个 backbone, 例如 MASt3R 或 VGGT, 取 `mu_X` 和 raw confidence; 在 ScanNet/ETH3D/DTU 做 residual vs score 的 coverage/AUSE。
2. V0/V1 adapter: 做 temperature scaling 与 isotonic calibration, 不训练大模型。
3. Trust3R threat check: 若 Trust3R 代码/权重可用, 直接把其 uncertainty trace 放进同一 calibration protocol; 若不可用, 先标注为 `待复现` 和 reviewer threat。
4. L1 3DGS 小闭环: 选一个已有 sparse-view 3DGS pipeline, 保持 `mu_G` 相同, 比较 deterministic/raw/calibrated 三种 prior。
5. Failure-slice 最小包: 先做 depth bin + boundary + low texture; Booster/Tricky24 后补。
6. 到 G0/G1 后再决定是否写论文主线; 未过门槛前不要写 abstract。

## 11. 当前最终判断

I2 不该放弃, 但必须换靶心。原始靶心是:

```text
给几何基础模型输出加 belief map。
```

这个在 Trust3R 出现后太危险。优化靶心应是:

```text
把已有几何可靠性分数校准成下游可消费的风险半径和软约束,
并用 sparse-view reconstruction 证明校准消费比直接消费更可靠。
```

一句话版:

```text
不是再做一个点图不确定性头, 而是做几何基础模型到 3D 重建系统之间的 calibrated consumption layer。
```

推荐下一版文档标题:

```text
I2: Consumer-Calibrated PointMap Priors for Sparse-View Reconstruction
```
