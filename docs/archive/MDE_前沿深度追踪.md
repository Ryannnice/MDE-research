# B 部分:2024–2026 前沿专题

## B1. 零样本米制深度(metric depth)—— 解开"深度×焦距"的纠缠

### 问题核心
单目米制深度难的根因:**深度与相机焦距本质纠缠**。同一张图,焦距不同对应的真实尺度就不同——忽略相机的模型只能给出"差一个未知尺度"的相对深度。2024 年的三个代表作各自给出不同立场。

### 三条技术路线

| 方法 | 是否需要内参(推理时) | 核心机制 |
|------|----------------------|----------|
| **UniDepth / V2**(arXiv:2403.18913 / 2502.20110) | **不需要** | 自提示相机模块(self-promptable camera module)预测稠密相机表示,再用它调制深度特征;**伪球面输出表示**解开相机与深度;V2 加边缘引导损失锐化边界 |
| **Metric3D v2**(arXiv:2404.15506) | **需要/利用** | **规范相机空间变换(canonical camera space)**:把图像归一化到共享的规范相机空间消除米制歧义;深度与表面法向**联合优化**互相增强 |
| **Depth Pro**(arXiv:2410.02073,Apple) | **不需要** | 亚秒级出高分辨率锐利米制深度;**从图像本身推断焦距**,无需 EXIF 元数据 |

### 两种哲学的对比
- **推理时免内参(UniDepth、Depth Pro)**:模型内部自己推断相机,任意图喂进去都能出米制结果。部署最省事,适合"野外"图像。
- **内参条件化(Metric3D v2)**:用已知内参映射到规范空间,在内参可得时跨极不同相机泛化更干净。适合有标定信息的工业/车载场景。

### 与原综述的关系
原综述把"米制 vs 相对深度"这个区分完全略过了——它默认所有方法输出的就是该数据集尺度下的度量深度。而 2024 年后,**"相对深度泛化好但无尺度" vs "米制深度有尺度但难泛化"** 成了核心张力,ZoeDepth(见合订本 9.3)、UniDepth、Metric3D v2 都是在弥合这条缝。这是理解当代 MDE 落地(AR 测量、机器人)的关键轴线。

> **判断**:对落地应用而言,UniDepth / Depth Pro 的"免内参米制"路线实用价值最高;但"从单图推焦距"本身仍是个弱病态问题,在极端视角/裁剪图上可能失准——这是它们共同的潜在软肋(待核实其在非常规焦距上的退化幅度)。

## B2. 视频深度估计 —— 从"逐帧闪烁"到"时序一致"

### 问题核心
把图像 MDE 逐帧套到视频上会**闪烁(flickering)**:相邻帧深度跳变、同一物体深度不稳。视频深度要的是**时序一致性(temporal consistency)**,尤其在复杂相机运动和动态物体下,且要能处理**超长视频**(数分钟以上)而不爆显存。

### 两大阵营

**阵营一:前馈/回归式(建在强图像深度骨干上)**
- **Video Depth Anything**(arXiv:2501.12375,CVPR 2025 Highlight):建在 **Depth Anything V2** 上,继承其强单帧空间先验,再把图像模型**适配到时序域**让深度跨帧稳定,同时保持高效——这是它与扩散类方法的核心区别。主打**超长视频 + 高效前馈**,不付扩散的算力代价。
- **oVDA / Temporally-Consistent Depth with Low Memory**(arXiv:2510.09182):借 **LLM 技巧**——推理时缓存潜在特征、训练时 mask 帧,削减显存同时保持一致性。

**阵营二:视频扩散先验式**
- **ChronoDepth**(arXiv:2406.01493,TRI):从 **Stable Video Diffusion** 派生。关键训练洞察:**先训空间层,再冻结空间层训时序层**,稳定性最佳;可处理任意长度开放世界视频。
- **DepthCrafter**(arXiv:2409.02095):生成时序一致的长深度序列且细节精细,**无需相机位姿/光流**等额外输入,扩散式。

### 阵营对比

| 维度 | 前馈式(Video Depth Anything) | 扩散先验式(ChronoDepth / DepthCrafter) |
|------|------------------------------|------------------------------------------|
| 效率 | 高,可扩到超长视频 | 偏重,算力代价大 |
| 细节 | 强(继承 DA V2) | 极精细 |
| 一致性 | 强 | 强 |
| 适用 | 长视频、实时倾向 | 离线高质量生成 |

### 与原综述的关系
原综述的"自监督"一章虽用到视频连续帧,但目标是**借帧间几何做训练监督**,而非**输出时序一致的视频深度**——两者目标不同。视频深度作为独立任务在原综述中完全缺席,却是 2024–2025 最热的应用分支之一(3D 视频、AR、SLAM 前端)。

> **本调研判断**:Video Depth Anything 的"在强图像基础模型上加轻量时序适配"范式,与 B1 的 UniDepth、合订本第 9 章的 Depth Anything 共享同一个时代逻辑——**先有强单图基础模型,再低成本适配到新维度(时序/米制)**。这是当代 MDE 的元方法论。

# A 部分:里程碑方法技术拆解

> 本部分对原综述只一笔带过的方法做**机制级**展开:架构、损失、关键公式与设计动机。每个方法标注其在综述中的引用编号。

## A0. 监督式架构演进的内在逻辑

原综述给了"Eigen→BTS→NeWCRFs/Binsformer"的链条,但没讲清**每一代到底解决了上一代什么问题**。补上这条逻辑线:

| 代 | 方法 | 核心创新 | 解决的前代痛点 |
|----|------|----------|----------------|
| 2014 | Eigen [20] | 多尺度 coarse+fine | 首次证明 CNN 可端到端估深度 |
| 2018 | DORN [21] | 序数回归 + 间隔递增离散化(SID) | 纯回归收敛慢、远处误差大 → 转为有序分类 |
| 2019 | BTS [22] | 局部平面引导(LPG) | 解码器上采样丢几何 → 用平面约束逐层恢复 |
| 2021 | AdaBins [23] | 自适应分箱 + mViT | 固定分箱浪费分辨率 → 每图自适应分箱 |
| 2022 | NeWCRFs [24] | 窗口全连接 CRF + Swin | CRF 全局推理太贵 → 限制在窗口内做注意力式 CRF |
| 2022 | Binsformer [56] | Transformer 化分箱 | AdaBins 的分箱与概率交互弱 → 用 DETR 式 query 生成分箱 |

主线可概括为两条交织的演化:**(1) 回归 ↔ 分类之争**(DORN/AdaBins/Binsformer 都在"把连续深度离散化再加权"这条路上精进);**(2) 如何在解码器里保住几何结构**(BTS 的平面引导、NeWCRFs 的 CRF)。

## A1. Monodepth2 [57] —— 自监督的"三件套"(ICCV 2019, Niantic)

自监督无深度 GT,靠**把邻帧 warp 到目标帧、比对光度差**来学深度。基础光度损失结合 SSIM 与 L1:

```
pe(Ia, Ib) = (α/2)(1 − SSIM(Ia, Ib)) + (1 − α)·‖Ia − Ib‖₁     (α≈0.85)
```

它的三个关键设计已成为自监督的事实基线:

**① 最小重投影损失(minimum reprojection,处理遮挡)**
以往方法对多个源帧(前/后帧)的重投影误差**取平均**——但某像素在目标帧可见、在某源帧可能被遮挡,平均会被坏匹配污染,在遮挡边界产生模糊伪影。Monodepth2 改取**逐像素最小值**:
```
Lp = min over t'  pe(I_t, I_{t'→t})
```
直觉:一个像素通常至少在某一源帧匹配良好,取最小即选中那个最佳视图、忽略被遮挡的帧。**锐化遮挡边界**。

**② 自动遮罩静止像素(auto-masking)**
当相机静止、物体与相机同速运动、或低纹理区时,"静态场景+运动相机"假设被违反,会让运动物体"塌陷成无穷远"。Monodepth2 用一个**无需额外网络**的逐像素二值掩码 μ:
```
μ = [ min_{t'} pe(I_t, I_{t'→t})  <  min_{t'} pe(I_t, I_{t'}) ]
```
逻辑:若把源帧 warp 到目标帧后并不比原始未 warp 的帧更匹配,说明该像素相对相机没动(静止相机或同速物体),从损失里**剔除**。优雅地过滤了同速物体与静止相机帧。

**③ 全分辨率多尺度采样**
解码器多尺度预测深度。不在各低分辨率上算光度损失(会让低纹理区产生深度"洞"),而是**把低分辨率深度上采样到输入分辨率**再重投影、在全分辨率算损失。减少纹理复制伪影与空洞。

最终损失:`L = μ·Lp + λ·Ls`,Ls 是对均值归一化逆深度的边缘感知平滑项(图像梯度大处降低平滑权重)。

> 意义:让一个相对标准的 ResNet 编码器-解码器在当时 KITTI 自监督上达到 SOTA,此后成为基线(SGDepth 等加语义引导处理动态物体即在其上扩展)。

## A2. AdaBins [23] —— 自适应分箱 + mViT(CVPR 2021)

**核心洞察**:深度分布因图而异(桌面特写 vs 风景),固定均匀分箱浪费分辨率。AdaBins 让分箱**逐图自适应**。

**两阶段流水线**:
1. **编码器-解码器骨干**(EfficientNet):产稠密特征图。
2. **mViT(mini Vision Transformer,即 AdaBins 模块)**:架在解码特征上,并行做两件事——

- **自适应箱宽估计**:Transformer 从解码特征输出一组**箱宽向量**,归一化后(总和=整个深度范围)定义箱中心落点。因逐图预测,分箱**集中在该场景真实出现的深度值上**。
- **箱中心计算**:箱宽确定后,箱中心由箱宽的累积和导出(每个中心位于其跨度中点)。
- **逐像素概率(range attention)**:Transformer 还产出注意力/嵌入输出,结合解码特征经 Softmax 得到**每像素在各箱上的概率分布**。

**最终深度** = 箱中心按逐像素 Softmax 概率的**加权和**:
```
depth(pixel) = Σ_k  softmax_k × bin_center_k
```
这种**分类-回归混合**避免了纯分类的离散化伪影(深度是平滑加权混合而非硬分配),又保留分箱带来的结构。

> 这就是原综述合订本 5.1 节提到的"AdaBins 自适应分箱"暗线的完整机制。Binsformer [56] 进一步用 DETR 式 query 生成分箱、强化概率分布与分箱预测的交互。

## B3. 扩散式深度估计主线 —— 从生成范式到免训练单步米制

> **本节是 A② 立项(`docs/ideas/可扩展研究方向.md` §4)的机制地基**,按一条逻辑链组织:**范式为何成立(B3.1)→ 慢在哪、怎么提速(B3.2)→ 免训练单步的内核(B3.3)→ 相对深度如何米制化(B3.4)→ idea 现状与精读路线(B3.5)**。原"Marigold 机制全解"扩为 B3.1。读完这一节即可独立理解 A② 的"免训练 ∧ 单步 ∧ 单目几何锚米制"三角从何而来、夹在谁中间。
>
> **快速定位**:范式根基→B3.1 · 提速谱系→B3.2 · ChordEdit/OT 内核→B3.3 · 米制锚→B3.4 · 读哪几篇→B3.5。配套 `papers/13_TrainingFree_SingleStep_Diffusion/`(D1–D12)与 `papers/11_FoundationModels_补充/`。

### B3.1 范式根基 —— Marigold 机制全解

**核心思想**:Marigold(Ke 等,CVPR 2024,arXiv:2312.02145)把预训练**文生图潜在扩散模型(Stable Diffusion)**改造成**仿射不变(affine-invariant)**的单目深度估计器。关键洞察:大型生成模型在文生图预训练中学到的丰富视觉先验,可迁移到稠密几何理解;无需从零训深度网络,用轻量微调把这份知识"重定向"出来。

**机制四要点**:
1. **潜在空间操作**:与 SD 一样,在冻结 VAE 的压缩潜空间而非原始像素上工作。输入 RGB 与深度图都用原 VAE 编码器编成潜码,高效且能直接复用预训练组件。
2. **图像条件由拼接实现**:把图像潜码与(带噪)深度潜码**拼接(concatenate)**后送入 U-Net,这是把生成深度绑定到具体输入场景的机制。
3. **只微调 U-Net**:冻结 VAE,仅微调 U-Net,优化标准扩散去噪目标。因此**数据高效**——作者称单 GPU 几天、仅用**合成深度数据**即可微调,却能零样本泛化到真实图。
4. **仿射不变深度表示**:深度图归一化为仿射不变形式(尺度与平移不固定),即预测**相对深度(差未知尺度与偏移)**。这正是它跨数据集泛化强的原因。

**推理与集成(ensembling)**:
- 从深度潜空间的随机噪声起步,跑标准 DDIM 式迭代去噪(以编码后的输入图为条件),再经 VAE 解码器解回深度。
- 因过程随机(依赖起始噪声),Marigold 用**多个不同噪声种子跑多次**,得到同一图的多张深度预测;再做**测试时集成**:联合求解每张的尺度+平移参数使彼此分歧最小,对齐后逐像素取中位数。降方差、出更锐利一致的深度。

> **意义与代价**:证明了"生成式图像先验可用极少任务数据转成 SOTA 稠密预测器"。但**多次采样 + 集成**正是它慢的根源——这就是 B3.2 要攻克的"少步/单步扩散深度"。后续 [E2E-FT(arXiv:2409.11355)](https://arxiv.org/abs/2409.11355) 证明这种慢部分源于推理流程缺陷,修掉后可塌缩到单步。

**衍生工作**:Marigold-DC(arXiv:2412.13389,稀疏深度引导做补全)、用散焦模糊线索做米制深度(arXiv:2505.17358,见 B3.4)、期刊版推广为通用图像分析微调协议(arXiv:2505.09358)。

### B3.2 提速谱系 —— 50 步集成如何塌缩到单步

Marigold 的 50 步 × 10 ensemble 是扩散深度落地的最大障碍。2024–2025 一连串工作把它压到 1–2 步,**但路线分两类**,这条分界正是 A② 的立足点:

| 工作 | venue | 提速手段 | 要训练? | 单步数 | 归档 |
|------|-------|----------|---------|--------|------|
| **E2E-FT** | arXiv 2409.11355 | ① 修 trailing-timestep 调度缺陷使**未微调** Marigold 单步即≈SOTA;② 再端到端微调 | 含**免训练**单步 + 之后微调 | 1(121ms@4090,~200×)| `13/[D2]` |
| **Lotus** | arXiv 2409.18124 | 改训练目标(x0-pred + detail preserver),t=1000 单步 | 要训练 | 1 | `13/[D3]` |
| **DepthFM** | AAAI25, 2403.13788 | **流匹配**:训练期用 OT 把噪声→深度的路径拉直 | 要训练(OT 在**训练期**)| 1–2 | `12/[H7]` |
| **FE2E** | arXiv 2509.04338 | 把编辑器型 DiT(用 LoRA 改 FM loss)转稠密估计器 | 要训练 | 少步 | `12/[H13]` |

> **关键分水岭**:E2E-FT 揭示"Marigold 慢有一半是推理流程 bug",**修掉后未训练模型就能单步**——这是 A② 最大的撞车对象(它顺手证明了免训练单步可行),但它**只修调度器,无 OT 轨迹塑形、无米制锚**。Lotus/DepthFM/FE2E 则把提速**学进权重**。→ A② 要占的缝:**提速发生在推理期、且是 OT 轨迹塑形而非仅修调度**(见 B3.3)。

### B3.3 免训练单步内核 —— ChordEdit 式 OT 时间平滑

A② 最干净的一块新意来自一个**编辑领域**的方法,而非深度领域:

**ChordEdit**([2602.19083](https://arxiv.org/abs/2602.19083), CVPR 2026 Oral + Best Student Paper HM)做单步图像编辑。扩散/流匹配里"原图→编辑图"的传输轨迹高度弯曲、高能量,单步大跨步积分会放大方向误差致崩坏。它用**最优传输(OT)**思想,在**去噪时间窗口内对观测代理速度场做时间加权平均**,把高能瞬时漂移场收缩成平直低能的等效恒速场——原论文正式命名 **CCF / Chord Control Field**(全文 33 处,p.7 §6.1 定义缩写;摘要另以 "smoothed variance-reduced editing field" 并行描述),沿它单步走就准。

三要件:① 时间加权的等效速度场;② **统一映射系数**(噪声/速度预测都投影到统一虚拟速度场,**与骨干无关**);③ 可选近端细化(NFE 1→2)。免训练、免反演,7GB / 0.2s。公式实锤(`pipeline_chord.py` 的 `_u_estimate`):

```
u_hat = (δ·dv_s + t_s·dv_s0) / (t_s + δ)      # 两个去噪时间点,无外部 OT 求解器
```

> ⚠️ **钉死澄清**:ChordEdit 的"时间"是**去噪步**,不是视频帧——勿误当时序平滑。可迁移内核 = **面对生成空间剧烈波动的向量场,不加步数不训新网,在去噪时间维做局部平滑把弯曲轨迹塌缩成单步低能直线**。

**迁到深度有一个非平凡重定义(真技术点)**:编辑任务两端锚都已知(源=原图 latent、目标=编辑图 latent 可估);**深度生成的目标端锚 = 条件深度后验 `p(d|I)` 的样本 `d₀`,恰恰是要求的未知量**,CCF"指向目标"的直线方向无法直接算。A② 的机制赌注:用骨干自身单步 **Tweedie/x0-prediction 出目标代理锚 `d̂₀`**,再在时间窗内对"指向 `d̂₀` 的速度场族"做 OT 加权平均。

> **可行性核查(2026-06-15)**:两块零件各有成熟数学基础——① Tweedie/x0 代理是 training-free guidance 标准做法([Universal Guidance](https://www.emergentmind.com/topics/universal-guidance-algorithm-for-diffusion-models)、[Understanding Training-free Guidance NeurIPS24](https://proceedings.neurips.cc/paper_files/paper/2024/file/c4edc5113b4ffd4632718558fb66b9ef-Paper-Conference.pdf));② OT 位移插值=恒速直线是 ChordEdit CCF 的理论核。**未验证的是"对指向同一未知 `d̂₀` 的速度场族做 OT 平均能否净修正单点 x0 偏差"这个组合**——这正是 A② 最毒的消融(§4.9 "OT 平滑只是装饰?")。机制骨架已落码自检:`A2_ccf_depth_skeleton.py`。
>
> **🔴 偏差陷阱(2026-06-22 精读 ChordEdit §3.2/4.1 + 附录 D)**:ChordEdit 的时间平均能成立,靠的是它的测量模型 `R = u_t + ε` 中 **ε 严格零均值**(原文 Eq 4.2),平均 = Jensen-L² 降方差(附录 D 证)。而 A② 的 Tweedie 锚 `d̂₀` 是 **MMSE 估计 = 有偏**,误差非零均值。**时间平均压方差,压不掉偏差。** → ChordEdit 的成功(δ 消融 Fig.9)**不能直接外推到深度**;A② 的净增益必须另有来源(几何锚纠偏,或偏差随 t 变化被多时间点抵消)。这是把 ChordEdit 迁到深度时最易被审稿人一击致命处。

**对照参照**:最接近的免训练轨迹加速 [ETC(2510.24129)](https://arxiv.org/abs/2510.24129) 用趋势外推而非 OT、且面向通用生成;其余 OT 直化([OFM](https://arxiv.org/pdf/2403.13117)、DepthFM)全在训练期。→ **"推理期 OT 时间平滑用于稠密深度回归"经穷尽搜索无人占**,是 A② 机制新意的核心(归档 `13/[D1]` ChordEdit、`13/[D6]` ETC)。

### B3.4 米制化 —— 相对深度后验如何锚到真实尺度

扩散深度骨干输出 affine-invariant **相对**深度(尺度+平移未知)。把它变**米制**(绝对米),免训练路线有三条先例,A② 要在它们的缝里:

| 工作 | 锚源 | 要训练? | 与 A② 的关系 | 归档 |
|------|------|---------|--------------|------|
| **GeoDiff** | 立体/双视图几何 | 免训练(采样期梯度引导 scale/shift)| **最近撞车**,但**需立体**→缝在"单目几何锚" | `13/[D4]` |
| **Defocus-Marigold** | 散焦模糊线索 | 免训练(推理期注入)| **最强方法先例**:同思路注入物理线索→米制,把散焦换成几何尺寸即 A② | `13/[D5]` |
| **AnchorD** | 传感器深度 + 因子图 | 免训练(patch-wise affine)| **抬高举证门槛**:证明免训练后处理也能 patch-wise 局部接地,A② 的"采样期注入"须赢它而非只赢 2-DOF 全局拟合 | `13/[D10]` |

> **两个生死消融(A② Linchpin,详见 §4.8)**:
> - **L1 米制生死**:采样期几何锚注入必须 **>** 后处理几何标定。对照臂须升到**两档**——(a) 全局 scale+shift;(b) patch-wise affine(AnchorD 式)。赢 (b) 才算真贡献。
> - **L2 单步生死**:单步注入几何锚必须 **≈** 多步 energy guidance 质量。机制弹药 [CoSIGN(2407.12676, ECCV24)](https://arxiv.org/abs/2407.12676):DPS 式逐步梯度引导在 1–2 NFE **直接失效**,但改"软约束+投影"可成——正撑 A②"把几何锚**烘焙进 CCF 方向**而非事后梯度下降"(归档 `13/[D11]`)。

**单步几何锚的内在张力**:能量引导天然要多步(迭代回推梯度),而 A② 要单步(无迭代空间)。解法是把 `E_geo` 烘焙进 CCF 的**等效直线方向构造**,一次性修正——这条能否成立是 A② 的核心科学赌注。

## B4. 非朗伯表面:透明 / 反光 —— 公认的"硬骨头"

### 问题核心
透明与反光表面**违背了几乎所有深度方法的基本假设**:折射、反射、透射破坏了立体、ToF、单目模型的前提。镜子显示的是反射场景的深度而非镜面本身;玻璃让光透过,传感器与模型都读错几何。这是基础模型(含 Depth Anything V2)仍**公认失败**的模式。

### 关键基准与数据
| 资源 | 贡献 |
|------|------|
| **Booster**(arXiv:2301.08245) | 镜面/透明表面深度基准,室内含镜子与透明物体,提供稠密 GT,专攻非朗伯材质与泛化两大开放挑战 |
| **非朗伯鲁棒性**(arXiv:2408.06083, 2024) | 记录零样本单目深度模型在一般场景表现好、却在非朗伯表面**持续失败**,并探索更鲁棒估计 |
| **Seeing and Seeing Through the Glass**(ICCV 2025, arXiv:2503.11633) | 多层深度真实+合成数据;在其上微调单层 SOTA,把透明物体四元组准确率从 ~55% 提到 ~75% |
| **3D 反射检测**(arXiv:2403.06538) | 面向移动机器人,解决玻璃/镜子破坏 2D Lidar 建图 |

### 方法趋势
最新工作转向**生成式**:用视频扩散做透明物体深度与法向(arXiv:2512.23705)、扩散式单 RGB-D(arXiv:2503.12779);扩散式法向估计 TransNormal 在 ClearGrasp 基准报告均误差降 **24.4%**。

**2024–2025 的清晰趋势**:标准模型在这些表面失败 → Booster 等专用基准暴露差距 → **扩散先验 + 针对性微调数据**成为领先修法。

> **本调研判断**:这条线与 `docs/ideas/可扩展研究方向.md` 的 Idea 5(失败模式诊断基准)直接呼应——透明/反光正是该基准应单独切片报告的维度。同时它给 Idea 1(HDR)一个旁证:**基础模型的失败模式高度集中在"物理成像假设被违反"处**(饱和、折射、反射),这正是物理驱动方向的共同立足点。

## C2. 零样本评测基准(测泛化的试金石)

| 基准 | 室内/外 | GT 来源/密度 | 定位 |
|------|---------|--------------|------|

> **基准组合的设计意图**:DIODE 给稠密室内外真实数据,ETH3D 加高分辨率真实立体/SLAM,vKITTI2 贡献 GT 完美的合成驾驶——一起**压力测试室内/外、真实/合成、稀疏/稠密**三个维度。这正是 `docs/ideas/可扩展研究方向.md` Idea 5(统一评测)要标准化的对象。

> **诚实标注**:"稠密 GT"的说法对 DIODE 室内与 vKITTI2 最成立;DIODE 室外仅 ~67% 稠密,ETH3D 立体/SLAM 深度也非均匀稠密。引用时勿一概而论。

## C3. 专门化数据集(对应失败模式)

- **Booster**(arXiv:2301.08245):镜面/透明表面,稠密 GT(见 B4)。
- **SCARED**(原综述 [62]):内窥镜微创手术场景,医学方向。
- **多模态恶劣天气数据集**(arXiv:2411.11455):雨雾雪夜,补恶劣天气下 GT 稀缺(见 `docs/ideas/可扩展研究方向.md` Idea 4)。
- **驾驶大规模深度集**(arXiv:2508.13977, 2025):主张 KITTI/nuScenes/DDAD 仍不够大,提出更大的驾驶深度集。

# D 部分:损失函数与训练技巧目录

> 原综述只给了**评测指标**(RMSE/AbsRel/δ),却完全没讲**训练损失**。但损失设计才是 MDE 方法间差异的核心之一。下面是从经典到当代的损失函数全谱,附公式与动机。

## D1. 监督式损失

### ① 尺度不变对数损失(SILog)—— Eigen 2014 [20]
动机:衡量深度的**相对关系**而非绝对尺度,使全局缩放的预测不被惩罚。设 dᵢ = log ŷᵢ − log yᵢ(n 个有效像素):

```
D(ŷ, y) = (1/n)Σᵢ dᵢ²  −  (λ/n²)(Σᵢ dᵢ)²
```

- λ=1 时完全尺度不变(第二项在对数空间抵消任意常数偏移即全局尺度);实践常用 λ≈0.5。
- **BTS/AdaBins 采用的训练损失变体**:`L_SILog = α·√[ (1/n)Σdᵢ² − (λ/n²)(Σdᵢ)² ]`,典型 λ=0.85、α=10。

> 这是监督式 MDE 最核心、流传最广的损失,直接源自原综述 [20]。

### ② 梯度匹配损失(gradient matching)—— Eigen & Fergus
动机:让预测深度梯度匹配 GT 梯度,**锐化边缘、抑制过平滑**。设 Rᵢ = log ŷᵢ − log yᵢ:
```
L_grad = (1/n)Σᵢ (|∇ₓRᵢ| + |∇_yRᵢ|)
```
MiDaS 的尺度不变版本对**多个降采样尺度 k**(常 4 尺度)求和。

### ③ BerHu(反 Huber)损失 —— Laina 2016 [48]
动机:小残差像 L1、大残差像 L2(标准 Huber 的反向),既在小误差上保持有意义梯度,又强调大离群。残差 x = ŷ − y:
```
B(x) = |x|              , |x| ≤ c
     = (x² + c²) / (2c)  , |x| > c
```
阈值逐 batch 自适应,典型 c = (1/5)·maxᵢ|xᵢ|。

### ④ 虚拟法向损失(Virtual Normal)—— Yin 2019/2021
动机:施加**高阶 3D 几何约束**而非逐像素监督。步骤:
1. 用内参把预测/GT 深度反投影为 3D 点云;
2. 随机采样三个非共线点组 {Pₐ, P_b, P_c};
3. 由两条边向量叉积构造"虚拟法向":`n = (Pₐ−P_b)×(Pₐ−P_c) / ‖·‖`;
4. 惩罚预测与 GT 虚拟法向之差:`L_VN = (1/N)Σᵢ ‖nᵢ^pred − nᵢ^gt‖₁`。

捕捉表面朝向与长程几何,补逐像素损失之缺。

**典型组合**:`L = L_SILog + α·L_grad + β·L_VN`。

## D2. 基础模型 / 零样本损失

### ⑤ 尺度平移不变损失(SSI / affine-invariant)—— MiDaS [Ranftl, TPAMI 2022]
**这是基础模型时代的核心损失**。模型预测**视差(逆深度)**,损失对全局尺度与平移不变,使深度约定不同的数据集可混合训练。两种实现:
- **闭式对齐**:逐图最小二乘求最优尺度 s 与平移 t 对齐预测到 GT,再算残差(有解析解)。
- **鲁棒归一化 + 截断 MAE(trimmed MAE)**:用中位数与平均绝对偏差归一化预测和目标,取绝对误差损失;为抗离群与噪声 GT,**截掉最大的约 20% 残差**再平均。

总损失 = 截断 SSI 数据项 + 加权多尺度梯度项(权重 α≈0.5)。

> 这就是为什么 MiDaS/Marigold 被称为预测**仿射不变**深度——输出仅在逆深度的未知仿射变换(尺度+平移)下正确。所有后续零样本与米制恢复工作都建在这个配方上。

## D3. 自监督损失(见 A1 Monodepth2 已详拆)

- **光度损失** `pe = (α/2)(1−SSIM) + (1−α)‖·‖₁`(α≈0.85)
- **最小重投影**(处理遮挡)、**自动遮罩**(过滤同速物体/静止相机)
- **边缘感知平滑** `L_s`:对均值归一化逆深度,图像梯度大处降权

## D4. 训练技巧速查

| 技巧 | 用途 | 代表 |
|------|------|------|
| 混合数据集训练 | 跨域泛化 | MiDaS、Depth Anything |
| 合成→真实伪标注(teacher-student) | 海量无标注利用 | Depth Anything V1/V2 |
| 先训空间层、再冻结训时序层 | 视频时序稳定 | ChronoDepth |
| 测试时多噪声集成(对齐+中位) | 扩散降方差 | Marigold |

> **易混淆点(诚实标注)**:文献里有另一个同名缩写 **SSIL(Scale-and-Shift-Invariant Loss)** 指的是**分类**任务里对 logits 仿射不变的损失,与这里的深度 SSI 损失**无关**,勿混。

## E2. 核心工程难题:相对深度 → 米制尺度的"锚定"

这是部署 MDE 最常见的坑。基础模型(MiDaS/Depth Anything/Marigold)给的是**仿射不变/相对深度**,实际系统要米制单位,必须"锚定":

| 场景 | 锚定手段 |
|------|----------|
| **移动 AR(ARKit/ARCore)** | 单目纯视觉绝对尺度不可观;融合 IMU(视觉惯性里程计 VIO),把预测深度锚到平台的稀疏特征点/VIO 尺度估计上恢复米制 |

| **内窥镜 SLAM** | CycleGAN 位姿模块 + 单目深度(arXiv:2408.03078) |
| **直接出米制** | 改用 B1 的 UniDepth/Depth Pro/Metric3D v2,跳过锚定 |

> **本调研判断**:这条"相对→米制"的工程裂缝,正是 B1 米制深度方向与 `docs/ideas/可扩展研究方向.md` Idea 3(几何尺度锚)的落地动机。"先得强相对深度基础模型、再锚定尺度"是当前移动/机器人 MDE 的事实管线。

## B5. 恶劣天气深度 —— md4all 范式与后继

### md4all(ICCV 2023, arXiv:2308.09711)—— 该方向的事实基线
**问题**:SOTA 单目深度在理想条件好,但夜间/雨天变得不可靠。md4all 用三招解决:

1. **昼→恶劣图像翻译**:用生成式翻译模型把标准白天图生成照片级真实的**模拟夜/雨版本**,提供高难训练输入而**无需新 GT 标注**。
2. **知识蒸馏/自蒸馏(核心)**:不在难样本上直接算训练信号(那里光度与位姿假设都崩),而是**把对应的简单白天图喂模型产出可靠深度,作为伪监督目标**,让模型学会把恶劣输入映射到它在干净图上会预测的同一鲁棒深度。
3. **自监督与全监督两种设定都适用**,且**架构无关**,可包裹现有深度骨干。

主要在 **nuScenes(昼/夜/雨)**与 **Oxford RobotCar** 上评测,通常以 **md4all-DD**(蒸馏+昼翻译变体)被引用。

### 后继方法
| 方法 | 改进 |
|------|------|
| **ACDepth**(arXiv:2505.12199) | 在 nuScenes 上 absRel 比 md4all-DD **夜间 +2.50%、雨天 +2.61%** |
| **Structure-Centric**(arXiv:2410.06982) | 同用蒸馏,主打结构中心的跨条件鲁棒 |
| **Robust-Depth**(arXiv:2307.08357) | 自监督,KITTI 与恶劣条件均强 |
| **扩散+对比学习**(arXiv:2404.09831) | 用扩散模型做鲁棒深度 |

> **与本仓库其它文档的呼应**:这印证了 `docs/ideas/可扩展研究方向.md` Idea 4 的判断——恶劣天气方向**已拥挤**(md4all 之后一串后继在小幅刷点),且核心瓶颈确实是 **GT 稀缺**,主流解法都是"昼→恶劣翻译 + 蒸馏",而非新架构。这条线作为独立主攻撞车风险高。

## B6. 事件相机 / 神经形态深度 —— 新兴传感器分支

### 为什么值得关注
事件相机(event camera)输出的是**异步亮度变化事件**而非帧,具有**高动态范围、低延迟、低功耗、无运动模糊**的特性——恰好对冲了传统 RGB MDE 在过曝/高速运动/暗光下的失败模式。这是原综述完全未触及的传感器维度。

### 两条技术路线

**① 脉冲神经网络(SNN)/ 神经形态**
- **Spike Transformer + 跨模态知识蒸馏**(arXiv:2404.17335, 2024):脉冲 Transformer 架构 + 从传统帧式模型向脉冲网络蒸馏。解决纯 SNN 通常欠拟合的问题——用非脉冲教师迁移知识。当前 SNN 深度的有力 SOTA 候选。
- **SNN 散焦深度**(PMC6403400):用变焦液态镜头 focus-sweep,**100 Hz、<200 mW** 功耗提取深度,生物启发的专门技术。

**② 非脉冲事件式深度(常规深度学习)**
- **Egomotion Compensation**(arXiv:2412.19067, 2024.12):用自运动补偿提升深度质量。
- **On-Device 自监督低延迟**(arXiv:2412.06359, 2024.12):端侧自监督,强调**毫瓦级低功耗、低延迟**,仅用事件。
- **跨模态一致性自监督**(arXiv:2401.07218, 2024):用跨模态一致性做自监督。

常用数据集:**MVSEC、DSEC、EventScape**。

> **本调研判断**:事件相机深度与 `docs/ideas/可扩展研究方向.md` Idea 1(HDR×深度)有深层共鸣——两者都在攻"传统 RGB 成像假设失效"的场景(过曝/暗光/高速)。事件相机用硬件天然规避动态范围问题,HDR 路线用算法补;可视为同一问题的软硬两解。**潜在交叉点**:事件 + HDR-RGB 融合深度,可能是个未被充分探索的缝(待核实新颖性)。

## A3. NeWCRFs [24] —— 神经窗口全连接 CRF(CVPR 2022)

> 接 A0 演进表。本节补原综述只列名的 NeWCRFs 的核心机制。

**它解决的问题**:CRF(条件随机场)能通过建模像素间关系来细化预测,曾广泛用于老式深度/分割流水线。但**全连接 CRF**(每像素与所有其它像素相关)计算极贵,因此经典 CRF 只能连接邻近像素,丢失长程上下文。NeWCRFs 要在**不付高昂代价**的前提下恢复全连接 CRF 的好处。

**关键思路:窗口化全连接 CRF**
不在整图上算 CRF,而是把特征图切成**窗口**,在每个窗口内跑全连接 CRF(与 Swin Transformer 自注意力同款技巧):窗口内每节点与其它所有节点全连接,保留丰富成对关系;因窗口小,计算量随图像尺寸**线性**而非二次增长。

**CRF 能量与注意力的等价(核心洞察)**:CRF 能量含**一元项**(基于节点自身特征赋深度的代价)与**成对项**(基于节点对关系的代价)。论文洞察:优化窗口内成对势的过程,与**多头自注意力/点积亲和度**计算**同形**——成对势本质成了节点特征间的亲和度,正是注意力用 query-key 积算的东西。于是把 CRF 推理实现为**神经网络模块**(neural CRF),网络直接学势函数,而非跑迭代均值场更新。

**整体架构**:① Swin Transformer 编码器提四级特征;② PPM(金字塔池化模块)头聚合全局/局部上下文产初始深度;③ NeWCRF 解码模块在四尺度各应用一次,取当前深度+对应编码特征用神经窗口全连接 CRF 细化,**由粗到细逐级**进行;④ **移位窗口**让信息跨窗口边界传播,使上下文经多层传遍全图。

> 这就是 A0 表中"CRF 全局推理太贵 → 限制在窗口内做注意力式 CRF"的完整机制:把"CRF 优化"重写为"注意力",从而可微、可端到端训练。

## B7. 多任务联合学习 —— 深度 + 语义 + 法向

### 为什么这些任务互相增强
深度、表面法向、语义在几何与上下文上**天然耦合**:**表面法向是深度的导数**,两者直接互相约束;**语义边界常与深度不连续/边缘对齐**;共享表示**正则化各任务**,降低每任务所需数据量。

### 代表方法
| 方法 | 任务组合 | 核心机制 |
|------|----------|----------|
| **Cross-task Attention**(arXiv:2206.08927) | 语义+深度+法向+边缘 | **跨任务注意力**:每任务特征互相 attend,几何与语义互相强化(室内外都验证) |
| **M2H / Multi-Mono-Hydra**(arXiv:2510.17363) | 语义+深度+边缘+法向 | **高效窗口式跨任务注意力**,用窗口注意力替代稠密全局注意力 |
| **Multi-View SAM**(arXiv:2311.00134) | 深度+分割 | 主张单视图本质受限,扩展到多视图 |

### 不确定性视角(2024 新趋势)
- **多任务不确定性量化**(arXiv:2405.17097):研究联合学深度+语义时不确定性如何表现,关乎安全关键场景。
- **证据多任务学习**(arXiv:2503.04441):引入**证据深度损失**把深度置信强度绑进联合目标——正是 `docs/ideas/可扩展研究方向.md` Idea 2(标定不确定性)的现成技术砖。

> **与原综述的关系**:原综述把多任务仅作为"挑战(2):多任务资源开销"一笔带过(视为负担)。当代视角恰相反——**跨任务注意力让多任务成为互相增强的杠杆**而非纯开销。关键转变是从"共享编码器"升级到"显式跨任务注意力"。原综述错过了这个转向。

## B8. 点图回归范式(DUSt3R / MASt3R)—— 2024 重大范式转变

> 这可能是 2024 年 3D 视觉最重要的范式转变,且**直接重定义了"单目深度"在更大图景中的位置**。原综述完全无法预见。

### DUSt3R(CVPR 2024,arXiv:2312.14132,Naver Labs Europe)
**核心范式转变**:不再走经典多视图几何流水线(特征匹配 → 基础/本质矩阵 → 三角化 → 光束法平差 BA),而是把成对重建**直接回归为点图(pointmap)**。
- **点图**是稠密 2D→3D 映射:为每个像素预测其 3D 坐标,且两视图的点图都表达在**共同参考系**(第一台相机坐标系)中。
- 这"松开了常规投影相机模型的硬约束"——**无需相机内参或位姿先验**(uncalibrated, unconstrained)。
- 用基于 **CroCo 跨视图补全预训练**的 Transformer 架构,**统一了单目与双目 3D 重建**。深度、内参、相对位姿、稠密对应等下游量都从预测点图经轻量后处理 + 全局对齐"掉出来"。

### MASt3R(arXiv:2406.09756)—— 在点图骨干上做匹配
加稠密局部特征头 + 匹配目标,把 2D 对应**锚定在 3D 点图表示**里;在某些匹配基准上比此前最佳方法绝对值高约 **30%**。

### 后继线
- **多视图扩展**(arXiv:2503.01661):把成对 DUSt3R 推广到单网络多视图,不再依赖成对对齐。
- **D²USt3R**(arXiv:2504.06264):引入 **4D 点图**处理动态场景,跨静态/动态区建立稠密对应。

### 为什么对单目深度意义重大
**关键概念跃迁**:把 3D 重建当作**对稠密点图的学习回归**,而非建在标定投影模型上的几何优化。这把脆弱的多阶段流水线塌缩成单个前馈网络,容忍未知相机与任意图像集合。

> **本调研判断**:这条线对单目深度的冲击是**根本性的**——深度不再是独立任务,而成了点图的一个**副产品**(投影点图的 z 分量)。它与 B1(米制深度)、合订本第 9 章(基础模型)合流,共同指向一个趋势:**MDE 正从"独立稠密预测任务"被吸收进"通用前馈几何基础模型"**。理解当代 MDE 必须把 DUSt3R 系作为它的上位框架来看。这是本追踪文档迄今最重要的一条补充。

## B9. 深度补全(depth completion)—— 稀疏转稠密

### 任务定位
与纯单目不同:输入 **RGB + 稀疏深度**(如 LiDAR 点),填补未测区域得稠密深度。这正是原综述引言提到的"激光雷达 5% 稀疏点 + 补全算法"那条线的现代深化。

### 空间传播网络(SPN)谱系
| 方法 | 贡献 |
|------|------|
| **CSPN / CSPN++** | 奠基:用学习的局部亲和度**迭代细化**深度 |
| **NLSPN**(非局部 SPN) | 用非局部邻居减少跨深度边界的混合 |
| **MSPN / Masked SPN**(arXiv:2404.19294, 2024) | **稀疏度自适应**:masked 设计处理变化的输入点密度,克服早期 SPN 假设固定稀疏度之弊 |

### 2024–2025 新方向:与基础模型结合
- **蒸馏单目基础模型做精细补全**(arXiv:2503.16970, 2025):把单目基础模型知识**蒸馏**进补全网络,解决稀疏监督问题——**把单目先验与稀疏 LiDAR 配对**,是当下显著新方向。
- 经典 RGB 引导基础:GuideNet(arXiv:1908.01238)、带不确定性的 RGB 引导 LiDAR 补全(arXiv:1902.05356)。

> **与 E2 工程的呼应**:深度补全本质上是 E2 节"相对→米制锚定"的另一形态——用稀疏真实测量给稠密预测注入绝对尺度与精度。**单目基础模型 + 稀疏 LiDAR 蒸馏**正成为机器人/车载落地的主流配方。

## B10. 评测方法论争议 —— "对齐"是否在掩盖真实误差?

> 这是 2025 年被正式提出的**元问题**,直击本仓库 `docs/ideas/可扩展研究方向.md` Idea 5 的要害,也解释了为何合订本 6.3 定量表要反复标注"不可直接比较"。

### 问题链条
现代 MDE 模型(Depth Anything、MiDaS 系)只预测**仿射不变深度**(差未知尺度与平移),无法直接算绝对深度指标。标准做法:打分前先**最小二乘对齐**到 GT(恢复尺度 s 和平移 t)。**争议就在这一步**。

### 三篇关键文献
| 文献 | 立场 |
|------|------|
| **How to Evaluate MDE?**(arXiv:2510.19814, 2025.10) | 把 MDE 评测列为开放问题:指标缺乏标准化、选择随意 |
| **Are We on the Right Way to Evaluate Depth Foundation Models?**(arXiv:2507.15321, 2025.07) | **核心批判**:当前评测重度依赖对齐指标,**对齐过程会掩盖真实误差、扭曲模型间比较**,可能让弱模型显得更好 |
| **第四届 MDE 挑战赛**(arXiv:2504.17787, 2025) | 当前社区标准协议:用 **2 自由度(尺度+平移)最小二乘对齐**,公平支持视差空间与仿射不变两类预测 |
| **On the Metrics for Evaluating MDE**(arXiv:2302.10007) | 检验指标排名是否真与下游 3D 检测性能相关——好的"指标是否测了有用东西"的合理性检查 |

### 实践要点
- 对仿射不变/视差预测,用 **2-DoF 最小二乘对齐**(对齐尺度+平移),匹配第四届挑战赛协议;
- 报告 AbsRel/RMSE/δ 标准指标时,**意识到对齐会吸收系统误差**;
- 尽量用**免对齐/轻对齐指标**避免掩盖误差(2507.15321 主张);
- 用**下游任务(如 3D 检测)**做合理性检查,验证指标测的是真正重要的东西。

> **本调研判断**:这条线把合订本 6.3 节的"不可比警告"上升为**领域级方法论危机**。它对 Idea 5(评测诊断基准)是强力背书——一个**按失败模式切片 + 报告免对齐指标 + 关联下游任务**的基准,正是社区当前缺的。如果要做 Idea 5,2507.15321 的"对齐掩盖误差"论点应作为核心动机。

## B11. 3D 高斯泼溅(3DGS)× 深度先验 —— MDE 的下游放大器

### 为什么 3DGS 需要单目深度
3DGS 在**稀疏视角**下严重欠约束(浮点伪影、几何扭曲、过拟合少量图)。单目深度先验成了**正则化器**:用估计深度的几何引导防止过拟合,是稀疏视角 3DGS 的标准救法。

### 两种用法(关键区分)
- **绝对深度监督**:直接拿预测深度当 GT 监督——但受单目尺度歧义之累。
- **相对/排序约束(更鲁棒)**:用单目深度的**排序关系**(谁更近谁更远)而非绝对值监督,**对尺度歧义更鲁棒**。这是更主流的选择。

### 代表工作谱
| 方法 | 贡献 |
|------|------|
| **Depth-Regularized 3DGS**(arXiv:2311.13398) | 奠基:用估计深度的几何引导防少图过拟合 |
| **SparseGS**(arXiv:2312.00206) | 深度先验 + 新深度渲染 + 浮点剪枝 + 未见视角正则,广引基线 |
| **Depth-Feature Consistency**(arXiv:2501.04628) | 用单目深度**排序**监督 patch 内深度分布一致性 + 平滑损失 |
| **Relative Depth Guidance**(arXiv:2501.11102) | 用相对(单目)深度引导修稀疏视角几何不一致,保持实时 |
| **Generalizable 3DGS from Mono Depth FM**(arXiv:2505.15185) | 直接建在单目深度**基础模型**上,免逐场景优化的可泛化 GS |

### 2025 趋势
- 多数把单目深度作绝对监督**或**排序约束(相对更抗尺度歧义);
- 越来越多把深度**配第二信号**(边缘/语义/特征一致性)而非单靠深度;
- 浮点剪枝 + 未见视角正则常与深度先验同行。

> **本调研判断**:3DGS 是单目深度**最重要的下游放大器**之一——它把"深度估计得多准"直接转化成"新视角合成质量"。这给 MDE 的精度提升提供了一个**有强落地价值的评测代理**:与其只看 AbsRel,不如看"用作 3DGS 先验时新视角 PSNR 提升多少"。这也呼应 B10——下游任务驱动的评测比纯深度指标更有意义。**Generalizable 3DGS from Mono Depth FM(2505.15185)与 B8 的 DUSt3R 系合流**,共同把单目深度推向"前馈 3D 重建的一个组件"。

## B12. VGGT —— 3D 基础模型(CVPR 2025,DUSt3R 系的集大成后继)

> 接 B8。如果说 DUSt3R 开了"点图回归"的范式,VGGT(Oxford VGG)把它推到了**3D 基础模型**的形态——这是单目深度被吸收进通用几何模型的最新、最完整证据。

### 核心(arXiv:2503.11651,vgg-t.github.io)
VGGT 是一个**前馈网络**,从 1 张、几张、到**数百张**图像,在**单次前向**中直接推断场景全部关键 3D 属性:
- **相机参数**(外参 + 内参/位姿)
- **深度图**
- **点图(point maps)**
- **3D 点轨迹(point tracks)**

关键贡献:把这一切**联合**在一个网络里做,而非依赖传统优化重的流水线(BA、迭代几何求解器)。

### 架构
多视图视觉 Transformer,核心是**交替注意力**:在**帧内注意力**(单图内)与**全局注意力**(跨所有图)间切换,让模型同时推理逐图细节与跨视图几何一致性。在大量带 3D 标注的公开数据集上训练,在相机位姿、多视图深度、稠密点云重建等多任务上达 SOTA。

### 生态
1B 参数 ONNX 导出已上 Hugging Face;后继包括几何理解与数据先验分析(arXiv:2512.11508)、多相机自驾的标定约束变体(arXiv:2511.22264)、免训练立体 Transformer(arXiv:2603.29368)。

> **本调研判断(本文档核心论点之一)**:VGGT 把 B8 的趋势钉死了——**单目深度已不再是一个独立任务,而是 3D 基础模型的一个输出头**。从 Eigen(2014)的"专用深度 CNN"到 VGGT(2025)的"深度只是数百张图前馈出的众多 3D 属性之一",这是 MDE 十年演化的终点形态。原综述(2023)站在这场转变的前夜却毫无察觉,这再次印证它是"前基础模型时代"的总结。
>
> **对选题的启示**:在 VGGT/DUSt3R 这类 3D 基础模型已统一深度/位姿/点云的格局下,纯"单图→深度"的增量改进空间被大幅压缩。`docs/ideas/可扩展研究方向.md` 的 idea 若要有生命力,应思考**如何嵌入或利用这类 3D 基础模型**(如 Idea 2 的不确定性可建在 VGGT 输出上;Idea 1 的 HDR 可作为 VGGT 的鲁棒前端),而非与之正面竞争。

## B13. 360° / 全景深度 —— 等距柱状投影的畸变之战

### 核心问题
等距柱状投影(ERP)带来两个顽疾:**畸变**(两极附近严重拉伸)与**不连续**(图像左右边界在真实场景中连续却被劈开)。2024 年的方法多在攻这一个或两个。

### 三大主流策略
| 策略 | 做法 | 代表 |
|------|------|------|
| **替代投影 + 融合** | 投到畸变小的球面/立方体贴图/切平面,再与全 FoV 的 ERP 视图融合 | 师生融合 ERP+球面(arXiv:2405.16858, CVPR 2024 W);立方体贴图 CViT(arXiv:2202.02796);MS360 切平面 |
| **几何感知算子** | 在真实球面邻域上操作 | 球面全连接 CRF(arXiv:2405.11564);球面几何 Transformer SGFormer(arXiv:2404.14979,攻两极畸变);PanoGabor(arXiv:2408.16227)用全景 Gabor 滤波 |
| **多表示融合** | 组合畸变轻的局部视图与畸变重但完整的全局视图 | MS360、师生融合 |

常用数据集:Matterport3D、Stanford2D3D。

> **与正文的呼应**:360° 深度是 NeWCRFs(A3)思想的延伸——球面全连接 CRF(2405.11564)正是把 NeWCRFs 的"窗口 CRF"搬到球面邻域。同时 SGFormer 把"几何感知注意力"做到球面,与 A3 的"CRF=注意力"洞察一脉相承。说明**几何感知算子**是贯穿平面与全景深度的统一主题。

## A4. DORN [21] —— 深度序数回归(CVPR 2018)

> 接 A0 演进表。DORN 是"回归→分类"转向的关键一步,获 Robust Vision Challenge 一等奖。

**核心思想**:不把深度估计当标准连续回归,而是**把深度范围离散化、重述为序数回归(ordinal regression)问题**。

**① 间隔递增离散化(SID,Spacing-Increasing Discretization)**:不均匀分箱,让箱**随深度增大渐宽**。给定 [α, β] 分 K 区间,阈值按**对数间隔** `t_i = exp( log(α) + (log(β)−log(α))·i/K )`,即对数空间均匀分箱(等比数列)。**动机**:深度不确定性随距离增长——近处应精确,远处本就更难。均匀离散化会过度惩罚大深度误差并浪费分辨率;SID 把细粒度箱给近处、粗粒度给远处,**降低大深度误差影响、均衡误差分布**。

**② 序数回归损失**:箱有序(箱5比箱3深),普通交叉熵丢掉这个序。DORN 对 K 个阈值产 K 个二分类器,各答"深度是否 > t_i?",损失对所有阈值求二元交叉熵之和。预测接近真值的箱比远箱惩罚小。这是其"更高精度+更快收敛"之源。

**③ 架构**:稠密 ASPP 式特征 + 全图编码器,避免反复降/上采样,保留空间细节。

> SID 让离散箱有意义(箱粒度对齐深度不确定性尺度),序数损失利用箱有序性。这是 A0 表"纯回归收敛慢、远处误差大→转有序分类"的完整机制,也是 AdaBins(A2)分箱思想的前驱。

## A5. BTS [22] —— 多尺度局部平面引导(LPG)

> 接 A0。BTS 是 NeWCRFs/GLPDepth/Binsformer 共同跟随的解码器范式。

**"From Big to Small"** 指解码流向:从稠密编码(小分辨率)瓶颈逐步引导回全分辨率。不在末端朴素上采样,而在解码器**多个尺度**放 **LPG 层**,让每尺度在合并前显式推理局部几何。

**4 参数平面系数(核心)**:每 LPG 层为每个局部 patch 预测平面 `n1·x+n2·y+n3·z=n4`((n1,n2,n3) 单位法向,n4 偏移)。为保单位法向,以球坐标 (θ,φ) 预测再转换:`n1=sinθcosφ, n2=sinθsinφ, n3=cosθ`,自动满足模为 1。

**平面→深度**:每 LPG 层在某降采样比(1/8、1/4、1/2)工作,patch 内系数共享,用归一化坐标 (u,v) 经**射线-平面相交** `c̃ = n4 / (n1·u + n2·v + n3)` 恢复逐像素深度。因系数**局部**定义,能拟合场景的分片平面结构。

**多尺度融合**:各尺度 LPG 输出与上采样解码特征**拼接**,末端卷积产全分辨率深度。

> 这是 A0 表"解码器上采样丢几何→用平面约束逐层恢复"的完整机制。BTS 的"局部平面假设"把场景显式建模为分片平面集合,后被 NeWCRFs(CRF)、Binsformer(分箱)以不同方式继承。至此 A 部分覆盖了监督主线 Eigen→DORN→BTS→AdaBins→NeWCRFs 的全部核心机制。

## A6. DPT [DPT/Ranftl] —— 稠密预测 Transformer(ICCV 2021)

> 接合订本第 9 章(基础模型时代)。DPT 是 MiDaS 卷积主干 ViT 化的标志,也是 Depth Anything 等的骨干原型。机制拆解如下。

**核心思想**:用 **ViT 替换**稠密预测编码器-解码器里的卷积主干。关键洞察:ViT 在每一层都保持**恒定空间分辨率**与**全局感受野**,不像 CNN 逐步降采样。这给出更精细、更全局一致的预测——对深度/分割这类稠密任务尤为重要。

**三阶段流水线**:

1. **Tokenization(ViT 编码器)**:图像切 patch(或经 ResNet stem 提特征,对应 ViT-Base/Large/Hybrid 变体),每 patch 成 token,加 readout(class)token,过多层 Transformer,**全程 token 数与维度不变**。

2. **Reassemble(从 token 桥回图像,DPT 的标志性步骤)**:Transformer 各阶段产的"token 袋"无内在空间结构,需重组为类图像特征图。三步:
   - **Read**:处理 readout token(忽略/相加/投影回其它 token);
   - **Concatenate/reshape**:按原空间位置把剩余 token 排回 2D 网格,恢复 H/p × W/p 的类图像表示;
   - **Resample**:用卷积投影到目标特征维并重采样到特定分辨率。**从不同 Transformer 层取 token**(浅层→高分辨率细节,深层→粗/全局特征),产多分辨率特征图。

3. **Fusion 解码器(卷积)**:用 RefineNet 式融合块逐级上采样合并多分辨率特征,产全分辨率稠密预测,顶端接任务头(深度回归/分割)。

**为什么更好**:① **每层全局上下文**——Transformer 全局 attend,即使在精细分辨率每个特征也能访问全图;CNN 只在深层(已丢空间细节后)才有大感受野;② **恒定分辨率**——ViT 不经池化丢空间粒度,重组特征保留更多细节。

变体:DPT-Large(最高精度)、DPT-Base、DPT-Hybrid(ResNet50 stem + Transformer,精度/效率平衡),后续 Intel 出 SwinV2 主干版。

> **承上启下**:DPT 是"MiDaS(混合数据集 + SSI 损失)→ ViT 骨干"的合体,直接催生了 Depth Anything 系的架构选择。理解 DPT 的 Reassemble 机制,就理解了为何 ViT 能做稠密深度——这是合订本第 9 章一笔带过的"DPT 把卷积主干换成 ViT"的完整内幕。

## B14. 可提示深度(promptable depth)—— PromptDA 与"提示"的真义

> 这是 2024 末的新方向,但**有个重要的命名陷阱必须先澄清**。

### Prompt Depth Anything / PromptDA(arXiv:2412.14015, CVPR 2025)
- **核心**:把**米制深度估计重述为"用米制提示(metric prompt)提示深度基础模型"**,在 4K 分辨率下出准确米制深度。
- **⚠️ 命名陷阱**:尽管叫"prompt",**它的提示不是文本/语言,而是低成本 LiDAR 信号(具体是 iPhone LiDAR)**作几何/传感器线索引导米制深度。如果想找文本条件深度,这是**完全不同的范式**。
- 资源:GitHub `DepthAnything/PromptDA`,HF checkpoint `prompt-depth-anything-vitl`。

### 相关:SAM × 深度(多为反方向)
多数工作是**用深度改进分割**(而非语言条件深度):
- 深度感知 SAM 做伪装目标检测(arXiv:2407.12339);
- 自提示深度感知 SAM 做实例分割,减少手工提示依赖;
- SAM4D(arXiv:2506.21547)把可提示分割扩到相机+LiDAR。

### 文本/语言条件
最接近语言条件的命中**都在分割侧而非深度**:SAM3(指令式开放词表概念分割)、CLIP-Guided SAM(arXiv 注入 CLIP 文本特征做语义条件)。

> **本调研的关键发现(对选题有用)**:检索**未找到专门做语言/文本条件深度估计的论文**。"可提示深度"目前指的是 **LiDAR 几何提示**(PromptDA),文本条件仍只活在分割文献里。这意味着 **"文本/语言条件的单目深度"可能是一个真空白**(待核实,见 F 部分新增条目)——例如"用文本指定关注区域/尺度先验来引导深度"。这与 `docs/ideas/可扩展研究方向.md` 的精神契合,且比已拥挤的方向更新。
>
> **诚实标注**:"检索为空 ≠ 确证不存在",且文本条件深度的实际效用存疑(深度是几何量,文本能提供什么有用约束?需想清楚动机才值得做)。

## B15. 测试时 / 在线适应 —— 部署后还在学

> 攻原综述"挑战(1):泛化"的另一条路。不同于基础模型靠海量数据预训练求泛化,这条线让模型**部署后在线适应当前场景/域**。

### 核心动机
预训练模型遇到新域(新城市、昼→夜、晴→雨)会掉点。**测试时适应(TTA)/ 在线适应**让网络在部署时**无需外部标注**地继续学习当前场景——天然适合自监督深度(它本就不需 GT,用帧间光度一致性即可在线自监督)。

### 代表工作
| 方法 | 思路 |
|------|------|
| **Continual Online Self-Supervised Depth**(arXiv:2103.00369) | 部署时用自监督边跑边训,从当前场景学,无需外部监督。最贴合"持续+在线+自监督" |
| **Test-time Domain Adaptation for MDE**(IEEE 10161304) | 专攻 MDE 的测试时域适应 |
| **Progressive Parameter-Efficient Adaptation**(arXiv:2312.13066) | 用参数高效适配(只调少量参数)应对自监督 MDE 的域偏移,"免全量重训" |
| **TTA for Depth Completion**(arXiv:2402.03312, 2024) | 在线 TTA,但针对深度补全(单图+稀疏深度) |

> **关键权衡(本调研判断)**:在线适应的诱惑是"零样本泛化失败时还有救",但代价是**灾难性遗忘**(适应新域时忘了旧域)与**实时训练开销**。参数高效适配(只调 LoRA 式少量参数)是当前折中。**与基础模型路线的关系**:两者互补——基础模型给强先验起点,TTA 在先验失效的长尾域上补救。可设想"Depth Anything 骨干 + 轻量在线适配头"的组合,这与 `docs/ideas/可扩展研究方向.md` Idea 2(在冻结骨干上加轻量头)同源。

## B16. 学习式深度 × SLAM —— 深度作为几何先验注入

> 接 E2(相对→米制锚定)与 B8(点图范式)。SLAM 是单目深度最重要的下游系统之一。

### DROID-SLAM(arXiv:2108.10869)—— 当代基石
核心:**循环迭代更新**相机位姿与逐像素深度 + **可微稠密光束法平差(Dense BA)层**把位姿与深度优化以可微方式绑定;支持单目/双目/RGB-D。名字 DROID = Differentiable Recurrent Optimization-Inspired Design,融合经典几何优化与深度循环网络。

### 注入学习式深度先验(2024 主线)
| 方法 | 贡献 |
|------|------|
| **Geometry-Guided Initialization**(arXiv:2406.00929, 2024) | 用**冻结的大规模预训练单目深度估计器**作自监督先验初始化稠密 BA。洞察:单目 VO/SLAM 苦于初始化与尺度歧义,强深度先验给 DBA 好起点,提升鲁棒 |
| **DROID-Splat**(GitHub ChenHoy) | 端到端:DROID 式跟踪 + 单目先验集成 + 相机标定 + 稠密高斯泼溅渲染。代表"学习深度先验 + 可微 BA + 现代稠密渲染"的融合趋势 |
| **CNN-SLAM 谱系**(arXiv:1704.03489) | 奠基:融合 CNN 稠密深度与直接单目 SLAM,在低纹理区(几何 SLAM 弱处)优先用预测 |

### 通用模式
冻结的预训练单目深度网络供给(米制或相对)先验 → 可微 BA 联合细化深度与位姿 → **解尺度歧义、提升低纹理/快速运动下的鲁棒**。

> **本调研判断(贯穿全文的收束)**:B16 + B8(DUSt3R)+ B12(VGGT)+ B11(3DGS)四条线指向同一个终局——**单目深度正被吸收为"前馈/可微几何系统"的一个组件**:在 SLAM 里是 BA 的先验,在 DUSt3R/VGGT 里是点图的副产品,在 3DGS 里是渲染的正则。**"独立的单图→深度"作为研究终点的时代正在结束;它的未来在于作为更大几何系统里的可靠先验。** 这是本追踪文档对整个领域走向的核心判断,也应是任何 MDE 选题的时代背景。

## B17. 动态场景深度 —— 运动物体破坏的静态假设

> 接 A1(Monodepth2 自动遮罩)。自监督深度的核心软肋是动态物体,这一节追踪 2024 的系统性解法。

### 核心问题
自监督深度训练假设**静态场景 + 运动相机**。运动物体打破它,造成**相机自运动与物体运动的歧义**——同速物体会"塌陷成无穷远"(A1 的自动遮罩只是粗略缓解)。

### 2024 代表方法
| 方法 | 机制 |
|------|------|
| **FlowDepth**(arXiv:2403.19294) | **动态运动流模块(DMFM)**:机制式解耦光流,warp 动态区修复光度不匹配 |
| **Manydepth2**(arXiv:2312.15268) | **运动引导代价体深度网**:动态物体与静态背景上都准,且保持高效 |
| **Mining Supervision for Dynamic Regions**(arXiv:2404.14908) | 专攻动态区**监督质量**,不再纯靠图像重建损失(它在运动处失效) |
| **Decomposed Object-aware Motion & Depth**(arXiv:2403.05895) | 联合学 3D 运动与深度,加解耦的物体感知运动模块 |

### 两条贯穿 2024 的主线
1. **光流解耦**为刚性(自运动引起)与动态分量,使动态部分被单独处理;
2. **用基础模型/预训练分割器**自举物体提议或掩码,替代早期手工运动先验(如 Dense Monocular Motion Segmentation,arXiv:2406.18837)。

> **本调研判断**:动态物体是自监督深度从 Monodepth2(2019)至今**始终未根治**的痼疾。2024 的转向值得注意——**从"启发式遮罩"升级到"基础模型分割 + 光流解耦"**,用更强的语义/运动先验显式建模动态。这与 B7(多任务)、B16(SLAM)共享同一思路:**用外部强先验补几何约束的盲区**。

## B18. 深度知识蒸馏 —— Depth Anything 的数据引擎内幕

> 接合订本第 9 章。第 9 章说 DA "靠 6200 万无标注图",这一节补**为什么以及怎么做**的机制内幕。

### 核心:数据引擎 + 师生伪标注(DA V1,arXiv:2401.10891)
动机:有标注深度数据稀缺昂贵(需深度传感器/双目/SfM),而无标注单目图近乎无限。流水线把这份充裕转成训练信号,三阶段:① **在约 1.5M 标注图上训教师**得到自举起点;② **数据引擎用教师在 ~6200 万无标注图上跑产稠密伪标签**——这步"显著扩大数据覆盖",是鲁棒与泛化的主杠杆;③ **在 标注 + 伪标注 上训学生**。

### 关键:为什么朴素伪标注不够
论文核心发现:**单纯加伪标注图并不能提升学生**。两个设计才让无标注数据起效:
- **对无标注图施强扰动**:学生吃重度增强(色彩畸变、CutMix)版本,而伪标签目标来自**未扰动**输入。逼学生学鲁棒表示而非照抄教师——"挑战学生";
- **辅助语义约束**:与冻结 DINOv2 式编码器做**特征对齐**,继承语义先验,保持深度语义连贯。

### V2 怎么改配方(arXiv:2406.09414)
保留师生/伪标注骨架,改数据策略:**用合成图替代标注真实图训教师**(干净精确无传感器噪声)→ **放大高容量教师(ViT-G)**产伪标签 → **用大规模伪标注真实图弥合合成→真实差距**,让学生从合成教学却泛化到真实照片。

> **贯穿判断**:DA 的"挑战学生 + 语义对齐"是原综述"半监督"思想的工业级放大——用海量无标注数据替代昂贵 GT。它与 B9(蒸馏单目模型做补全)、B15(在线适应)共同说明:**蒸馏与伪标注是当代 MDE 利用无标注数据的核心引擎**。

## B19. ZoeDepth 机制深拆 —— 米制分箱 + attractor

> 接合订本 9.3。第 9 章只说 ZoeDepth"两阶段弥合相对与米制",这一节补**米制分箱头与 attractor 机制**的内幕。

### 两阶段架构
- **阶段一:相对深度预训练**。从 MiDaS 式网络(BEiT-L/Swin 等 Transformer 主干)起步,在 **12 个数据集**上用尺度不变相对深度训练。相对深度给出场景的序与比例结构(谁近谁远)但无真实尺度;跨多数据集训练带来强跨域泛化。
- **阶段二:米制微调**。把预训练的 MiDaS 主干在 **2 个米制数据集**(室内 NYU、室外 KITTI)上微调,补上真实尺度(米)。旗舰模型 **ZoeD-M12-NK**(M12=12 个相对数据集 + N+K=NYU+KITTI)。

### 米制分箱模块 + attractor 机制(核心创新)
轻量头挂在解码器上:
- **每域一个独立米制分箱模块**(室内一个、室外一个),让单模型兼顾两类场景;
- 不直接预测连续深度,而**把深度范围离散成箱、以箱中心加权和**预测深度(承 AdaBins/LocalBins 的自适应分箱);
- **attractor(吸引子)机制**是新颖的箱调整设计:箱中心不固定,"吸引子"把箱中心**拉向深度值可能集中的区域**;箱以**乘性方式调整、跨解码层细化**,使离散化适配每图实际深度分布,比均匀分箱更锐更准。

### 推理时
**潜在分类器(router)**决定用哪个米制分箱头,自动把输入图路由到室内或室外模块。这实现零样本迁移:相对深度主干广泛泛化,域专用米制头供给正确尺度。

> **承上启下**:ZoeDepth = MiDaS 相对预训练(合订本 9.2)+ AdaBins 分箱(A2)+ attractor 新机制 + 域路由。它把本文档多条线**缝合**成一个系统,是"相对→米制"工程裂缝(E2)的代表性闭环解法。

## B20. 专门域深度 —— 内窥镜 / 手术 / 水下

> 接原综述 [61][62](作者团队的内窥镜方向)。这是把通用深度模型适配到**敌对成像环境**的活跃分支,也是原综述作者团队的主场。

### 共性挑战(为何通用模型失效)
- **镜面高光 + 非朗伯湿组织表面**混淆基于纹理的深度线索;
- **缺 GT 深度**:真实手术场景难装深度传感器,监督数据稀缺 → 逼向自监督/零样本/合成数据;
- **尺度歧义**:恢复米制/绝对尺度反复成难题;
- **域差**:自然图基础模型与手术/水下外观分布差距大;
- **实时轻量**:术中直播需求。

### 2024 方法按策略分组
| 策略 | 代表 |
|------|------|
| **基础模型适配** | Surgical-DINO(arXiv:2401.06013,**LoRA 式低秩适配 DINOv2**,冻骨干训小适配层);Depth Anything 内窥镜适配(arXiv:2409.07723) |
| **自监督 + 时序一致** | Transferring Relative Depth with Temporal Consistency(arXiv:2403.06683,用视频帧时序自监督迁移现成相对深度模型) |
| **合成先验 + 域不变学习** | 镜面手术环境合成先验(arXiv:2512.23786);域不变特征 + 潜在一致性做绝对深度(arXiv:2511.02247) |
| **尺度/米制恢复** | 内窥镜零样本米制深度(arXiv:2509.18642);几何建模做尺度感知(arXiv:2408.07266) |
| **效率** | 轻量 CNN-Transformer 混合(arXiv:2308.02716) |

> **本调研判断**:专门域(尤其内窥镜)是检验"基础模型时代方法论"的绝佳试验场——它把 B18(蒸馏)、B15(适配)、B4(非朗伯)、B2(时序)的挑战**全部叠加**。当下主流答案高度一致:**冻结基础模型 + 参数高效适配(LoRA/adapter)+ 合成/时序自监督补 GT**。这也呼应原综述作者团队 [61] 的 appearance flow 工作——医学场景始终是该团队的核心关切。**水下深度**与内窥镜共享(光衰减、散射、尺度歧义、GT 稀缺)的挑战,方法可概念迁移,但本轮检索未覆盖水下专门文献(待补)。

## B21. 水下单目深度 —— 补 B20 缺口

> 补 B20 末尾标注的水下文献缺口。水下与内窥镜共享(光衰减、散射、尺度歧义、GT 稀缺)的挑战,但有自己的成像物理。

### 特有挑战
- **光衰减(attenuation)**:不同波长光在水中衰减率不同(红光先消失),造成色偏;
- **后向散射(backscatter)**:悬浮颗粒散射光,产生雾状朦胧;
- **GT 稀缺**:水下难装深度传感器,真实深度数据极少 → 逼向合成/扩散生成。

### 2024 代表方法
| 方法 | 贡献 |
|------|------|
| **Atlantis**(CVPR 2024,arXiv:2312.12471) | 旗舰:用 **Stable Diffusion 生成带深度监督的照片级水下图**,克服真实水下深度数据稀缺 |
| **真实基准 + 合成微调**(arXiv:2507.02148) | 专攻**米制**深度,指出光衰减/散射致可靠性受限,配真实基准 + 视觉基础模型合成微调 |
| **光场图像恢复 + 深度**(MDPI 12/6/935) | 把**光衰减/散射系数显式纳入成像模型(IFM)**,补此前 IFM 忽略这些效应之缺 |
| **WaterGAN**(arXiv:1702.07392) | 奠基:无监督生成成对"原始水下/真彩空气"图 + 深度,做实时单目色彩校正 |

> **本调研判断**:水下深度是\"物理驱动\"思想(`docs/ideas/可扩展研究方向.md` Idea 1)的天然试验场——成像物理(衰减/散射)**必须**显式建模,否则深度无解。Atlantis 用扩散生成补 GT 的思路,与 B18(Depth Anything 合成数据)、B4(透明反光合成数据)一脉相承:**当真实 GT 无法获取时,合成/生成数据是当代统一答案**。水下 + 内窥镜(B20)共同构成\"敌对成像环境深度\"这一子领域,方法高度互通。

## B22. MonST3R / 4D 点图 —— DUSt3R 系扩展到动态场景

> 接 B8(DUSt3R)、B12(VGGT)、B17(动态场景)。这是点图范式攻克动态场景的关键一步,把"3D 点图"推广到"4D 点图"。

### MonST3R(arXiv:2410.03825,monst3r-project.github.io)
**核心问题**:DUSt3R 为静态场景设计,假设帧间世界不动,运动破坏此假设。
**简单而有效的洞察**:**为每个时间步估计一个独立点图**,而非强求单一一致几何。按时间戳表示几何,运动物体自然被处理——每帧有自己的 3D 点云。
- 从动态视频产出**时变动态点云 + 每帧相机位姿与内参**,基本前馈。
- **实践贡献**:不设计新架构,而主要**在少量合适的动态数据集上微调现有 DUSt3R 网络**,证明点图表述能以有限数据迁移到动态内容。下游支持动态场景的视频深度与相机位姿估计。

### 4D 点图后继浪潮
| 方法 | 贡献 |
|------|------|
| **D²USt3R**(arXiv:2504.06264) | **4D 点图**,跨静态/动态区从视图对建稠密对应 |
| **Dynamic Point Maps/DPM**(arXiv:2503.16318) | 把标准点图推广为 4D,支持运动分割、场景流、3D 物体跟踪 |
| **Driv3R**(arXiv:2412.06777) | 面向自驾,稠密 4D 重建比需全局对齐的方法快约 **15×** |
| **训练免方法**(arXiv:2503.24391) | 发现 **DUSt3R 注意力层已编码相机与物体运动**,免训练解耦做动态区分割 |
| **C4D**(arXiv:2510.14960) | "4D Made from 3D",从单目视频经双重对应恢复每帧稠密点云+位姿+内参 |

> **本调研判断(收束 DUSt3R 主线)**:从 DUSt3R(静态点图,B8)→ MonST3R(逐时间步点图)→ D²USt3R/DPM(4D 点图)→ VGGT(数百图 3D 基础模型,B12),这条线在两年内把\"几何重建\"从静态成对推到了**动态、多视图、前馈、统一**。单目深度在此完全成为\"4D 点图在某时刻某视角的投影 z 分量\"。**这再次印证本文档核心论点(B16 收束)**:单目深度的未来不是独立任务,而是动态 4D 几何基础模型的一个导出量。一个值得注意的趋势是 arXiv:2503.24391 的发现——**预训练点图模型的注意力层已隐式编码运动**,意味着许多\"动态深度\"能力可能已潜伏在静态模型里,只待解耦提取(待核实其普适性,见 F 部分)。

## A7. Binsformer [56] —— DETR 式自适应分箱(补全监督主线)

> 接 A2(AdaBins)、A0。Binsformer 是"分箱"路线的 Transformer 化集大成,补全监督主线最后一块。

**它要改进 AdaBins 的两点**:① 自适应分箱的**恰当生成**(箱中心应逐图自适应);② 概率分布与分箱预测间的**充分交互**(AdaBins 里两者近乎独立计算)。

**核心概念跃迁:把自适应分箱重述为集合到集合预测(set-to-set)**,这正是 DETR 在目标检测里所做的事。
- **DETR 式 query 机制**:DETR 用一组可学习"object query"过 Transformer 解码器,每个 query 学着 attend 图像特征并产一个输出(一个物体)。Binsformer 直接借用——用一组**可学习的 bin query**,每个 query 过解码层、attend 多尺度图像特征,输出**一个自适应箱的定义**。于是箱不再手工设定或由简单模块产出,而是**以输入图为条件、由 Transformer query 解码出来**。
- **两个预测分支**:从解码器输出产 ① **箱中心**(回归侧,逐图深度锚点);② **逐像素概率分布**(分类侧,每像素属各箱的程度)。最终深度 = 箱中心按该像素概率分布的**线性组合**(分类-回归混合)。
- **交互组件**:箱中心与像素概率不再孤立计算,而通过 Transformer **共享同一组解码 query 表示**,使分箱与分类分数被联合告知。

**DETR 框架的优势**:箱数与特征分辨率解耦(当集合预测处理);多尺度特征交互经交叉注意力自然实现;自适应箱真正响应逐图场景内容(每 query attend 该图特征)。论文还用辅助场景分类损失。

> **A 部分监督主线完整收束**:Eigen(多尺度回归,A 隐含)→ DORN(序数分类,A4)→ BTS(局部平面,A5)→ AdaBins(自适应分箱,A2)→ NeWCRFs(窗口 CRF,A3)→ **Binsformer(DETR 式分箱,A7)**;骨干侧 DPT(A6)提供 ViT 化。这条线的两大主题——**"回归↔分类"之争**与**"几何结构保持"**——在 Binsformer 处合一:它用 DETR query 同时优雅处理了"如何生成箱"(分类结构)与"箱如何随图自适应"(逐图几何)。

## B23. 概率/分布式深度 —— 超越点估计

> 接 B7(证据多任务)、`docs/ideas/可扩展研究方向.md` Idea 2(标定不确定性)。这一节系统梳理"输出深度分布而非单值"的方法谱,是不确定性方向的技术底座。

### 为什么要分布而非点估计
点估计给"深度=5.2m"但不说"我有多确信"。安全关键场景(自驾、机器人、手术)需要**逐像素不确定性**。不确定性分两类:
- **偶然不确定性(aleatoric)**:数据固有噪声(透明/反光/远处),**单次前向**可估;
- **认知不确定性(epistemic)**:模型对未见数据的无知,传统需**集成或 MC-Dropout**(多次前向)。

### 方法谱
| 方法 | 思路 |
|------|------|
| **MonoProb**(arXiv:2311.06137) | 自监督,输出**可解释不确定性**(其值直接反映预测的期望误差) |
| **Generating Probabilistic Depth**(arXiv:1906.05739) | 奠基:用条件 GAN 采样近似输出场景深度的**完整概率分布** |
| **分类法做 UQ**(arXiv:2202.12369) | 把深度回归重述为**分类**以获不确定性量化(呼应 DORN/Binsformer 的分箱) |
| **梯度法 UQ**(arXiv:2502.05964) | 用**梯度信号**量化来自动态物体/阴影等图像歧义的不确定性 |
| **Uncertainty from Motion**(MIT EEMS) | 偶然不确定性单次前向可估,认知不确定性传统需多次;提出用**运动**作更廉价信号 |
| **神经形态分布**(arXiv:2605.10675,⚠️ID 待核实) | 从事件流预测逐像素深度分布,比较高斯/对数正态/证据学习框架 |

### 关键联系
**分箱方法天然是概率式的**:AdaBins/Binsformer 输出的"逐像素 Softmax 分布"本身就是一个深度分布,取期望得点估计、取熵/方差即得不确定性。这是 A2/A7 与本节的隐藏纽带——**分类-回归混合架构免费附带了不确定性**,只是早期工作没充分利用。

> **本调研判断**:这正是 `docs/ideas/可扩展研究方向.md` Idea 2 的技术地基。**证据回归(evidential)**(B7 的 2503.04441、神经形态的证据框架)是当前最轻量的标定方案——单次前向出深度+标定方差,无需集成。把它接到 Depth Anything/Binsformer 的分箱输出上,是"标定不确定性基础模型"最可行的起点。**与 B10(评测争议)呼应**:有了标定不确定性,就能做免对齐的、报告置信区间的评测,缓解"对齐掩盖误差"。

## A8. GLPDepth [55] —— 全局-局部路径 + Vertical CutDepth(补全 A 部分)

> 接 A0、A5(BTS 路线后继)。GLPDepth(GLPN)是 BTS 路线上轻量高效的一支,补全 A 部分最后一个具名监督方法。

**架构(arXiv:2201.07436)**:
- **分层 Transformer 编码器**(捕获并传递全局上下文),逐步降空间分辨率、扩感受野——对像素级预测关键;
- **轻量但强的解码器**,保留局部连通性;
- **选择性特征融合(SFF)**:融合模块让解码器**选择性组合局部与全局特征**,论文将性能提升归因于此。

**Vertical CutDepth(为深度定制的数据增强)**:在 CutDepth 基础上**沿垂直轴施加裁剪**,契合自然场景中深度**自上而下变化**的规律(天空远、地面近),提升泛化与鲁棒。

> **A 部分全景收束**:GLPDepth 代表 BTS 路线在**效率/泛化**维度的优化(轻量解码 + SFF + 针对性增强),与 NeWCRFs(精度优先)、Binsformer(分箱)构成 BTS 后继的三个不同侧重。至此 A 部分(A0–A8)完整覆盖了原综述监督主线点名的全部方法:Eigen/DORN/BTS/AdaBins/NeWCRFs/GLPDepth/Binsformer + 骨干 DPT,以及自监督基线 Monodepth2。**A 部分作为"原综述只列名、本文档补机制"的目标已达成。**

## B24. 深度 × 世界模型 / 视频生成 —— 2025 的几何回归生成

> 接 B11(3DGS)、B12(VGGT)。这是单目深度在生成式 AI 时代的新下游,也是"几何回归生成"趋势的最前沿。

### 2025 趋势:几何从"下游消费者"变为"对等建模目标"
过去深度是生成视频的下游消费者;2025 的转向是**把 3D 几何直接融进视频生成器**,而非事后拼接。三种实现路径:
| 路径 | 代表 | 做法 |
|------|------|------|

> **本调研判断(全文最终收束)**:B24 把本文档反复论证的趋势推到极致——**单目深度正从"独立预测任务"变为"生成式世界模型的几何建模目标/约束"**。结合 B8/B12/B22(点图与 4D 几何基础模型)、B11/B16(3DGS/SLAM 下游),可以画出单目深度的完整时代轨迹:

> - **2019–2024**(本文档 9 章 + B1/B18):基础模型 + 零样本泛化,深度成通用先验;
> - **2024–2025**(B8/B12/B22/B24):被吸收进前馈/可微/生成式的**几何基础模型与世界模型**,深度成为 4D 几何的一个导出量或建模约束。
>
> **对选题的终极启示**:在世界模型/几何基础模型主导的格局下,纯单目深度的增量改进价值持续走低;真正有生命力的方向是**(a) 攻基础模型仍失败的物理边界**(HDR/透明/恶劣天气/水下——Idea 1)、**(b) 给几何/世界模型提供可靠的不确定性与标定**(Idea 2)、或 **(c) 重做被对齐掩盖的评测体系**(Idea 5、B10)。这三者都不与基础模型正面竞争,而是补其盲区——这是本追踪文档对所有 MDE 选题的最终建议。

## B25. 高分辨率深度 —— 攻原综述"挑战 3:高分辨率内存瓶颈"

> 原综述把"高分辨率内存瓶颈"列为挑战却没给方向。这一节补当代的核心解法:**分块(tiling/patch)+ 融合**。

### 核心矛盾
基础深度模型多在 ~1 兆像素分辨率训练,直接喂 4K 图要么爆显存、要么细节糊。**分块策略**:在高分辨率裁剪块上跑模型捕获局部细节,再与全局估计融合保持结构一致。

### 代表方法谱
| 方法 | 贡献 |
|------|------|
| **Boosting Monocular Depth**(arXiv:2105.14021, 2021) | 奠基:**内容自适应多分辨率合并**,把预训练网络推过其原生 ~1MP 输出上限,恢复细粒度细节 |
| **PatchFusion**(CVPR 2024,arXiv:2312.02284) | **端到端分块框架**,建在 ZoeDepth 上,RMSE 改善 17.3%/29.4%;**模型无关**(可当 wrapper);核心含 patch 融合网络、Global-to-Local(G2L)模块、一致性感知推理(CAI) |
| **Seamless Patchwise Refinement**(arXiv:2503.22351, 2025) | 攻分块法两大软肋:**重组时的深度不连续伪影** + 测试时低效 |

### 反复出现的两大挑战
1. **接缝/不连续伪影**:patch 边界处深度跳变;
2. **推理开销**:处理众多 tile 的成本。

> **本调研判断**:高分辨率深度是个**纯工程但高价值**的方向——它不改变深度估计的本质,只解决"如何把强模型放大到 4K"。PatchFusion 的"模型无关 wrapper"思路尤其实用:任何基础模型(Depth Anything/ZoeDepth)都能套上它升到高分辨率。这与 `docs/ideas/可扩展研究方向.md` 的精神一致——**不与基础模型竞争,而做它的放大器**。接缝伪影问题本质是 B10(评测)与边界质量的交叉,可作为一个聚焦的小切口。

## B26. 伪 LiDAR / 单目 3D 目标检测 —— 自驾核心下游

> 接 E2(米制锚定)、B16(SLAM)。这是单目深度在自动驾驶里最重要的下游,也揭示了一个深刻的"表示"洞察。

### 核心概念
LiDAR 贵、相机便宜但缺深度。**伪 LiDAR**:把图像深度图**转成点云表示**(模仿 LiDAR 信号),再喂给 LiDAR 式 3D 检测架构。

### 奠基洞察(arXiv:1812.07179)——表示比精度更重要
关键发现:**数据表示比深度精度本身更关键**。标准 CNN 在图像空间的深度图上操作,卷积会**混合物理深度差异极大的像素**;转成伪 LiDAR 点云让检测器在**3D 物理空间**推理,带来巨大精度提升。这是个深刻的洞察——**同样的深度,换个表示(图像网格→点云)就大幅改变下游可用性**。

### 方法演进
| 方法 | 贡献 |
|------|------|
| **Pseudo-LiDAR++**(arXiv:1906.06310) | 改进底层深度(尤其立体),刷 KITTI |
| **Refined Monocular PseudoLiDAR**(arXiv:1911.09712) | 治理单目深度生成的高密度点云的噪声与歧义,低开销 |
| **Pseudo-Stereo**(arXiv:2203.02112) | 从单图**合成虚拟立体视图**,改善伪 LiDAR 管线的深度感知 |
| **Per-Object Depth**(arXiv:2206.03666) | 聚焦**物体级**深度精度(检测最敏感处),而非全场景 |
| **Joint Datasets Training**(arXiv:2310.00920) | 跨多数据集训练,降低对昂贵 3D LiDAR 标注的依赖 |

> **本调研判断**:伪 LiDAR 的"表示 > 精度"洞察对整个 MDE 领域有启示意义——它说明**深度估计的价值高度依赖下游如何消费它**。这呼应 B11(3DGS)、B10(评测)的共同主题:**评判深度好坏不应只看 AbsRel,而要看在目标下游(3D 检测/新视角/SLAM)里的实际效用**。对自驾,"物体级深度精度"(Per-Object)比全图平均指标重要得多——这是 `docs/ideas/可扩展研究方向.md` Idea 5(失败模式/任务导向评测)的又一有力论据。

## B27. 立体基础模型 —— 单目先验反哺立体(2025)

> 接 B12(VGGT)、合订本第 9 章。一个有趣的回环:单目深度基础模型如今反过来成为**立体匹配**的先验,催生"零样本立体"。

### 背景:立体匹配的泛化困境
传统立体匹配跨域泛化差,需逐域微调。2025 的突破是把视觉基础模型的零样本泛化带进立体。

### 代表方法
| 方法 | 贡献 |
|------|------|
| **FoundationStereo**(CVPR 2025,arXiv:2501.09898) | 立体深度的基础模型:**大规模合成数据 + 单目视觉基础模型先验**,无需逐域微调即在未见真实数据上出准确视差。立体零样本新标杆 |
| **Fast-FoundationStereo**(arXiv:2512.11130,⚠️ID 待核实) | 提速后继,比 FoundationStereo 快 **10×+**,接近实时,精度接近 |
| **Stereo Anywhere**(arXiv:2412.04472) | 耦合几何立体约束 + 单目深度 VFM 先验,专攻**立体或单目单独都失败处**(无纹理/镜面) |
| **DEFOM-Stereo**(arXiv:2501.09466) | 在**深度基础模型**之上建立体匹配,零样本泛化显著强于此前 SOTA |

### 共同策略
**用单目深度基础模型作先验 + 大规模(常合成/混合)训练**,实现鲁棒跨域泛化。后续分两支:推精度(Stereo Anywhere、DEFOM)、推效率(Fast-FoundationStereo)。

> **本调研判断(一个深刻回环)**:这条线揭示了 MDE 演化的一个意外回环——**单目深度从立体的"穷人替代"(原综述视角),反过来成了立体的"泛化引擎"**。Stereo Anywhere 的"哪里单目或立体单独失败就用另一个补"尤其点睛:单目(强语义先验、无尺度)与立体(强几何、有尺度)**互补**,基础模型时代终于能把两者优雅融合。这与 B16(深度×SLAM)、B26(伪 LiDAR)共同说明:**单目深度的现代价值常在于作为更大几何系统的先验组件**,而非独立终产物——这是贯穿本文档的核心判断的又一例证。

## B28. 深度从散焦(DfD)/ 聚焦(DfF)—— 经典线索的深度学习复兴

> 接原综述引言提到的"聚焦/散焦"线索(传统深度线索之一)。深度学习让这条经典线索重获新生。

### 物理基础
散焦模糊程度与物体到焦平面的距离相关——**模糊即深度线索**。给定多张不同对焦的图(焦栈 focal stack)或单张带散焦的图,可反推深度。这是**物理驱动**的天然范例(成像物理直接给出深度约束)。

### 代表方法
| 方法 | 贡献 |
|------|------|
| **DDFS**(arXiv:2202.13055,IJCV 2023) | 焦栈喂网络 + **显式散焦模型**,跨相机设置(光圈/焦距)泛化 |
| **Aberration-Aware DfD**(arXiv:2402.18175, 2024) | 自监督估计**空间变化 PSF**,关注镜头像差 |
| **Event Focal Stack**(arXiv:2405.06944, 2024) | 用**事件相机焦栈**做单目深度,避开立体匹配(呼应 B6 事件相机) |
| **Self-Supervised DfD**(arXiv:2303.10752) | 全自监督的散焦深度 |
| **3DGS Siamese Defocus**(arXiv:2409.12323, 2024) | 建在 3D 高斯泼溅上的散焦深度(呼应 B11) |

### 与主流单目深度的关系
DfD/DfF 需**特殊采集**(焦栈/已知光圈),不是纯单图,但**深度线索物理可靠**(不像纯单目靠学习先验"猜")。手机**双像素(dual-pixel)**传感器是其落地载体(本轮检索未充分覆盖,待补)。

> **本调研判断**:DfD 复兴印证了原综述展望的"物理驱动深度学习"方向**确实成立**——把成像物理(散焦 PSF)显式建进网络,得到比纯学习先验更可靠、更可泛化的深度。这与 B21(水下成像物理)、`docs/ideas/可扩展研究方向.md` Idea 1(HDR 物理前端)同属一个大主题:**当几何/学习先验不可靠时,回到成像物理是稳健的出路**。DfD 是这个主题里最成熟的一支,可作为 HDR×深度等新物理驱动方向的方法论参考。

## B29. NeRF / 神经场中的深度监督 —— 单目深度作几何先验

> 接 B11(3DGS)。在 3DGS 之前,NeRF 是单目深度先验的另一大下游;两者共享"稀疏视角下用深度约束几何"的逻辑。

### 演进三阶段(处理深度先验的递进)
| 阶段 | 方法 | 思路 | 软肋 |
|------|------|------|------|
| ① 稀疏精确深度 | **DS-NeRF**(arXiv:2107.02791) | 用 SfM/COLMAP 稀疏 3D 点作**免费深度监督**约束射线终止;少视角更好、训练快 2–6× | 稀疏点只约束少数射线 |
| ② 稠密补全深度 | **Dense Depth Priors**(arXiv:2112.03288) | 把稀疏 SfM 深度**补全成稠密**再作先验,覆盖全图 | 引入预测误差 |
| ③ 不确定性感知稠密深度 | EMD 监督(arXiv:2403.13206)、单目深度适配(arXiv:2305.19201)、深度+法向带标准差(arXiv:2407.05666) | 承认先验**含噪/尺度歧义**,用分布式(EMD)或标准差加权监督 | — |

### 核心难题(贯穿)
单目深度**相对准、尺度/平移未知**,naive 当伪 GT 会引入偏差。前沿在于**处理尺度歧义与不确定性**,而非假设先验正确——这与 B11(3DGS 用相对/排序约束更鲁棒)、E2(米制锚定)完全同构。

> **本调研判断**:NeRF 与 3DGS 的深度监督史是同一个故事——**单目深度作为"几何正则化器"注入神经场重建**,核心挑战始终是"如何用一个尺度歧义、含噪的相对深度先验,而不让它的错误污染重建"。答案演化为"分布式/不确定性感知监督"。这再次印证 B23(概率深度)的价值:**带标定不确定性的深度先验,正是这些下游(NeRF/3DGS/SLAM)最需要的**——它们能据此对先验"该信多少"加权。这是 `docs/ideas/可扩展研究方向.md` Idea 2 的又一下游动机。

## B30. 深度 × 法向 / 内在图像 —— 几何量的相互推导

> 接 B7(多任务联合)。这一节聚焦深度与表面法向、内在图像的**几何耦合**——它们不是独立任务,而是同一几何的不同侧面。

### 深度 ↔ 法向(导数关系)
表面法向是深度的空间导数,两者数学上直接互推:
| 方法 | 机制 |
|------|------|
| **Metric3D v2**(arXiv:2404.15506) | **联合深度-法向优化模块**:从米制深度蒸馏知识,让法向估计器学到超越显式法向标签的能力;单模型零样本兼顾两任务 |
| **Adaptive Surface Normal Constraint**(arXiv:2402.05869, 2024) | 自适应法向约束**强制深度与法向预测的跨属性一致性**,解决几何上下文捕获不可靠致一致性弱 |
| **Normal-Distance Assisted**(arXiv:2309.10592) | 加法向-距离头(逐像素法向 + 平面到原点距离),**由二者导出深度**,法向与距离互相正则 |

### 深度 ↔ 内在图像分解(双向互助)
**Intrinsic Image Decomposition for Robust Depth on Reflective Surfaces**(arXiv:2503.22209)双向运行:**深度**帮多视图一致性对齐坐标系以助分解;**分解**识别反光区(否则破坏深度)。这直接攻 B4(非朗伯反光)的失败模式。

> **本调研判断**:深度、法向、内在图像(反照率/着色)本质是**同一物理场景几何/光学的不同投影**。当代趋势是**用它们的物理耦合关系互相约束/监督**(法向=深度导数、反光区识别助深度)。这与 B7(跨任务注意力)是一体两面:B7 是"用注意力让任务互相 attend",B30 是"用几何/物理关系让任务互相推导"。后者更硬(有物理方程支撑),也更可靠。**对选题**:在反光/透明等失败模式上(B4),引入内在图像分解显式建模反射,可能比纯深度网络更治本——这是 `docs/ideas/可扩展研究方向.md` 失败模式攻坚(Idea 1/5)的一条具体技术路径。

## B31. 深度 × 光流 / 场景流联合估计 —— 几何运动的统一分解

> 接 A1(Monodepth2)、B17(动态场景)。深度、光流、自运动在自监督框架里**天然耦合**——它们共同构成了"像素如何在帧间移动"的完整解释。

### 核心思想:把场景运动分解为刚性 + 非刚性
总场景运动 = **刚性分量**(相机自运动 + 静态深度可解释)+ **残差非刚性分量**(独立运动物体)。自监督联合学习正是利用这个分解互相约束。

### 演进
| 方法 | 贡献 |
|------|------|
| **Joint Stereo Videos**(arXiv:1810.03654, 2018) | 奠基:用深度+自运动+流**把场景分解为运动区** |
| **无真实标签学三者**(arXiv:2203.15089, 2022) | 用投影几何 + 视图合成**联合估计光流+深度+场景流**,无需真实标签 |
| **Ego-Motion Rigidity 场景流**(arXiv:2309.01296, 2023) | 显式利用**自运动刚性**从两帧出场景流 |
| **FlowDepth**(arXiv:2403.19294, 2024) | **动态运动流模块(DMFM)**解耦光流、warp 动态区修复光度不匹配(亦见 B17) |
| **Zero-Shot Scene Flow in the Wild**(arXiv:2501.10357, 2025) | 在 3D 端点误差上超 SOTA,零样本泛化到随手拍视频 |

### 贯穿主线
从"假设全场景刚性"到"显式分离动态区避免光度不匹配"。**深度提供静态几何,光流提供 2D 运动,自运动连接两者,场景流是 3D 运动的统一表示**——四者互为约束。

> **本调研判断**:深度与光流是自监督几何学习的"双生子"——光度一致性损失同时是两者的监督信号。FlowDepth(2024)的解耦思路与 B17(动态场景)、B30(深度×法向)共享同一方法论:**用辅助几何量(流/法向/分解)显式建模主任务(深度)的失效因素**。**对选题**:联合估计天然产生"运动掩码",可作为 B23(不确定性)的一种结构化先验——动态区即高不确定区。这又指向 `docs/ideas/可扩展研究方向.md` Idea 2。

## B32. 雷达-相机 / 热成像多模态融合深度 —— 攻恶劣条件的硬件互补

> 接 B5(恶劣天气)、B6(事件相机)、E2(米制锚定)。当 RGB 在夜间/雨雾失效时,雷达与热成像提供硬件级互补。

### 为什么多模态
- **毫米波雷达**:穿透雨雾、直接给**稀疏但真实的米制深度**(天然解尺度歧义),但极稀疏、噪声大;
- **热红外**:夜间/零光照可成像(测温度而非可见光),补 RGB 暗光失效;
- **融合动机**:用稀疏雷达的米制尺度 + 相机的稠密结构,或用热成像替代失效的 RGB。

### 代表方法
| 方法 | 模态 | 贡献 |
|------|------|------|
| **RIDERS**(arXiv:2402.02067, 2024) | 毫米波雷达 + **单目热红外** | 在可见光/近红外 LiDAR 都难的条件下出鲁棒米制深度。三要素齐全:雷达融合 + 热成像 + 夜间鲁棒 |
| **Radar-Camera Learned Metric Scale**(arXiv:2401.04325, 2024) | 雷达 + RGB | 单图 + 稀疏噪声雷达点 → 稠密米制深度 |
| **LiRCDepth**(arXiv:2412.16380, 2024) | 雷达 + RGB | 轻量,知识蒸馏 + 不确定性引导 |
| **Thermal Multi-spectral Consistency**(arXiv:2103.00760, 2021) | 纯热视频 | **多光谱一致性损失**训深度+位姿,低光/零光工作,无雷达 |

### 关键工程价值
雷达直接提供米制尺度 → **天然解 E2 的"相对→米制锚定"难题**(雷达点就是稀疏 GT 锚)。热成像 → 直接绕开 RGB 的暗光失效(对应 `docs/ideas/可扩展研究方向.md` Idea 1 的"成像假设失效"另一形态)。

> **本调研判断**:多模态融合是攻"挑战(1)长距离/恶劣条件"最务实的工程路线——**用硬件互补硬碰 RGB 的物理边界**,而非纯靠算法补救。它与 B6(事件相机)、B28(散焦)共同构成"非常规传感器深度"这一大类。**与软件路线(Idea 1 HDR、B5 恶劣天气)的关系**:互补而非替代——多模态需额外硬件(成本/标定),软件路线零硬件改动但受限于 RGB 信息上限。选题时应权衡:**目标场景能否加传感器?** 能则融合最稳,不能则物理驱动软件路线。RIDERS 的"雷达+热成像"组合尤其适合夜间野外机器人——这恰是原综述\"挑战(1)\"点名的野外机器人场景。

## G1. 单目深度的时代轨迹(三幕)

| 时代 | 时间 | 范式 | 深度的角色 | 本文档对应 |
|------|------|------|-----------|-----------|
| **专用任务时代** | 2014–2022 | 单图→深度的回归/分类,单数据集刷榜 | 独立的稠密预测任务 | 原综述 + A0–A8 |
| **基础模型时代** | 2019–2024 | 混合数据 + 海量无标注 + 零样本泛化 | 通用、即插即用的深度先验 | 合订本第 9 章 + B1/B18/B19 |
| **几何/世界模型时代** | 2024–2026 | 前馈/可微/生成式几何基础模型 | 4D 几何的一个导出量 / 生成约束 | B8/B12/B22/B24 |

**一句话**:单目深度正从"独立终产物"演化为"更大几何系统里的可靠先验组件"。

## G2. 五条贯穿性主线

1. **回归 ↔ 分类之争**:DORN(序数)→ AdaBins(自适应分箱)→ Binsformer(DETR query)→ 概率深度(B23)。分箱本质是离散化的概率分布,天然附带不确定性。
2. **几何结构保持**:BTS(局部平面)→ NeWCRFs(窗口 CRF=注意力)→ 球面 CRF(360°)→ 虚拟法向损失。几何感知算子是平面/全景/多视图的统一主题。
3. **数据稀缺的破解**:监督 GT 昂贵 → 自监督(光度一致性)→ 半监督 → 合成数据 → 师生伪标注(Depth Anything)→ 扩散生成(Atlantis/透明)。**合成/生成数据是当代统一答案**。
4. **相对 → 米制的尺度回归**:仿射不变损失(MiDaS)放弃尺度换泛化 → ZoeDepth/UniDepth/Metric3D 各法补尺度 → 雷达/SfM 锚定。尺度歧义是 MDE 的病态核心。
5. **失败模式集中在物理假设被违反处**:透明/反光(折射)、过曝/暗光(动态范围)、雨雾水下(散射)、动态物体(静态假设)。**回到成像物理是稳健出路**(物理驱动主线)。

## G3. 当代方法的元配方

绝大多数 2024–2026 SOTA 可归纳为同一模板:

```
强单图基础模型(DPT/DINOv2 骨干 + 混合数据 + 伪标注)
        │
        ├── 低成本适配到新维度  → 时序(视频深度)/ 米制(分箱头)/ 域(LoRA)
        ├── 作为先验注入几何系统 → SLAM(BA)/ NeRF/3DGS(正则)/ 立体(零样本)
        └── 被吸收进统一几何模型 → 点图(DUSt3R)/ 4D(MonST3R)/ 世界模型
```

**关键洞察**:很少有人再从零训深度网络;主流是"**冻结/复用强基础模型 + 轻量适配或注入**"。

## G4. 对选题的总结性建议(收束 `docs/ideas/可扩展研究方向.md`)

在基础模型/几何模型主导下,纯单图→深度增量价值持续走低。三条不与基础模型正面竞争、而补其盲区的方向:
- **(a) 攻物理边界**:HDR / 透明反光 / 恶劣天气 / 水下(物理驱动 + 内在分解 + 多模态)→ Idea 1;
- **(b) 给几何/世界模型供可靠不确定性与标定**:证据回归 + 单步深度的标定 → Idea 2;
- **(c) 重做被对齐掩盖的评测**:失败模式切片 + 免对齐 + 任务导向 → Idea 5 / B10。

> 三者共性:**把单目深度当作"更大系统的可靠先验"来改进,而非孤立刷指标**。这是本追踪文档对所有 MDE 选题的最终判断。

## B33. 单图 → 3D —— 深度驱动的 3D 照片与场景生成

> 接 B11(3DGS)、B24(世界模型)。这是单目深度最直观的消费级下游:把一张照片变成可漫游的 3D。

### 经典范式:分层深度修复(Layered Depth Inpainting)
原始"3D Photography"(Shih 等,2020)流程:单目深度 → 把图层化为分层深度表示 → **修复(inpaint)被遮挡区**(视角移动后露出的背景)→ 渲染新视角。核心难点:**视差揭示的遮挡区在原图不可见,必须生成**。

### 2024–2025 演进:扩散修复 + 显式几何
| 方法 | 思路 |
|------|------|
| **VistaDream**(arXiv:2410.16892) | 现代继承者:单视图建全局粗 3D 骨架 → **迭代扩散式 RGB-D 修复**填未观测区 → 多视图一致新视角。把老式修复网络换成扩散 |
| **A Recipe for 3D Worlds**(arXiv:2503.16611) | 显式深度抬升:生成全景 → 米制深度估计抬升到 3D → 条件修复填遮挡 |
| **Direct & Explicit 3D Gen**(arXiv:2411.10947) | 把生成深度像素**反投影到 3D**,经高斯泼溅渲染或抽网格 |
| **Flash3D**(arXiv:2406.04343) | 前馈、可泛化,快速单图重建+新视角(免逐场景优化) |

### 关键依赖
这些方法**深度依赖单目深度质量**——深度错则 3D 结构歪、新视角穿帮。米制深度(B1)与高分辨率深度(B25)直接决定 3D 照片质量。

> **本调研判断**:单图→3D 是单目深度**最大众化**的下游,也是个严苛的质量放大镜——深度的任何边界错误/尺度歪曲在新视角下立刻可见(B11 同理)。2024 的转向(修复网络→扩散)与 B3(Marigold)、B24(生成式世界模型)合流:**生成先验填补深度无法提供的"不可见区"信息**。深度给可见区几何,扩散补不可见区——两者互补构成完整 3D。**对评测的启示**:3D 照片质量(用户能否无穿帮漫游)是比 AbsRel 更贴近终端价值的深度评测代理(呼应 B10/B26 的任务导向评测主张)。

## B34. 状态空间模型(Mamba)主干 —— 线性复杂度的全局上下文

> 接 A6(DPT/ViT)、E4(效率)。Mamba 是 2024 挑战 Transformer 的新主干,核心卖点是**线性复杂度的全局感受野**。

### 为什么 Mamba 适合深度
深度估计受益于**全局上下文**(一个像素的深度取决于整个场景),这历史上把领域推向 ViT。但 **Transformer 随图像分辨率二次增长**。Mamba(arXiv:2312.00752,选择性状态空间)保持全局感受野却把复杂度降到**线性**,对高分辨率与受限硬件更友好。

### 视觉主干基础
- **VMamba**(arXiv:2401.10166):用堆叠 VSS 块,核心是 **2D 选择性扫描(SS2D)**——四条扫描路线弥合 Mamba 的 1D 顺序扫描与图像 2D 空间结构的鸿沟,线性复杂度下得全局感受野。
- **EfficientViM**(arXiv:2411.15241):隐状态混合的状态空间对偶(HSM-SSD),低算力捕获全局依赖。

### 深度专用 Mamba
| 方法 | 定位 |
|------|------|
| **MambaDepth**(arXiv:2406.04532) | 纯 Mamba 编码器-解码器 + 跳连,**自监督**,用长程依赖恢复精细结构 |
| **LMDepth**(arXiv:2505.00980) | Mamba 块线性解码深度,**轻量**,面向真实/边缘部署(亦见 E3) |

> **本调研判断**:Mamba 主干是对 A6(DPT 的 ViT 化)的效率回应——**在保留全局上下文的前提下打破 Transformer 的二次瓶颈**,直击原综述"挑战(3):高分辨率内存"。它与 B25(分块高分辨率)是两条互补路线:Mamba 从**架构**降复杂度,PatchFusion 从**推理策略**绕显存。**诚实判断**:Mamba 在深度上的精度优势相对 ViT 尚不决定性(主要赢在效率/分辨率扩展),其价值更多在**边缘部署**(E3)与**高分辨率**场景,而非刷 KITTI 榜首。是否采用取决于部署约束而非纯精度。

## A9. CAM-Convs [43] —— 相机感知卷积(补全原综述 [43] 机制)

> 接合订本 [43](原综述列在"深度学习基础/架构")。本节补其机制,它是 B1(米制深度免内参/条件化)的思想前驱。

**它解决的问题**:在一台相机的图上训练的深度网络**无法泛化到另一台相机**。改变焦距、主点、传感器尺寸会破坏标准深度网络——模型过拟合到训练相机的内参。这阻碍了混合不同相机的数据集,也限制真实部署。

**核心机制**:深度估计在几何上与相机内参绑定,但普通卷积无法访问这些信息。CAM-Convs 把**相机标定注入网络**——在卷积前,把由内参导出的**逐像素坐标图与视场图**拼接到输入特征上。具体加入:
- **中心化坐标图 (cx, cy)**:相对主点的像素坐标,让网络知道光心位置;
- **视场(FoV)图**:由焦距算出的逐像素角度信息,编码每像素张多大场景角;
- **归一化坐标图**:跨传感器归一化的空间位置。

这些额外通道由内参一次性算出、喂入卷积;因在**多尺度**应用,相机感知传遍编码器-解码器各分辨率。

**为何泛化**:把内参作为**显式输入**而非让网络隐式记忆,同一组训练权重在测试时即可适配新焦距/主点/传感器尺寸——只需喂入新相机对应的坐标与 FoV 图。从而能混合多相机数据训练、零样本泛化到未见相机。概念上类似 CoordConv(给卷积加坐标通道),但**专为深度几何特化**(图编码真实标定而非通用像素位置)。

> **承上启下**:CAM-Convs(2019)是 B1 中 UniDepth"自提示相机模块"、Metric3D v2"规范相机空间"的**思想前驱**——都在解同一个问题:**如何让深度网络感知相机内参以跨相机泛化/恢复米制**。原综述把它列在"基础架构"却未点明这层意义。它也是 G2 主线 4(相对→米制)的早期一环:显式建模内参是解尺度歧义的根本手段之一。

## B35. 深度鲁棒性 / 损坏基准 —— 标准榜单之外的真实考验

> 接 B10(评测争议)、B4/B5(失败模式)。这一节追踪"模型在损坏/分布外输入下还稳不稳"的系统性评测。

### 核心动机
标准基准(KITTI/NYU 干净图)上的高分**不等于真实鲁棒**。真实部署遇到噪声、模糊、天气、压缩失真。损坏基准系统性地施加扰动来暴露脆弱性。

### 代表基准
| 基准 | 贡献 |
|------|------|
| **Robust MDE: New Baseline & Benchmark**(IJCV) | 对 6 数据集(iBims/NYUv2/KITTI/ETH3D/DIODE/TUM)施 **15 种常见损坏**,室内外通用,最常引用 |
| **Robust OOD Depth under Corruptions**(arXiv:2310.15171) | 跨室内外**基准 42 个深度模型**;关键发现:无专门鲁棒训练则**显著退化** |
| **Procedural Scene Perturbations**(arXiv:2507.00981, 2025) | 主张标准基准涨点**不反映真实鲁棒**,用程序化场景扰动压测大模型 |
| **RobustSpring**(arXiv:2505.09368, 2025) | 对光流/场景流/立体施 **20 种损坏**,时间/立体/深度**一致**(非逐帧独立噪声) |
| **EndoDepth**(arXiv:2409.16063 等) | 内窥镜域鲁棒性基准(呼应 B20) |

### 对抗鲁棒性(另一威胁模型)
- **Physical 3D Adversarial Attacks**(arXiv:2403.17301, 2024):超越贴片攻击,用**物理 3D 对抗物体**攻击自驾 MDE;
- **Stabilized Adversarial Training**(arXiv:2411.02149, 2024):用对抗训练**防御性**提升跨域泛化。

### 三个评测轴
1. **损坏类型**:合成常见损坏 vs 程序化/物理扰动;
2. **威胁模型**:非对抗(损坏/OOD)vs 对抗(贴片/物理物体);
3. **域**:通用场景 vs 内窥镜/自驾专门域。

> **本调研判断**:鲁棒性基准是 B10(评测争议)的实践延伸——它把"平均 AbsRel 不够"具体化为"换成损坏输入就崩"。arXiv:2507.00981(2025)的核心论点尤其呼应本文档反复主张:**标准榜单涨点 ≠ 真实可用**。这对 `docs/ideas/可扩展研究方向.md` Idea 5(评测诊断基准)是强力背书——一个理想的 MDE 基准应同时覆盖**失败模式切片(B4)+ 损坏鲁棒(本节)+ 任务导向(B26)+ 免对齐(B10)**四个维度。当前没有任何单一基准做到这四点,这本身就是个清晰的选题缺口。

## B36. 视觉基础模型为何隐含 3D 几何 —— 探针分析

> 接 B18(Depth Anything 用 DINOv2 特征对齐)、B12(VGGT)。一个根本问题:DINOv2 等只在 2D 图上训练的模型,为何能给深度估计提供如此强的先验?

### 核心问题
这些模型在 **2D 数据**上训练,任何 3D 理解都是**涌现(emergent)**而非显式监督。它们到底隐含了多少几何?

### 关键探针研究
| 研究 | 发现 |
|------|------|
| **Probing 3D Awareness of VFMs**(CVPR 2024,arXiv:2404.08636) | 探针测**冻结表示**是否编码 3D:① 单视图深度/法向(可见面 3D 结构)② 跨视图一致性。规范参考 |
| **Probing for Complex 3D Scene Understanding**(arXiv:2409.03757) | **DINOv2 在几何/3D 任务上综合最强**;视频模型擅物体级,扩散模型助几何,语言预训练模型在稠密几何上落后 |
| **General Protocol to Probe LVMs**(arXiv:2310.06836) | **Stable Diffusion 与 DINOv2** 特征都擅判别式学习场景几何、支撑关系、阴影、深度 |
| **3D-Aware Fine-Tuning**(arXiv:2407.20229) | **反面警示**:纯 2D 训练的基础模型 3D 理解**有限**,在 3D 感知数据上微调可测量地改善——勿过度解读"涌现深度" |

### 为什么 DINOv2 这么强
DINOv2(自监督、无标签学鲁棒特征)在判别式几何任务上反复夺冠。可能原因:大规模自监督迫使模型学到**视角/光照不变的物体表面表示**,这与深度所需的几何结构高度相关。这就是为什么 Depth Anything(B18)、Surgical-DINO(B20)都选 DINOv2 作骨干并冻结它。

> **本调研判断(对全文的理论补全)**:本节回答了贯穿全文的一个隐含问题——**为什么"冻结基础模型 + 轻量适配"(G3 元配方)能成立**?因为基础模型的特征里**已经隐含了大量几何/深度信息**,适配头只需把它"读出来"而非"学出来"。但 arXiv:2407.20229 的反面警示也重要:涌现几何**有上限**,纯 2D 先验不足以支撑精确米制/复杂 3D——这正是为什么仍需 B1(米制专门头)、B8/B12(显式几何模型)。**对选题**:探针视角提供了一个少有人走的方向——与其训新深度模型,不如**研究/利用基础模型里已有的几何表示**(如更好的特征读出、或诊断它们在 B4 失败模式上为何失效)。这是个偏分析、低算力、可解释性导向的切口,适合理论倾向的研究。

## B37. 前馈 3D 重建模型的几何可解释性 —— 推断还是记忆?

> 接 B8(DUSt3R)、B12(VGGT)、B36。当 DUSt3R/VGGT 直接前馈出几何,一个尖锐问题随之而来:它们是在**做几何推断**,还是在**记忆学到的先验**?

### 核心拷问
**On Geometric Understanding and Learned Priors in Feed-forward 3D Reconstruction Models**(arXiv:2512.11508,⚠️ID 待核实)分析 DUSt3R、VGGT、Depth Anything 3,追问:输出中多少反映**真正的几何推断**,多少只是**学到的数据先验**?

这是个深刻的可解释性问题:
- 若模型主要靠**先验**(记忆"卧室长这样、深度大概如此"),则在分布外/反常几何场景会失败(呼应 B35 鲁棒性、B4 失败模式);
- 若模型真在**推断几何**(从视差/透视/明暗解算),则应在新几何上泛化。

### VLM 的几何局限(平行证据)
- **Probing Depth and Height of VLMs**(arXiv:2408.11748):VLM 能处理基本形状/尺寸,但**深度与高度推理欠佳**;
- **Abstract 3D Perception**(arXiv:2511.10946,⚠️ID 待核实):VLM 在空间认知/物理理解上有同样缺口。

> **本调研判断**:这条线是 B36 的自然延伸,也是对 G1"几何模型时代"的**批判性审视**——我们把深度交给了 DUSt3R/VGGT 这类前馈模型,但**它们真懂几何吗?** 若大量依赖先验,则"单目深度被吸收进几何基础模型"(本文档核心论点)就有隐忧:看似统一强大,实则可能在罕见几何/损坏输入上脆弱。**这给 `docs/ideas/可扩展研究方向.md` 的选题一个深层动机**:Idea 5(诊断基准)若能区分"几何推断 vs 先验记忆"(例如构造先验失效但几何可解的测试集),将是极有价值的可解释性贡献。**诚实标注**:本节多篇 arXiv ID(2512/2511)疑似未来日期,结论方向可信但具体出处须核实(见 F 部分)。

## B38. 深度估计的系统性偏见 —— 精度高 ≠ 行为对

> 接 B10(评测争议)、B35(鲁棒性)。本节原拟查"肤色公平性",但检索显示该角度在深度领域基本空白;却意外带出更有价值的"人类式系统性偏见"主线。

### 诚实的检索发现
- **肤色/人口学公平性**:在 MDE 领域**基本无专门文献**。深度的公平性研究目前都是"方法论公平"(模型间比较是否公平,见 B10),而非人脸识别那种"受保护属性公平"。**这是一个真空白**(但效用存疑:深度是几何量,肤色偏见主要在 ToF/结构光**传感器**层面,而非单目算法层面——值得查传感器侧文献)。
- 意外收获:**人类式系统性偏见**这条线有扎实研究。

### 人类式系统性深度偏见
**Human-like monocular depth biases in DNNs**(PLOS Comput Biol)记录:人类与深度网络都有一致的系统性深度误差——
- **深度压缩**:远距离被低估;
- **垂直偏置**:下视场的物体被感知为更近;
- **凸性偏好**。

**Accuracy Does Not Guarantee Human-Likeness**(arXiv:2512.08163,⚠️ID 待核实)更尖锐:**基准精度更高 ≠ 误差模式更接近人类感知**。这对"系统性误差"很关键——**聚合精度指标会掩盖结构化、可预测的失败模式**。

### 程序化扰动暴露材质/场景偏见
**Procedural Scene Perturbations(PDE)**(arXiv:2507.00981)用程序化生成 3D 场景,**受控变化材质/纹理/几何/光照**来隔离模型在哪里崩——这是触及"材质/场景类型"偏见的正路。

> **本调研判断**:本节揭示两个层次的偏见。① **人类式偏见**(深度压缩/垂直偏置)说明深度网络继承了与人类视觉相似的结构性误差——这对 AR/机器人意味着\"系统性、可预测\"的偏差,可建模校正。② **\"精度≠行为对\"**(2512.08163)与 B10、B35、B37 形成合力,共同构成本文档的一个核心方法论主张:**平均 AbsRel 是个危险的单一指标,它掩盖了系统性偏见、损坏脆弱性、与先验依赖**。**对选题**:研究并显式建模/校正这些系统性偏见(如深度压缩的距离相关校正),是个少有人做、有明确物理意义、且能直接提升下游(3D 重建尺度准确性)的方向。**肤色公平性**虽在算法层空白,但若聚焦传感器+算法联合的 RGB-D 公平性,可能是个有社会价值的真空白(待核实传感器侧文献)。

## B39. 深度估计的能耗 / 绿色 AI —— 一个未被系统量化的缺口

> 接 E1/E4(模型规模/效率)。本节诚实标注:**深度专用的碳足迹/能耗量化几乎是空白**,只有通用 Green AI 文献可借。

### 诚实的检索发现
检索\"深度估计能耗\"**未返回任何专门文献**。所有 Green AI 工作都围绕 LLM 训练/推理(如 ML-EcoLyzer arXiv:2511.06694、OpenCarbonEval arXiv:2405.12843、How Hungry is AI arXiv:2505.09598)。**深度估计的能耗从未被系统基准化**——这本身是个值得指出的领域缺口。

### 可从通用 Green AI 借鉴的原则
| 原则 | 对 MDE 的含义 |
|------|--------------|
| **推理成本高度依赖模型选择** | DA-V2 ViT-S(25M)vs ViT-G(1.3B)的推理能耗差数十倍(对应 E1) |
| **推理而非仅训练是主要成本** | 深度模型常**部署后长期高频推理**(机器人/AR),累积推理能耗远超一次训练 |
| **算法/模型选择常以极小精度损失换大能效** | 蒸馏(B18)、Mamba 主干(B34)、单步扩散(B3.2)都是能效杠杆 |
| **标准化测量工具成熟中** | ML-EcoLyzer 等可跨硬件测碳/能/水,可移用于深度模型 |

### 为何 MDE 的能耗值得单独量化
- 深度模型常在**边缘/电池设备**(无人机、AR 眼镜、手机)上**实时连续**运行——能耗直接决定续航与可行性;
- 扩散式深度(Marigold,B3)的多步采样能耗远高于前馈(Depth Anything),但**这个权衡从未被碳足迹量化**;
- 视频深度(B2)逐帧推理的累积能耗是部署关键,却无基准。

> **本调研判断**:这是本文档识别出的一个**清晰、具体、低门槛的选题缺口**——**为单目深度建立能耗/碳足迹基准**:横向测 Depth Anything 各档、Marigold(扩散)、Mamba 版(B34)、单步扩散在固定硬件上的 Wh/帧 与精度的帕累托前沿。它复用现成工具(ML-EcoLyzer)、契合边缘部署(E3)的现实约束、且与 `docs/ideas/可扩展研究方向.md` Idea 5(评测基准)同源——评测不应只测精度,还应测**能效**。在边缘 AI 与可持续 AI 双重关切下,这类工作引用潜力与社会价值都高。**诚实标注**:这是\"缺口识别\",非\"已有方法综述\"——下游若做需自建测量管线。

## B40. 深度 × 具身智能 / 导航闭环 —— 深度作为策略的感知前端

> 接 B16(SLAM)、E2(米制锚定)、B23(不确定性)。这一节看深度如何嵌入机器人**感知-决策闭环**,是单目深度最终极的落地形态之一。

### 三种集成范式
| 范式 | 代表 | 思路 |
|------|------|------|
| **端到端 RL** | Complex Obstacle Avoidance via RL(arXiv:2209.00296)、Mamba 运动(arXiv:2508.11849) | RGB→伪激光测量→RL 策略;伪激光缩小观测空间使 RL 更快、迁移更好 |
| **师生蒸馏** | Teacher-Student MDE(arXiv:2603.01999,⚠️ID 待核实) | 特权教师蒸馏成纯靠单目深度(微调 DA-V2)的学生策略——**当代主流配方** |
| **深度作上下文** | Risk-Aware MPC(arXiv:2508.07387) | 不直接拿噪声深度做碰撞检测,而用深度作**学习型碰撞模型**的上下文,预测风险分布再 risk-aware 规划 |

### 三个贯穿模式
1. **Depth Anything V2 已成默认单目深度骨干**(常逐域微调);
2. **师生蒸馏是把鲁棒策略送上廉价单目硬件的主流配方**;
3. 领域分化为**端到端 RL** vs **深度作上下文信号**(给独立规划器/碰撞模型),后者对深度噪声更鲁棒。

> **本调研判断**:具身导航是检验深度"够不够好"的终极试金石——它不在乎 AbsRel,只在乎**机器人撞不撞**。Risk-Aware MPC(2508.07387)的设计哲学尤其深刻:**与其信任噪声深度的绝对值,不如把深度当上下文 + 显式建模风险分布**——这正是 B23(概率深度)、`docs/ideas/可扩展研究方向.md` Idea 2(标定不确定性)在机器人闭环里的终极动机。**深度的不确定性比深度本身更重要**:知道"哪里不确定"才能安全规划。这把本文档反复强调的"标定不确定性"主线落到了最有说服力的应用上。**与 G4 呼应**:具身导航是"深度作为更大系统可靠先验"的典范——深度不是终产物,而是策略的感知前端。

## B41. 深度合成数据引擎 —— 程序化 / 仿真生成 GT 的工具链

> 接 B18(Depth Anything 用合成数据)、C1(Hypersim/vKITTI)、B4/B21(透明/水下用合成补 GT)。本节系统梳理"GT 从哪来"——这是基础模型时代的隐形基础设施。

### 为何合成数据是基础设施
真实深度 GT 昂贵稀缺(传感器/标定)。合成数据**四大优势**(Kubric 总结):便宜、富标注、完全可控、规避隐私/版权。它是 Depth Anything V2"用合成图训教师"(B18)、Marigold"合成数据微调"(B3)的供给侧基础。

### 两大锚定引擎(不同哲学)
| 引擎 | 哲学 | 特点 |
|------|------|------|
| **Infinigen**(arXiv:2306.09310)+ Infinigen Indoors(arXiv:2406.11824) | **全程序化生成** | 每个资产(形状/纹理)由随机数学规则**从零生成**,非资产库;经渲染管线得**精确 GT 几何含深度**;可无限量渲染 |
| **Kubric**(arXiv:2203.03570,Google) | **可扩展多物体视频 + 稠密标注** | 富自动标注(实例分割/深度/光流);通常**组合现有资产** |

### 域随机化与 sim-to-real
- **DREDS**(arXiv:2208.03792):**域随机化增强深度仿真**,用 PBR 模拟主动立体传感器,专攻镜面/透明物体(传感器失效处,呼应 B4);
- **Domain-Transferred Synthetic**(arXiv:2405.01113):3D 合成环境 + CycleGAN 域迁移改善单目深度。

### 2024–2025 新方向
- **BRIDGE**(arXiv:2509.25077):另辟蹊径——**RL 优化的深度→图像**框架,生成 2000 万+ RGB 图各配几何精确深度,而非渲染完整 3D 场景;
- **程序化零样本立体**(arXiv:2504.16930):靠程序化生成数据集出高质量视差 GT。

> **本调研判断**:合成数据引擎是基础模型时代**被低估的关键基础设施**——Depth Anything 的成功一半归功于数据引擎(B18),而数据引擎的上游正是 Infinigen/Kubric 这类生成器。**G2 主线 3(数据稀缺破解)的终点就是这里**:从手工标注→自监督→伪标注→**程序化无限生成**。一个值得注意的张力:合成数据的**域差(sim-to-real gap)**始终存在,DREDS 的域随机化、CycleGAN 迁移都在补;Depth Anything V2 则用"大规模伪标注真实图"弥合(B18)。**对选题**:程序化生成 + 物理精确渲染,恰是攻 B4/B21(透明/水下,真实 GT 无法获取)的天然出路——为这些失败模式定制 Infinigen 式生成器(含正确的折射/散射物理),可能比改进网络更治本。这把 `docs/ideas/可扩展研究方向.md` Idea 1(物理驱动)与数据引擎结合成了一条具体路径。

## B42. Depth Anything 3(2025.11)—— 单目深度并入几何基础模型的最新铁证

> 接 B8/B12/B22(点图与几何基础模型)、合订本第 9 章。DA3 是本文档核心论点("单目深度被吸收进几何基础模型")**最新、最直接的实证**,发布于 2026 年视角下的最近期(2025.11)。

### 核心转变(arXiv:2511.10647,"Recovering the Visual Space from Any Views")
DA3 从单图深度**跨越为统一几何基础模型**,跨**任意视图配置**工作:单图、多视图、或视频,有/无已知相机位姿皆可。这是 DA 系列从"单目深度模型"到"几何基础模型"的范式跃迁。

### 关键技术点
- **朴素 Transformer 架构**:不引入专门几何模块,延续 DA 系\"简单可扩展优于技术复杂\"的传统(呼应 A6 DPT、B18);
- **统一深度-射线表示**:旗舰 DA3-GIANT 用单一表示同时处理深度与相机射线,使一个模型横跨深度/位姿/3D 任务;
- **四大能力均达 SOTA**:相机位姿估计、任意视图几何重建、视觉渲染、**单目深度估计**(系列起点任务)。

### 模型变体与工程
- **DA3-GIANT**:多视图深度+位姿+3D 高斯的旗舰;
- **DA3NESTED-GIANT-LARGE**:any-view Giant + metric Large,**米制几何重建**,推荐全能版;
- **DA3-Streaming**:滑窗流式推理处理超长视频,**<12GB 显存**(呼应 B2 视频深度、E1 部署)。

> **本调研判断(对核心论点的最终确证)**:DA3 把本文档反复论证的趋势**钉成事实**——DA 系列自己的演化轨迹就是全领域的缩影:**DA1/DA2(单图深度基础模型)→ DA3(任意视图几何基础模型)**。单目深度在 DA3 里明确降格为\"四大能力之一\",与位姿/渲染/多视图并列。这与 VGGT(B12)、MonST3R(B22)殊途同归——**几何基础模型已成定局,单目深度是其一个输出头**。一个 Medium 分析的点评尤其精辟:DA3 可能\"是个伪装成几何基础模型的强深度教师配方\"——即其真正价值在于**生成高质量深度监督**(呼应 B18 数据引擎)。**对选题的终极校准**:任何 2026 年的 MDE 选题都必须以 DA3/VGGT 这类几何基础模型为既定背景,在其盲区(物理边界 Idea 1、不确定性 Idea 2、评测 Idea 5)做文章,而非重做\"更好的单图→深度\"。

## B43. 深度 × VLM 空间推理 —— 语义强、度量弱的鸿沟

> 接 B14(可提示深度)、B36(VFM 几何探针)。视觉语言模型(VLM)能描述场景却**算不准距离**,这一节追踪用深度补 VLM 空间推理短板的工作。

### 核心张力:语义强、度量弱
**Metric Depth From VLMs**(arXiv:2509.25413)的尖锐论断:**即便 GPT-5 仍难从 2D 理解 3D**,而专门的纯视觉深度模型达\"超人\"表现。这是 2025 年的核心矛盾——通用 VLM 语义强但米制弱。

### 奠基:SpatialVLM(arXiv:2401.12168)
在**大规模合成空间 QA 数据**上训练 VLM,使其能做 3D 空间推理、直接从 2D 图生成米制距离估计。明确把现有 VLM 框为在**定量空间关系上有盲点**——这是后续 2025 工作的共同出发点。

### 2025 两条修补策略
| 策略 | 代表 | 思路 |
|------|------|------|
| **显式注入深度** | SD-VLM(arXiv:2509.17664)、SSR(arXiv:2505.12448) | 把深度编码进 VLM;SSR 用\"理由引导\"改善 VLM **如何用**深度线索而非仅有访问权 |
| **先重建 3D 再推理** | Ego3D-VLM(arXiv:2509.06266) | 从估计的全局 3D 坐标建认知地图;绝对距离估计涨 ~56%,远超类别任务的 ~12% |

### 关键发现
**显式 3D 坐标 grounding 对米制任务帮助远大于类别任务**(Ego3D-VLM 的 56% vs 12%)——说明 VLM 的几何短板是\"硬\"的,需真实几何信号注入,语义微调补不了。

> **本调研判断**:这条线揭示了单目深度的一个**新兴战略价值**——它是 VLM 通向\"空间智能\"的关键拼图。VLM 语义强、度量弱;单目深度恰好相反(几何强、无语义)。两者融合(SD-VLM 注入深度、Ego3D 先重建)是 2025 明确趋势。**对选题**:这给 B14 的"文本条件深度"空白(F3-5)一个反向答案——比起"用文本指导深度",**"用深度增强 VLM 空间推理"是更有需求、更有据可循的方向**。它把单目深度从"几何任务"接入了"多模态智能"主线,呼应 B24(世界模型)、G1(深度成为更大系统的组件)。**与 B36 呼应**:既然纯 VLM 几何弱(本节)、而 DINOv2 式 VFM 几何强(B36),融合两者的特征可能是 VLM 空间推理的捷径——这是个有据可查的具体技术路径。

## B44. 联邦 / 隐私保护深度学习 —— 不共享原始图像的分布式训练

> 接 B40(具身导航)、B15(在线适应)。当深度模型部署在车队/手机等隐私敏感场景,如何在不上传原始相机数据的前提下集体训练?

### 核心动机
车载/室内相机数据含隐私(人脸、车牌、家居)。**联邦学习(FL)**让各设备本地训练、只上传梯度/权重,不共享原始图像。与**自监督深度**天然契合——后者本就不需 GT,各车用自己的视频流自监督训练。

### IBM 系列工作(2024 主线)
| 方法 | 贡献 |
|------|------|
| **Federated Self-Supervised Depth**(arXiv:2310.04837) | 奠基:FL + 自监督深度,车辆**不共享原始相机数据**即可训练;增强隐私、减网络消耗、抗连接中断 |
| **FedSCDepth**(VTC Fall 2024) | 自监督 FL 隐私保护训练,KITTI Eigen split 测试损失 <0.13(待核实) |
| **BOFedSCDepth**(FGCS/ITSC 2024) | 在联邦平均上加**贝叶斯优化**,改善效力与效率 |
| **Continual Federated Depth**(VTC Fall 2024) | 扩展到**持续学习**,处理模型随时间适应(接 B45) |

演进逻辑:联邦自监督 → 贝叶斯优化调聚合 → 持续/在线学习维度。

> **本调研判断**:联邦深度学习把"数据稀缺破解"(G2 主线 3)推向**隐私维度**——不是没有数据,而是数据**不能集中**。这对自驾车队、AR 眼镜、家用机器人是真实约束。它与自监督深度(无需 GT)是**天作之合**:联邦只上传梯度、自监督只需原始视频,两者结合既保隐私又免标注。**与本文档安全意识呼应**:深度数据(尤其室内/车载)是隐私敏感的,联邦学习是负责任部署的技术路径。**对选题**:这是个相对小众但有明确社会价值的方向,且与边缘部署(E3)、持续学习(B45)天然组合。诚实标注:目前文献集中于单一团队(IBM),领域尚未饱和,但也意味着需求验证尚不充分——选题前应确认隐私约束在目标场景是否真的是瓶颈。

## B45. 持续学习 / 灾难性遗忘 —— 学新域时别忘旧域

> 接 B15(测试时适应)、B44(持续联邦)。深度模型部署后持续学新场景时,会**遗忘旧场景**(灾难性遗忘)。本节系统梳理缓解之道。

### 核心矛盾:稳定性-可塑性权衡
**Continual Learning of Unsupervised Monocular Depth from Videos**(arXiv:2311.02393)立起问题:朴素持续训练会让旧域性能随新域学习而退化。**稳定性(记旧)vs 可塑性(学新)** 的权衡是核心。

### 缓解策略谱
| 策略 | 代表 | 思路 |
|------|------|------|
| **提示学习(参数隔离)** | Domain Shared/Specific Prompt(OpenReview FNYMLAhOHW) | 提示分**域共享**(跨域共性)+ **域专属**(每域细节)分量,避免存旧数据又限制域间干扰。最贴合增量 MDE |
| **原型(prototype)** | ProtoDepth(arXiv:2503.12745) | 无监督持续**深度补全**,用原型缓解遗忘 |
| **终身多域** | Lifelong Multi-Domain Metric Depth(arXiv:2303.05050) | 跨多域**米制**深度(自驾/机器人),讨论传感器相关质量问题 |
| **联邦持续** | Federated Continual Depth(NSF PAR) | FL + 持续应对室内训练数据漂移 |

### 一个领域特性
深度文献以**域增量(domain-incremental)** 和终身学习为主,而非真正的**类增量(class-incremental)**——因为深度是**稠密回归**任务,没有离散类别。

> **本调研判断**:持续学习与 B15(测试时适应)是同一硬币两面——TTA 关注"快速适应当前",持续学习关注"适应时别遗忘"。两者共同的现代答案是**参数高效方法**(提示学习、LoRA、原型),只调少量参数既学新又不大幅扰动旧知识。这与 B20(LoRA 适配内窥镜)、`docs/ideas/可扩展研究方向.md` Idea 2(冻结骨干加轻量头)同源——**\"冻结大模型 + 轻量可塑组件\"是基础模型时代应对一切适应/持续/不确定性问题的统一工程范式**。**对选题**:在 Depth Anything 这类基础模型上做"持续学习的提示/适配器",是个有现实需求(部署后场景漂移)、低算力、且与主流范式契合的方向。它把 B15/B20/B44/B45 串成一条"基础模型的终身部署"主线。

## B46. 偏振光形状 / 深度(Shape from Polarization)—— 物理驱动的新模态

> 接 B28(散焦)、B30(法向)、B21(水下)。偏振是又一条物理可靠的深度/法向线索,与 RGB 互补。

### 物理基础
光从表面反射时偏振态(偏振度 DoP、偏振角 AoLP)与**表面法向**相关——偏振即几何线索。给定几张不同偏振片角度的图,可反推表面朝向,再积分得形状/深度。

### 两大固有挑战
1. **方位角歧义(azimuthal ambiguity)**:每个 AoLP 测量对应多个可能的表面朝向;
2. **标注数据稀缺**:偏振 GT 难采集。

### 代表方法
| 方法 | 贡献 |
|------|------|
| **Deep Shape from Polarization**(arXiv:1903.10210) | 奠基:首次把深度学习引入 SfP |
| **SfP in the Wild**(arXiv:2112.11377) | 从孤立物体推进到**完整场景**,直面数据稀缺 |
| **Deep Polarization Cues**(arXiv:2407.08149, 2024) | 单次拍摄 4 张偏振图,**扩展到次表面散射(SSS)**,处理半透明 |
| **Neural Inverse Rendering SfP**(arXiv:2407.09294, 2024) | **自监督**逆渲染联合估法向+深度,避开 GT 瓶颈 |
| **Polarization + RGB stereo**(arXiv:1903.12061) | 用粗但米制准的立体深度**解 SfP 的方位角歧义** |
| **Underwater Descattering SfP**(arXiv:2603.00908,⚠️ID 待核实) | 水下联合去散射+法向(呼应 B21) |

### 解歧义的两条路
- **融合 RGB/立体**:用另一模态的粗深度消歧(1903.12061);
- **物理先验/自监督逆渲染**:用成像物理约束(2407.09294)。

> **本调研判断**:SfP 与 DfD(B28)、事件相机(B6)、多模态融合(B32)同属"非常规传感器/线索深度"大类,共性是**用额外物理信号补 RGB 单目的病态性**。SfP 尤其擅长**无纹理/镜面**表面(那里 RGB 立体/单目都弱,因偏振对表面朝向直接敏感)——这恰好攻 B4(非朗伯)的失败模式。**对选题**:偏振 + RGB 融合深度,在透明/反光/无纹理场景可能比纯 RGB 基础模型更可靠,是 `docs/ideas/可扩展研究方向.md` Idea 1(物理驱动攻失败模式)的又一具体载体,且需特殊硬件(偏振相机)——属"能加传感器则融合最稳"(B32 判断)的范畴。

## B47. AR 遮挡渲染中的深度 —— 实时、边界、错误容忍

> 接 E3(下游集成)、B25(高分辨率)、B40(具身)。AR 是单目深度最严苛的实时消费场景:深度错一点,虚实遮挡就穿帮。

### 核心矛盾:深度误差直接毁掉遮挡幻觉
**Virtual Occlusions Through Implicit Depth**(arXiv:2305.07014)点破:**估计深度的微小误差就毁掉遮挡掩码、破坏 AR 幻觉**,尤其在**物体边界**附近的实时场景。它提出隐式深度法产更干净的遮挡边界——**关键洞察:AR 要的不是精确深度值,而是精确的"谁在前"的二元遮挡关系**。

### 产业方案
| 方案 | 特点 |
|------|------|
| **ARCore Depth API** | 用 RGB 每帧生成深度图,让虚拟物正确前后遮挡;人作为环境深度的一部分处理 |
| **ARKit People Occlusion** | 专门的**分割式**人物遮挡(与 ARCore 的通用深度遮挡不同路) |

### 研究方向
- **Dynamic Occlusion Handling**:用原始深度做粗前后景分割,再用颜色细化边缘混合系数(平滑边界);
- **DepthLab**(augmentedperception.github.io/depthlab):用实时深度做 3D 交互 UX。

> **本调研判断**:AR 遮挡揭示了深度评测的一个**被忽视维度**——**边界质量与遮挡关系正确性,比平均深度精度更重要**。这与 B26(伪 LiDAR 的"表示>精度")、B10(对齐掩盖误差)、B38(精度≠行为对)形成强合力:**平均 AbsRel 是个对 AR 几乎无意义的指标**;真正要测的是"遮挡边界对不对、虚实穿不穿帮"。Virtual Occlusions 的"隐式深度专攻边界"思路尤其点睛——**与其追求全图深度准,不如专攻遮挡关键的边界区**。这给 `docs/ideas/可扩展研究方向.md` Idea 5(失败模式/任务导向评测)一个极具体的子方向:**AR 遮挡质量基准**(边界 IoU、遮挡翻转率),比 KITTI AbsRel 更贴近消费级 AR 的真实价值。ARKit 用分割而非深度做人物遮挡也很有启发——有时**绕过深度、直接解下游问题**更稳。

## B48. 鱼眼 / 广角 / 全向相机的畸变深度

> 接 A9(CAM-Convs 相机感知)、B13(360°全景)。鱼眼/广角相机(自驾环视、AR/VR、机器人)FoV 大但**畸变严重**,标准针孔深度模型直接失效。

### 核心问题
鱼眼投影非线性,边缘严重拉伸。自监督深度的**投影-重投影几何**(A1 Monodepth2 的核心)若用针孔模型会算错,必须把**鱼眼相机模型嵌进 warp**。

### 三种处理相机模型的策略
| 策略 | 代表 | 思路 |
|------|------|------|
| **嵌入固定/已知相机模型** | Real Scale SfM Fisheye(arXiv:2409.15054, 2024) | 把鱼眼模型直接嵌进投影-重投影损失,**恢复米制**深度,提升精度与训练稳定 |
| **联合学习畸变模型** | Learnable Camera Distortion(arXiv:2110.10415) | 可学习轴对称相机模型,同时吃鱼眼与针孔图,畸变参数**学出来**而非假设 |
| **绕开鱼眼训练数据** | Calibration Tokens(arXiv:2508.04928, 2025) | 把针孔数据集**重标定成鱼眼图**,用标定 token 适配基础模型,**无需鱼眼数据**、全自监督 |

早期:Scale-Aware Fisheye for 自驾(arXiv:1910.04076,>180° FoV 常引基线)、Ordinal Distillation for AR/VR HMD(arXiv:2205.02930)。

> **本调研判断**:鱼眼深度是 A9(CAM-Convs)"相机感知"思想的极端版——畸变越大,**显式建模相机几何**越关键。三种策略恰好对应通用主线:嵌入已知内参、联合学习内参、或适配基础模型(Calibration Tokens 与 B20 的 LoRA 适配同源——**把基础模型适配到新相机几何而非重训**)。2508.04928 的"重标定针孔数据成鱼眼"尤其巧妙,呼应 B41(合成/变换数据补稀缺)。**对选题**:鱼眼/全向是 AR/VR 与环视自驾的刚需,而基础模型(在针孔图上训)对鱼眼支持差——"高效适配基础模型到任意相机模型"是个有明确需求的工程缺口,且与 A9/B13/B20 一脉相承。

## B49. 深度模型量化 / 压缩 —— 把基础模型塞进边缘设备

> 接 E1/E3/E4(部署/效率)、B18(蒸馏)、B34(Mamba 轻量主干)。基础模型强但大,边缘设备(手机/无人机/AR 眼镜)装不下,压缩是落地最后一公里。

### 三大压缩技术
| 技术 | 代表 | 要点 |
|------|------|------|
| **知识蒸馏** | Cross-Architecture KD(arXiv:2404.16386, 2024) | 把**Transformer 深度知识蒸馏进 CNN 学生**,保精度降算力——直击边缘部署;Distillation Creates Stronger MDE(arXiv:2502.19204)反向证明蒸馏还能**提升**学生精度 |
| **量化** | QuartDepth(arXiv:2503.16709) | 后训练量化把权重/激活压到 **4-bit**,面向 ASIC 硬件加速,比 INT8 更激进 |
| **剪枝** | Joint Filter Pruning(arXiv:1905.05212) | 移除最不重要的通道/特征,得轻量模型 |

### 组合与排序
**Ordered Compression Pipeline**(arXiv:2604.04988,⚠️ID 待核实)的经验:**INT8 QAT 带来主要运行时增益**,剪枝作为容量预压缩使后续低精度优化更稳,KD 恢复精度。即推荐顺序:剪枝 → 量化(QAT)→ 蒸馏恢复。

### 与高效架构的关系
压缩大模型 vs 从头设计高效模型(EfficientDepth arXiv:2509.22527、Mamba B34)是**两条互补路线**:前者复用基础模型能力,后者原生高效。

> **本调研判断**:量化/压缩是 G3 元配方("冻结基础模型 + 适配")在**部署侧**的对应物——基础模型在云端训好,压缩让它跑在边缘。**诚实标注的工程缺口**:检索显示**无专门针对 TensorRT 的深度模型 INT8 部署论文**(F 部分可补),实践需自己组合"蒸馏 CNN 学生 + TensorRT INT8 PTQ/QAT"。这与 B39(能耗缺口)、E1(无硬 FPS 数据)共同指向一个事实:**MDE 的边缘部署工程虽是刚需,却缺系统化的公开基准与最佳实践**——这本身是个有产业价值的填空方向(基准 + 工具,呼应 Idea 5)。压缩 + 量化 + 能耗(B39)三者合起来,就是一份完整的"MDE 边缘部署效率手册"的内容,目前学界还没人系统做。

## B50. 自监督尺度一致性 / 尺度漂移 —— SC-Depth 与几何一致性损失

> 接 A1(Monodepth2)、B16(SLAM)、B31(场景流)。这是自监督深度一个根本性痼疾的解法,补全自监督主线。

### 核心问题:尺度漂移(scale drift)
自监督单目深度网络在视频上训练,只学到**每帧独立的未知尺度因子**。因各帧尺度独立,预测深度与位姿沿序列**尺度漂移**——这让需要全局一致几何的任务(如重建长视频的相机轨迹)无法使用。

### SC-Depth(Bian 等)—— 奠基
| 论文 | 贡献 |
|------|------|
| **SC-SfMLearner**(arXiv:1908.10553,NeurIPS 2019) | 提出**几何一致性损失**:预测相邻两帧深度+相对位姿 → 用位姿把一帧深度 warp 到另一帧 → 惩罚 warp 深度与直接预测深度在对应像素的差异。逐对传播单一一致尺度 |
| **SC-Depth**(arXiv:2105.11610,IJCV 2021) | 期刊扩展版,首次证明无标签单目视频训练的网络能预测**长序列尺度一致**的轨迹 |

**副产品:自发现掩码**——几何不一致高的像素(运动物体/遮挡)在光度损失里被降权,缓解静态假设违反(呼应 A1 自动遮罩、B17 动态)。

### 后继
- **Robust SC-Depth for Dynamic Scenes**(TPAMI,arXiv:2211.03660):扩展到动态内容鲁棒;
- **Pose Constraints**(arXiv:2304.08916):尺度一致预测可**一次算出米制尺度因子复用**;
- **Scale-Consistent SLAM via Scene Coordinate Embeddings**:把尺度一致思想用进 SLAM 抗轨迹漂移。

> **本调研判断**:几何一致性损失是自监督深度的关键一环——它把 A1(Monodepth2 的逐帧光度)升级为**跨帧尺度一致**,是自监督深度能用于 SLAM(B16)/轨迹重建的前提。它与 E2(米制锚定)互补:SC-Depth 保证**内部尺度一致**(各帧同尺度),E2/锚定提供**绝对米制**(尺度对到真实米)。两者合起来才得到全局米制一致的深度。**对完整性**:本节补全了自监督主线的最后一块——Monodepth2(逐帧光度+遮挡,A1)→ 动态处理(B17)→ 场景流联合(B31)→ **尺度一致(本节)**,自监督深度的核心机制至此在本文档完整覆盖。

## B51. 遥感 / 航拍 / 卫星深度与高度估计

> 接 C1(数据集)、B1(米制)、E2(尺度锚定)。俯视视角的深度有自己的范式——常表述为"高度估计"而非"深度估计"。

### 深度 vs 高度的视角差异
地面相机估"到物体的距离(深度)";俯视相机估"地物的海拔/高度(height)"。后者是遥感的主流表述,但数学同构(都是从单图回归几何量)。

### 高度估计(卫星/航拍)
| 方法 | 贡献 |
|------|------|
| **HeightFormer**(arXiv:2310.07995,RS 16(2):295) | 最常引基线:多级特征交互 + **图像自适应分类-回归头**。因航拍高度动态范围大,纯回归难,故**逐图自适应分箱再细化**(直接呼应 A2 AdaBins/A7 Binsformer 的分箱思想!) |
| **Semi-supervised Height**(arXiv:2511.13552) | 标注+无标注混合训练,把单目高度作 LiDAR/多视立体的廉价替代 |
| **Sparse LiDAR-Guided Correction**(arXiv:2505.06905) | 注入稀疏 LiDAR 修正超高分辨率影像的结构误差(呼应 B9 补全) |

### UAV 米制深度(尺度恢复)
- **Low-overlap Drone Photogrammetry**(arXiv:2503.04513):用航空三角测量的**连接点(tie points)** 把相对深度锚到米制;
- **Leveraging Global DEMs**(arXiv:2409.05142):把全球数字高程模型(DEM)的稀疏高程投到相机视图作**米制锚**——无本地 GT 时用全球 DEM。

### 低空 UAV 自监督深度
Global-Local 多尺度增强网(RS 15(13):3275)、轻量方向感知法(Applied Sciences 16(11):5229),用帧间光度一致性,无需深度标签,后者面向机载实时。

> **本调研判断**:遥感高度估计是个**自成体系却与地面深度同构**的分支——HeightFormer 的"图像自适应分箱"与 A2/A7 的 AdaBins/Binsformer **本质同源**,印证分箱思想跨视角通用(亦见 B13 360°)。其尺度恢复手段(tie points、全球 DEM)是 E2"米制锚定"在航拍域的特化变体——**用领域特有的外部参照(DEM/三角测量)替代地面的 LiDAR/IMU**。**对选题**:遥感深度/高度是个有独立数据集、独立应用(测绘/灾害评估/城市建模)、且能直接复用基础模型(Depth Anything 微调)的方向。**一个值得注意的统一性**:无论地面/航拍/全景,核心难题都是同两个——**尺度歧义**(靠外部参照锚定)与**动态范围/分箱**(靠自适应离散化),这进一步印证 G2 的主线归纳。

## B52. 引导式深度超分(Guided Depth SR)—— RGB 引导低分辨率深度上采样

> 接 B9(深度补全)、B25(高分辨率深度)。与"单图估深度"不同,GDSR 输入是**低分辨率深度图 + 高分辨率 RGB**,用 RGB 结构引导深度上采样。这是 ToF/低成本深度传感器的刚需(它们分辨率低)。

### 核心问题与陷阱
LR 深度(ToF/结构光常见)缺细节,HR RGB 提供丰富边界。难点是**跨模态融合**:既要借 RGB 边界,又要避免**纹理复制伪影(texture-copy)**——把 RGB 纹理错误地刻进平坦深度区。

### 方法演进
| 方法 | 思路 |
|------|------|
| **Guided SR as Pixel-to-Pixel**(arXiv:1904.01501) | 影响力重述:把 GDSR 设为像素到像素变换 |
| **Deep Anisotropic Diffusion**(arXiv:2211.11592) | RGB 引导的学习式各向异性扩散 |
| **Spherical Space Decomposition**(arXiv:2303.08942) | 球面空间**分离共享 vs 模态专属特征**,抗纹理复制 |
| **IGAF**(arXiv:2501.01723, 2025) | 增量引导注意力融合,逐步组合 HR RGB 与 LR 深度特征 |
| **Cross-Modal State Space**(arXiv:2605.11934,⚠️ID 待核实) | Mamba 式跨模态局部扫描,避开昂贵注意力(呼应 B34) |

### 共同主线(2024+)
注意力/状态空间做跨模态融合、**分离共享与模态专属特征**(避免纹理复制)、降跨模态注意力成本。

> **本调研判断**:GDSR 与单目深度是**互补而非竞争**——GDSR 假设已有(低分辨率)深度传感器,单目深度从零估。它的核心教训("分离共享与模态专属特征以避免纹理复制")对**所有 RGB 引导的深度任务**(补全 B9、超分本节、多模态融合 B32)都通用:**RGB 提供边界但别被 RGB 纹理污染**。这也呼应 B30(深度×内在图像)——区分哪些 RGB 信号是几何相关(边界)、哪些是纹理无关(反照率)。**对选题**:在 Depth Anything 时代,GDSR 可重构为"用基础模型的强相对深度 + 稀疏/低分辨率真实深度做引导超分",把基础模型先验与传感器测量结合——这是 B9/B25/B52 共同指向的"基础模型 + 稀疏真实测量"配方在超分场景的实例。

## B53. 前馈单图 3D 高斯 —— 深度作为像素对齐的几何骨架

> 接 B11(3DGS×深度)、B33(单图→3D)、B34(Mamba)。这是 2024 单图 3D 重建的范式突破:**单次前向**出 3D 高斯,跳过逐场景优化。深度在其中是隐式的几何骨架。

### 核心:像素对齐的高斯预测
**Splatter Image**(CVPR 2024,arXiv:2312.13150)是奠基:用图到图网络处理输入视图,输出另一张图,**每个像素持有一个有色 3D 高斯的参数**。这种**像素对齐表示**使其超快(交互速率)。关键洞察:把 3D 重建变成 2D 图到图任务,而每像素的高斯位置本质由**深度**决定。

### 方法谱
| 方法 | 贡献 |
|------|------|
| **Splatter Image**(arXiv:2312.13150) | 像素对齐高斯,单图超快重建 |
| **Triplane Meets GS**(CVPR 2024,Zou 等) | 双 Transformer 网,快速可泛化单视图重建 |
| **Gamba**(arXiv:2403.18795) | GS + **Mamba** 状态空间做单视图重建(呼应 B34) |
| **Flash3D**(arXiv:2406.04343) | 预测**递增深度偏移的分层高斯**处理遮挡区,从物体扩到完整场景(呼应 B33 分层深度修复) |
| **L4GM**(arXiv:2406.10324) | 扩到 **4D**:从视频出逐帧高斯 + 时序上采样平滑 |
| **LGM**(Tang 等,ECCV 2024) | 大型多视图高斯模型,融多视图为 3D 高斯再转网格 |

### 深度的角色
这些方法里**深度是隐式的几何骨架**——像素对齐高斯的位置由(预测的)深度沿射线放置决定。Flash3D 的"分层深度偏移"尤其直接:它本质是在预测**多层深度**来覆盖遮挡区。

> **本调研判断(收束 3D 重建主线)**:前馈单图 3D 高斯把 B33(单图→3D)从"深度+修复的多步管线"压成"单次前向出高斯"。深度在此**从显式输出降为隐式中间表示**——像素对齐高斯 = 每像素的(深度 + 外观 + 协方差)。这与 B8/B12/B22(点图/VGGT)殊途同归:**深度被吸收进更丰富的逐像素 3D 表示**(高斯/点图),不再单独输出。Flash3D 的分层深度偏移又与 B33 的分层深度修复、B8 的多层深度(Seeing through Glass)呼应——**"单层深度不够,需多层覆盖遮挡"是个反复出现的洞察**。**对全文核心论点的又一确证**:单目深度正成为更丰富 3D 表示(高斯/点图/4D)的一个分量,而非终产物(G1/G4)。

## B54. 人体中心深度 —— 人脸 / 手 / 人体网格

> 接 B26(物体级深度)、B23(概率深度)。当场景主体是人,深度估计常与**人体网格恢复(HMR)**耦合——既要人相对相机的深度,又要人体表面几何。

### 任务谱
| 方向 | 代表 | 要点 |
|------|------|------|
| **全身网格(脸+手+体)** | Multi-HMR(arXiv:2402.14654) | 单次出多人全身 3D 网格,含手部与表情;PyMAF-X(arXiv:2207.06400)强调手脸对齐 |
| **深度引导/米制感知** | MetricHMSR(arXiv:2506.09919,⚠️ID待核实)、Depth-Aware Transformer(arXiv:2403.09063) | 把深度感知建进架构,恢复**米制**人体网格 + 周围场景 |
| **歧义建模** | Probabilistic HMR(arXiv:2411.16289) | 用 2D 姿态不确定性建**多个合理网格分布**,应对深度歧义/遮挡/截断 |

### 与通用深度的关系
人体深度的特殊性:① 人是**铰接非刚体**,深度随姿态剧变;② 有强**人体形状先验**(SMPL 等参数化模型),可约束深度;③ **米制尺度**对 AR/动捕关键(人高约 1.7m 是天然尺度锚)。

> **本调研判断**:人体中心深度是 B26(物体级深度)思想在"人"这个特殊类别上的深化——**有强先验(SMPL)的类别,深度可被参数化模型强约束**,这比通用单目深度更可解。它与 B23(概率深度)的结合(Probabilistic HMR 建网格分布)尤其自然——深度歧义在人体上表现为"多个合理姿态",用分布建模比点估计合理。**对选题**:人体高度(~1.7m)是个免费的米制尺度锚——这给 `docs/ideas/可扩展研究方向.md` Idea 3(几何尺度锚)一个具体且高频的实例:**用图中人的已知身高统计反推场景米制尺度**。这在含人场景(AR/监控/动捕)里是个实用且少被系统利用的锚。

## B55. 主动立体 / 结构光的学习式深度 —— 投射图案 + 神经解码

> 接 B27(立体基础模型)、B4(非朗伯)、B41(合成数据)。当被动 RGB 不够,**主动投射图案**(结构光/主动立体)提供额外几何信号,深度学习正在改造这条经典硬件路线。

### 物理基础
投影仪投出已知图案(点/条纹),图案在物体表面的形变编码了深度。传统靠手工解码图案;深度学习用**神经特征解码**图案形变,更鲁棒(尤其无纹理/反光表面——B4 的失败模式)。

### 2024–2025 方法
| 方法 | 思路 |
|------|------|
| **Neural Feature Decoding 结构光**(arXiv:2512.14028,⚠️ID待核实) | 从投射图案与捕获 IR 图提神经特征,在**特征空间建代价体** + 显式几何先验;单次拍摄 |
| **Active Stereo in the Wild / Virtual Pattern**(IJCV 2025) | 不靠物理投影仪,而**把与场景几何一致的虚拟图案投到左右立体图**(稀疏深度提示引导),从普通立体硬件获主动立体的鲁棒 |
| **ActiveNeuS / 结构光 SDF**(arXiv:2405.12006) | 用**神经符号距离场(SDF)** 从投射图案重建深度 |
| **Gated Stereo**(arXiv:2305.12955) | 主动门控立体出高分辨率长距离深度 |

### 两种设计哲学
- **拥抱物理投影仪 + 学习解码图案**(Neural Feature Decoding);
- **合成虚拟图案让被动立体表现得像主动**(Virtual Pattern Projection)。

> **本调研判断**:主动立体/结构光是攻 B4(透明/反光/无纹理)失败模式的**硬件级解法**——主动图案在那些 RGB 失效处仍提供几何信号。它与 B6(事件相机)、B32(雷达/热成像)、B46(偏振)同属"用额外物理信号补 RGB 病态性"的大类。Virtual Pattern Projection 的"用虚拟图案让被动立体变主动"尤其巧妙,呼应 B41(合成/变换数据补真实之缺)——**在算法里合成本该由硬件提供的信号**。**对选题**:结构光是消费级深度相机(iPhone Face ID、Kinect)的底层技术,"用基础模型先验增强结构光解码"或"用结构光的可靠 GT 蒸馏单目模型"都是软硬结合的可行方向,呼应 B14(PromptDA 用 LiDAR 提示)的"几何信号增强基础模型"思路。

## B56. 跨光谱 / 近红外 / 低光深度 —— RGB 失效时的光谱互补

> 接 B5(恶劣天气)、B6(事件相机)、B32(雷达/热成像)、B55(主动)。夜间/暗光下 RGB 光度一致性崩溃,跨光谱与暗光专门方法成为出路。

### 核心问题
夜间图像破坏自监督深度的**光度一致性假设**(A1 的基础)——暗区噪声大、亮区过曝、纹理消失。两条解法:换/加光谱,或专门处理暗光 RGB。

### 跨光谱 / 多光谱融合
| 方法 | 思路 |
|------|------|
| **Adaptive Stereo Multi-Spectral**(arXiv:2411.03638, 2024) | 融可见 + NIR + 热红外,**全光照条件**;核心"退化掩码"在弱光破坏立体匹配处回退到鲁棒的**单目热深度** |
| **Cross-spectral Gated-RGB Stereo**(Princeton) | 门控成像 + 主动照明,夜间保持一致照明(呼应 B55 主动) |
| **Cross-spectral Stereo Fusion**(Springer 2023) | 把跨光谱立体线索融进单目估计器 |

### 暗光专门 RGB 方法
| 方法 | 思路 |
|------|------|

### 热成像深度
- **Deep Depth from Thermal**(arXiv:2503.22060):恶劣天气/光照下鲁棒;

> **本调研判断**:暗光/跨光谱深度是 B5(恶劣天气)的近亲,共享"RGB 成像假设失效"根因,也共享解法谱:**换光谱(热/NIR)、加主动光、或合成退化数据训练**。Self-Supervised in the Dark 的"不用夜间图却泛化到夜间"呼应 md4all(B5)的昼→夜翻译思想。**与 `docs/ideas/可扩展研究方向.md` Idea 1(HDR×深度)的深层联系**:暗光与过曝是同一动态范围问题的两端——HDR 处理过曝、暗光方法处理欠曝,**两者本可统一为"非理想曝光下的鲁棒深度"**,这是个比单做 HDR 更完整的选题框架。Confidence-Aware Distillation 的"用 RGB 置信度引导跨模态蒸馏"也是可复用技巧:**当某模态在某区域失效时,用另一模态的置信度加权**,对一切多模态深度(B32/B55/B56)通用。

## H1. 为什么这张表本身就有陷阱(先读这段)

## B57. 产业应用 —— 农业 / 工业巡检 / 仓储 / 林业

> 接 B40(具身导航)、E3(下游集成)。本节落到地面:单目深度在真实产业里到底怎么用,以及一个反复出现的经济驱动力。

### 核心经济驱动:无传感器部署(sensor-free)
2024 最强主线是**用单目深度替代 LiDAR/结构光**,把深度能力**retrofit 进现有单摄像头**。这对成本敏感的农业、建筑、零售是采用的关键。

|----|------|------|

### 专利信号(技术成熟度指标)
2024 多项专利围绕核心技术:通用 MDE 框架、自监督光度代价体、**人体深度**、稀疏监督边界细化、**在线深度尺度迁移做绝对深度**——专利密度本身说明产业界在认真落地。

> **本调研判断**:产业应用揭示了一个与学术界不同的价值排序——**成本(免传感器)> 精度**。工业安全那篇"YOLO + 单目深度、retrofit 现有相机"尤其典型:产业界不追求 SOTA AbsRel,而要"够用且零硬件改动"。这呼应 B26(伪 LiDAR 的"表示>精度")、B47(AR 的"遮挡关系>精度值)——**真实应用几乎从不把平均 AbsRel 当首要指标**,而看任务成败、成本、retrofit 难度。"在线深度尺度迁移做绝对深度"的专利则是 E2(米制锚定)、B1(米制深度)在产业界的对应物,说明**尺度恢复是落地的真痛点**。**对选题**:产业落地的真瓶颈是"免传感器 + 够用米制 + 零改造",而非刷榜——这给偏应用的研究一个务实方向:不是更准的深度,而是**更易部署、尺度更可靠、成本更低**的深度系统。

## E5. 开源生态 / 工具链 —— 上手单目深度的实际路径

> 接 E1(模型规模)、B18/B42(Depth Anything/DA3)。本节是可立即验证的实操信息:用什么库、哪些预训练模型、怎么跑。

### 核心框架:Hugging Face Transformers
Transformers 把单目深度作为**一级任务**支持,API 跨版本稳定:
```python

pipe = pipeline("depth-estimation", model="depth-anything/Depth-Anything-V2-Small-hf")
depth = pipe(Image.open("input.jpg"))["depth"]
```

### Hub 上的主要预训练模型
| 模型 | 仓库 | 类型 |
|------|------|------|
| **Depth Anything V2** | `depth-anything/Depth-Anything-V2-{Small,Base,Large}-hf` | 相对,合成数据训,Transformers 直插 |
| **Depth Pro** | `Hnabil/depth-pro` | Apple 零样本**米制**,高分辨率锐利 |
| **MiDaS 3.1** | `qninhdt/MiDaS` | 相对,长青基线 |
| **Distill-Any-Depth** | `xingyang1/Distill-Any-Depth-Large-hf` | 跨教师**蒸馏**的近期 SOTA(呼应 B18) |
| **Marigold/ApDepth** | `developy/ApDepth` | 扩散式,走 **Diffusers** 而非标准 depth pipeline(B3) |

### timm 的角色
多数 Transformer 深度模型(DPT、Depth Anything)的主干(ViT/BEiT/DINOv2)来自或兼容 **timm**;Transformers 把它作特征提取器拉入,故 timm 通常是**传递依赖**而非直接调用。

### 推理之外的工具
- **Monocular-Depth-Estimation-Toolbox**(zhyever,基于 MMSegmentation):训练/评测工具箱;
- **FiftyOne + Transformers**:数据集上可视化深度预测、定性调试;
- **Diffusers**:扩散式深度(Marigold)的路径。

### 选型实用建议
- 需**真实米制**(测距/机器人)→ 米制模型(Depth Pro);
- 需**相对深度图**(排序/分割/3D 效果)→ Depth Anything V2(当前强默认 + 干净 Transformers 集成);
- 扩散法用推理速度换细节质量。

> **本调研判断**:开源生态的成熟本身是\"基础模型时代\"的标志——单目深度已从\"复现论文\"降级为\"`pipeline()` 三行调用\",门槛极低。这印证 G3 元配方的现实基础:**人人都能冻结一个强基础模型当起点**。Distill-Any-Depth 出现在 Hub 上(蒸馏多教师)也呼应 B18/B49——**蒸馏是把基础模型能力压进可部署模型的主流路径**。**对选题与实操**:任何 MDE 研究都应以\"`pipeline('depth-estimation', 'Depth-Anything-V2')` 能跑出什么\"为基线——若新方法不能明显超过这个三行基线,价值存疑。这是 2026 年做 MDE 的现实起跑线。

## B58. BEV 鸟瞰图感知中的深度 —— 自驾 3D 检测的隐形瓶颈

> 接 B26(伪 LiDAR)、B1(米制)、A9(相机感知)。自驾的多相机 3D 检测主流走 BEV(鸟瞰图)路线,而深度是其中决定成败的隐形瓶颈。

### 奠基:Lift-Splat-Shoot(LSS,arXiv:2008.05711)
核心:为每像素预测**离散深度箱上的类别分布**,据此把像素特征"lift"成 3D 视锥点(按深度分布加权),再"splat"到 BEV 栅格。**关键局限**:深度分布是**隐式学习**的,无直接监督告诉模型正确深度——这正是后续工作的靶子。

### BEVDepth(arXiv:2206.10092)—— 加显式深度监督
直击 LSS 软肋,三贡献:
- **显式深度监督**:把 LiDAR 点投到图像生成 GT 深度图,**直接监督**预测深度,而非让它从检测损失隐式涌现;
- **相机感知深度模块**:把内外参喂进深度网络(呼应 A9 CAM-Convs);
- **深度细化 + 高效体素池化**。
核心论断:**可靠深度是相机-only 多视图 3D 检测的瓶颈**,显式监督它才解锁精度。

### 2024–2025 演进
| 方向 | 代表 | 要点 |
|------|------|------|
| **不确定性深度** | GaussianLSS(arXiv:2504.01957) | 把深度建模为**高斯**,经 splatting 捕获深度**不确定性**(呼应 B23) |
| **冻结基础模型** | Frozen FM BEV(arXiv:2501.08118) | 插入冻结 DINOv2(特征)+ Metric3Dv2(深度),超基线 **7.4 IoU**——强现成深度先验可替代任务专用监督(呼应 B36/G3) |
| **几何建模** | Instance LSS(arXiv:2411.06173) | 实例级表示改进视图变换的物体几何 |

### ⚠️ 一个反直觉发现
相机-LiDAR 融合 Transformer(arXiv:2312.14919)报告:**朴素提升深度估计并不总转化为更好的检测**,某些融合并未如预期利用深度。这是对"更好深度=更好检测"假设的重要反例。

> **本调研判断**:BEV 深度是伪 LiDAR(B26)思想在现代自驾的主流化——但 BEVDepth 揭示的"深度是隐形瓶颈,需显式监督"比伪 LiDAR 更进一步。**最值得注意的是那个反例**(2312.14919):**更准的深度不一定带来更好的下游**——这与本文档反复强调的"任务导向评测"(B26/B47/H)形成微妙张力:深度对下游的贡献是**非线性、依赖架构**的,不能假设"深度涨点→任务涨点"。**Frozen FM BEV 超基线 7.4 IoU** 则是 G3 元配方(冻结基础模型当先验)在自驾的强证据——Metric3Dv2 的现成深度先验直接顶替了费力的 LiDAR 监督。**对选题**:这给"标定不确定性"(Idea 2)又一动机——GaussianLSS 证明 BEV 检测能**消费深度不确定性**(高斯而非点),所以产出标定不确定性的深度模型有明确下游买家。

## B59. 深度时序一致性度量 —— 如何量化"闪烁"(OPW)

> 接 B2(视频深度)、B17(动态)、B10(评测)。视频深度的核心质量是"不闪烁",但怎么**量化**闪烁?这一节补评测侧的关键工具。

### 核心度量:OPW(光流 warping 误差)
量化时序稳定的最广用指标,三步:
1. 用光流(常 RAFT)估相邻帧 t-1→t 的运动;
2. 用光流把 t-1 的深度图 **warp** 到 t 的坐标系;
3. 比较 warp 深度与 t 实际预测深度。

差异即闪烁:时序一致的模型在运动补偿后跨帧深度应一致。遮挡/去遮挡区(用前后向光流一致性检测)通常 mask 掉(那里 warp 无效)。NVDS(arXiv:2307.08695)推广 OPW 为标准并出 VDW 基准。

### 互补度量
- **时序深度梯度/平滑损失**:直接惩罚帧间深度变化(Super-Long Videos,arXiv:2501.12375);
- **重投影几何一致性**:测试时调优类方法用相机几何而非 2D 流强制 3D 一致;
- **必须与 AbsRel/δ 联报**:因为模型可**用时序平滑但不准的深度作弊**地最小化闪烁——稳定性必须与逐帧精度一起读。

### 实现陷阱(诚实标注)
| 陷阱 | 说明 |
|------|------|
| **光流质量是混杂因素** | OPW 继承光流误差,无纹理/快速运动区的差光流虚增 warping 误差 |
| **尺度对齐** | 相对/仿射不变模型需逐帧尺度+平移对齐再 warp,否则全局尺度漂移伪装成闪烁 |
| **遮挡掩码必需** | 无前后向一致性掩码则去遮挡区主导误差 |
| **动态场景破坏刚性假设** | 运动物体合法改变帧间深度;STATIC(arXiv:2412.01090)用静/动分割避免误罚合法运动 |

> **本调研判断**:OPW 是 B2(视频深度)的评测基石,也是 B10(评测争议)主题在时序维度的体现——**它同样有"对齐"陷阱(尺度漂移伪装成闪烁)和"作弊"陷阱(平滑但不准)**。"必须与 AbsRel 联报"正是 B10/H 部分"单一指标危险"主张的又一例。OPW 对光流质量、遮挡掩码、动态分割的依赖也说明:**时序一致性的"正确度量"本身是未完全解决的问题**——这呼应 B10(How to Evaluate MDE 把评测列为开放问题)。**对选题**:一个"动态感知、免光流混杂、与精度联合"的视频深度时序度量,是 Idea 5(评测基准)在视频维度的具体缺口;STATIC 的静/动分割是个起点但未必通用。

## B60. 语义占用预测(Occupancy)中的深度 —— 从 BEV 到稠密 3D 体素

> 接 B58(BEV 感知)。占用网络是 BEV 的稠密 3D 升级——不只预测鸟瞰平面,而是预测整个 3D 空间每个体素的占用与语义。深度在 2D→3D lift 里再次是关键。

### 核心问题
相机-only 占用网络成本低,但**从 2D 图恢复 3D 几何难**。深度是 2D→3D lift 的核心——深度错,体素就放错位置。

### 用深度增强占用的方法
| 方法 | 思路 |
|------|------|
| **Depth Awareness + Semantic Aid**(arXiv:2505.20951) | 显式深度感知 + 语义辅助模块,锐化 2D→3D lift |
| **Depth- and Region-Guided Occupancy**(arXiv:2603.01007,⚠️ID待核实) | 环视相机的深度引导 + 区域级线索 |
| **Monocular SSC via Depth-Spatial Alignment**(arXiv:2311.17084) | 单相机语义场景补全,深度与空间特征对齐 + 体素自适应 |

### 效率与不确定性
- **SparseOcc**(arXiv:2312.17118):稀疏 3D 表示 + 稀疏 query,避稠密体素计算;
- **Real-time Sparse Conv**(arXiv:2403.08748):稀疏卷积做车载实时;
- **Uncertainty-Aware OCC**(arXiv:2406.11021):加不确定性估计应对有限观测的可靠性(呼应 B23)。

> **本调研判断**:占用预测是 B58(BEV)的稠密 3D 延伸,深度的角色一致——**2D→3D lift 的几何基石**。它再次印证 BEVDepth 的核心论断:相机-only 3D 感知里**深度是瓶颈**。Uncertainty-Aware OCC 又一次说明下游(占用)能消费深度不确定性(B23/Idea 2)。**对全文论点的延续**:从伪 LiDAR(B26)→ BEV(B58)→ 占用(B60),深度在自驾 3D 感知里始终是"被显式监督/增强的中间几何表示",而非独立输出——又一个"深度作为更大系统组件"(G1/G4)的实例。**对选题**:占用网络对深度不确定性的明确需求,是"标定不确定性深度"(Idea 2)在自驾的又一买家;而"用基础模型深度先验顶替 LiDAR 监督"(B58 的 Frozen FM)在占用上同样可行。

## B61. 可微渲染 / 逆向图形监督 —— 分析合成(analysis-by-synthesis)

> 接 A1(光度重投影)、B11(3DGS)、B29(NeRF)。可微渲染让"渲染"可反传梯度,从而能用"渲染出的图像与输入的差异"监督深度——这是自监督深度的几何化升级。

### 核心思想:分析合成
不直接监督深度,而是:深度→几何表示→**可微渲染**回图像→比对输入图→反传优化深度。本质是 A1 光度一致性的推广——从"两帧 warp"升级为"完整可微渲染管线"。

### 代表方法
| 方法 | 思路 |
|------|------|
| **Robust Geometry-Preserving Depth via Diff Rendering**(arXiv:2309.09724) | 无需额外数据/标注,**渲染新视角强制真实 3D 结构**——强自监督深度 |
| **Multi-View Diff Rendering Refinement**(arXiv:2410.03861, 2024) | 取单目深度→网格化→**多视图可微渲染细化**,SfM 点云定全局尺度,优化多视图一致性(光度+几何损失) |
| **Diff SDF Rendering**(arXiv:2405.08733) | 把轮廓扩成薄带做 SDF 可微渲染 |

### 与 3DGS/NeRF 的关系
3DGS(B11)、NeRF(B29)本质都是**可微渲染**框架——它们用渲染损失优化 3D 表示,深度是副产品/正则。本节是把这一思想**显式用于深度细化/估计**。

> **本调研判断**:可微渲染监督是 A1(Monodepth2 光度重投影)的理论终点——**光度一致性 = 最简单的可微渲染(纯 warp)**,而完整可微渲染(网格/SDF/高斯 + 光照)是其完全体。这条线把自监督深度(A1/B50)、3DGS(B11)、NeRF(B29)、逆向图形统一在"用渲染损失反传监督几何"的框架下。Multi-View Refinement(2410.03861)的"单目深度→网格→多视图渲染细化"尤其实用:**用基础模型出粗深度,再用可微渲染做几何细化**,是 B25(高分辨率)、B52(超分)之外的又一种"基础模型 + 后处理细化"路径。**对选题**:可微渲染细化能直接套在任意基础模型深度输出上(模型无关),且零额外标注(自监督),呼应全文反复出现的"不与基础模型竞争,做它的细化器/放大器"主张(B25/B52/G4)。

## B62. 序数 / 相对深度标注与数据集 —— 众包排序的力量

> 接 合订本 [66](Depth in the Wild)、B43(VLM 空间推理)。这一节追溯一个关键洞察:人类标不准米制深度,却能可靠地判断"谁更近"——这撑起了"野外"深度数据。

### 奠基:Depth in the Wild(DIW,arXiv:1604.03901)
约 **49.5 万张野外图**,每张标注一对随机采样点的**相对深度序**(哪个更近)。核心洞察:**众包标注者对成对序数判断("A 比 B 近")远比米制深度估计可靠**,故能扩展到 RGB-D 传感器够不到的无约束真实图。配套用**排序损失(ranking loss)** 从纯序数标签学出类米制深度。

### 同一思想的延伸
| 方法 | 贡献 |
|------|------|
| **Surface Normals in the Wild**(arXiv:1704.02956) | 众包表面法向标注,助深度预测 |
| **Augmented Ordinal Relationships**(arXiv:1806.00585) | 从**立体视频自动挖**相对深度对,降众包成本/扩规模 |
| **Ordinal Depth for 3D Human Pose**(arXiv:1805.04095,CVPR 2018) | 把成对序数原则用于关节深度序(呼应 B54) |
| **Relative Human**(CVPR 2022) | 多人野外图众包"深度层"标注,序数范式用于多人场景 |
| **Amodal Depth Anything**(arXiv:2412.02336, 2024) | 相对深度 + 大规模数据,**amodal**(含遮挡补全)深度 |

### 序数监督的深远影响
DIW 的"排序损失"思想直接进入了基础模型时代——MiDaS 的尺度平移不变损失(D2)、Depth Anything 的相对深度本质都是"放弃绝对值、保排序结构"的延续。**相对深度 = 序数关系的稠密版**。

> **本调研判断**:DIW(2016)是个被低估的里程碑——它确立了\"**序数关系比米制值更易获取也更鲁棒**\"这一原则,这条原则贯穿到今天的基础模型(相对深度泛化好正因为它只需保排序,见 G2 主线 4)。它与 B43(VLM 度量弱但能判相对)、B38(人类式深度偏见)形成有趣呼应:**无论人还是模型,\"相对/序数深度\"都比\"绝对/米制深度\"更可靠**。这给 `docs/ideas/可扩展研究方向.md` 一个底层启示:**在序数空间做文章往往比在米制空间更稳**(如序数评测指标、序数一致性损失)。Augmented Ordinal 的"从立体视频自动挖序数对"也是 B41(数据引擎)的早期形态——用几何自动生成标注替代众包。

## B63. 深度 × 触觉 / 多感官融合 —— 视觉够不到时用触摸补

> 接 B40(具身导航)、B54(人体)、B32(多模态)。机器人抓取时,视觉常被手/物遮挡,触觉提供互补的近距几何。

### 核心动机:视觉遮挡时触觉补位
抓取瞬间手指遮挡目标,纯视觉深度失效。触觉传感器(GelSight 等)在接触点提供**局部高分辨率几何**,与视觉深度互补。

### 方法谱
| 方法 | 思路 |
|------|------|
| **PseudoTouch**(arXiv:2403.15107, 2024) | 学视觉-触觉低维嵌入,**从视觉预测"表面手感"**,无需真的触摸所有物体(视觉→触觉跨模态) |
| **In-Hand Pose via Visual-Tactile**(arXiv:2506.10787,⚠️ID待核实) | 触觉在**高遮挡**时显著提升位姿精度——正是纯视觉的失效模式 |
| **低分辨率触觉 + 视觉 + 本体感**(arXiv:2503.19893) | 二值低分辨率接触 + 视觉,证明**不必高分辨率触觉**也有增益 |
| **红外/可见图 3D 视触重建**(arXiv:2506.15087) | 表面法向估计精度提升 ~40% |

### 反复出现的局限
**对未见物体泛化弱**(arXiv:2206.05714):融合在已知物体 + 遮挡下可靠,但迁移到新物体仍弱。这是融合方法的共性瓶颈。

> **本调研判断**:触觉-视觉融合是 B32(雷达/热成像)、B55(主动)、B56(跨光谱)"用额外模态补 RGB 病态性"思想在**接触式近距**场景的延伸——共性是\"视觉在某条件失效时,用另一模态补\"。它与 B40(具身导航)、B54(人体)共同说明:**深度的价值最终在物理交互里兑现**,而物理交互恰恰是视觉最易失效处(遮挡/接触)。Confidence-Aware 跨模态加权(B56)、风险感知规划(B40)在此同样适用——**用触觉的高置信局部几何修正视觉深度的遮挡区**。**对选题**:视触融合是个有明确机器人需求、但泛化未解的方向;PseudoTouch 的"从视觉预测触感"反向思路(免去物理触摸)尤其新颖,与单目深度的"从单图推几何"哲学一致——都是用易得模态(视觉)推断难得信息(触感/深度)。
