# 评测协议与可比性

来源: `../综述/03_MDE前沿追踪_全文.md` 的 B10、B35、B38、B39、B47、B58、B59、H 部分,并吸收 C2、E1、E5、F 部分的待核实原则。

## 总判断

MDE 的评测最大风险不是“指标不够多”,而是协议混淆。相对深度、米制深度、对齐后指标、免对齐指标、视频时序指标、下游任务指标经常被混表比较。归档反复强调:

```text
先问协议,再看数字。
```

没有明确 relative/metric、crop/cap、对齐、分辨率、`nfe_real`、硬件和下游消费方式的数字,不应用来支撑强结论。

## Metric vs Affine-invariant

| 协议 | 做法 | 适用模型 | 风险 |
|---|---|---|---|
| Affine-invariant | 对预测和 GT 做 scale-shift/2-DoF 最小二乘对齐后算指标 | MiDaS、DPT、Depth Anything relative、Marigold | 对齐会吸收系统误差 |
| Metric | 直接用米制预测算 AbsRel/RMSE/δ,不做 median/affine 对齐 | Metric3D、UniDepth、Depth Pro、米制化 I1 | 更贴近落地,但相机/尺度要求更严 |
| Relative/ordinal | 只比较排序、边界或相对关系 | 3DGS 排序约束、DIW、AR 遮挡 | 不能说明真实测距能力 |

如果一个方法主张“米制 grounding”,必须进入 metric 协议。只在 affine 对齐后变好,不能称为米制改进。

## 对齐争议

归档的 B10 记录了 2025 年评测方法论争议:

- 现代基础模型常预测仿射不变深度,标准做法是对齐 GT 后评分。
- 对齐会掩盖真实尺度和平移误差,可能让弱模型显得更好。
- 第四届 MDE 挑战赛采用 2-DoF scale-shift 对齐作为相对深度协议,但这不等于 metric 协议。
- 好的评测应同时报告对齐前、对齐后和下游任务结果。

建议报告 `align_gain`:对齐后指标相比对齐前改善了多少。`align_gain` 过大说明模型本身尺度不可靠。

## 零样本表格的陷阱

归档 H 部分给出近似零样本 AbsRel 表,但强烈标注需核实。核心不是数字本身,而是警告:

- Depth Anything/MiDaS/DPT 的数值往往是对齐后。
- Metric3D/UniDepth/Depth Pro 的数值应按无对齐米制读。
- KITTI、NYUv2、DIODE、ETH3D 的 crop、cap、有效像素和 GT 密度不同。
- 同分布监督模型与零样本跨数据集模型也不可直接混比。

引用任何数值前必须回查原文和评测脚本。没有真实 CSV 时,本仓库一律写 `待跑`。

## `nfe` 与 `nfe_real`

扩散/流匹配深度必须区分:

- `nfe`:表面采样步数。
- `nfe_real`:真实调用 U-Net/denoiser/flow model 的次数。

如果方法为了单步 CCF、Tweedie 代理或多时间点速度场额外调用模型,就必须把这些调用计入 `nfe_real`。速度表还应报告硬件、分辨率、batch、显存和是否 ensemble。

## 视频时序一致性:OPW

视频深度的核心质量是“不闪烁”。OPW 是常用度量:

1. 用光流估相邻帧运动。
2. 用光流把上一帧深度 warp 到当前帧。
3. 比较 warp 后深度与当前帧预测。

必须注意:

- 光流质量是混杂因素。
- 遮挡/去遮挡区域需要 mask。
- 相对深度模型要处理逐帧尺度漂移,否则尺度变化会伪装成闪烁。
- OPW 不能单独看,因为模型可以输出平滑但错误的深度作弊。

视频评测应联报逐帧精度、OPW、动态/静态区域分解和长视频显存/延迟。

## 鲁棒性和损坏基准

标准 KITTI/NYU 干净图高分不等于真实鲁棒。归档的 B35 总结了多类鲁棒性评测:

| 轴 | 内容 |
|---|---|
| 常见损坏 | 噪声、模糊、压缩、天气、亮度变化等 |
| 程序化扰动 | 受控改变材质、纹理、几何、光照,隔离失败根因 |
| 对抗威胁 | 物理 3D 对抗物体、对抗训练 |
| 专门域 | 内窥镜、自驾、夜间、林业等 |

理想基准应覆盖失败模式切片、损坏鲁棒、任务导向和免对齐/少对齐协议。归档认为当前没有单一基准完整覆盖这四点。

## 系统性偏见

归档 B38 的重要发现是:深度领域几乎没有算法层“肤色公平性”文献,但有人类式系统性深度偏见研究:

- 深度压缩:远距离被低估。
- 垂直偏置:下视场物体更容易被认为近。
- 凸性偏好。
- 精度更高不等于误差模式更合理。

这再次说明平均指标会掩盖结构化错误。程序化扰动基准可以用受控场景检查模型是否靠先验记忆,还是在做真正几何推断。

## 任务导向指标

| 下游 | 更相关的指标 |
|---|---|
| 3DGS/NeRF | novel view PSNR/SSIM/LPIPS、浮点伪影、几何一致性 |
| BEV/3D 检测 | NDS、mAP、IoU、体素位置误差、深度分布质量 |
| Occupancy | mIoU、体素占用/语义错误、uncertainty-aware lift |
| AR | 遮挡边界 IoU、遮挡翻转率、穿帮率、延迟 |
| SLAM | ATE/RPE、尺度漂移、跟踪丢失率 |
| 导航/规划 | collision rate、risk calibration、rollout drift |
| VLM 空间推理 | 距离/方位误差、3D QA 正确率 |

归档中 BEV 的反例很关键:更准深度不一定带来更好检测。下游消费方式必须和深度指标一起设计。

## 能耗与边缘效率

归档 B39 认为 MDE 专门能耗/碳足迹基准几乎空白。实际部署中,推理能耗往往比训练更重要,因为机器人、AR、车载系统持续运行。

建议未来评测至少记录:

- latency / FPS。
- 显存峰值。
- 能耗 Wh/帧 或 Joule/帧。
- 模型参数量和 FLOPs。
- 硬件、精度模式、TensorRT/ONNX/CoreML 等推理栈。
- 精度-能耗帕累托曲线。

这对比较 Depth Anything V2 小/大模型、Marigold 多步扩散、Mamba 轻量模型和单步扩散尤其关键。

## 数据集可比性

| 基准 | 特点 | 注意 |
|---|---|---|
| KITTI | 室外驾驶,稀疏 LiDAR GT | crop/cap 约定影响大 |
| NYUv2 | 室内 RGB-D | 室内同分布结果不等于零样本 |
| DIODE | 室内+室外同一套件 | 室外 GT 稠密度较低 |
| ETH3D | 高分辨率真实立体/SLAM | 深度非均匀稠密 |
| iBims-1 | 高质量室内边界 | 适合边界/结构检查 |
| vKITTI2 | 合成驾驶全稠密 GT | 合成到真实域差 |

专门失败模式还需要 Booster、SCARED、水下、恶劣天气、事件相机、全景、鱼眼等数据集。不要把“平均泛化”替代“失败切片”。

## 本仓库建议报告模板

最少报告:

```text
模型:
预测类型: relative / metric / ordinal / distributional
评测协议: no-align / median-scale / scale-shift / affine-invariant
数据集与 split:
crop/cap/有效像素:
指标: AbsRel/RMSE/delta + 对应任务指标
align_gain:
failure slices:
nfe 与 nfe_real:
硬件/分辨率/延迟/显存:
待核实项:
```

对 I1:

- metric 主表。
- affine 辅表。
- `nfe_real <= 2` 的诚实速度。
- 采样期锚 vs 全局后处理 vs patch-wise 后处理。

对 I2:

- ECE/NLL/AUSE。
- failure-slice uncertainty lift。
- 下游错误传播减少。

对评测类 idea:

- 必须证明新协议揭示了旧平均指标看不到的错误,而不是只增加一个表格。
