# 扩散深度与近单步米制化

来源: `../综述/03_MDE前沿追踪_全文.md` 的 B2、B3、B24、B28、B33、B39、B49、B59,并吸收 E4、G 部分关于效率和生成式几何的判断。

## 核心判断

扩散式深度的价值在于把大规模生成模型的视觉先验转成稠密几何输出。它的痛点也清楚:Marigold 式多步采样和多 seed ensemble 太慢,且输出多为 affine-invariant 相对深度。2024-2026 的主线就是把它压到 1-2 步,并尝试把相对深度后验锚到真实米制尺度。

对 I1 类方向而言,单独“单步”已经不够新,单独“米制锚”也不够新。可争取的缝是:

```text
training-free + near-single-step + diffusion depth + sampling-time metric grounding + honest nfe_real
```

## Marigold 范式

Marigold 把 Stable Diffusion 的潜在扩散模型改造成单目深度估计器:

- RGB 和深度都经冻结 VAE 编码到 latent space。
- 图像 latent 与带噪深度 latent 拼接后送入 U-Net。
- 冻结 VAE,主要微调 U-Net。
- 使用合成深度数据训练,输出 affine-invariant depth。
- 推理从随机深度噪声开始迭代去噪,再经 VAE 解码成深度图。
- 多 seed ensemble 通过 scale-shift 对齐和逐像素中位数降低方差。

优势是 zero-shot 泛化强,代价是 50 步乘多 seed 的推理成本,并且不是直接米制。

## 提速谱系

| 工作 | 核心手段 | 是否训练 | 对 I1 的意义 |
|---|---|---|---|
| E2E-FT | 修 trailing timestep 调度缺陷,未微调 Marigold 单步已很强,再端到端微调 | 含免训练起点和微调版本 | 最大撞车对象,证明单步本身不够新 |
| Lotus | 训练单步扩散深度,x0-pred 和细节保持 | 需要训练 | 训练派单步对照 |
| DepthFM | flow matching / OT 在训练期拉直噪声到深度路径 | 需要训练 | OT 直化已出现,但在训练期 |
| FE2E | 从编辑器型 DiT/LoRA/FM loss 转稠密估计 | 需要训练 | 编辑器到 dense estimator 的近邻 |
| Marigold-SSD / Need for Speed | 单步或少步深度/补全 | 需要微调或稀疏深度 | 不是纯 training-free 单目 |

分水岭不是“是否一步”,而是提速发生在训练期还是推理期。I1 若主张免训练推理期轨迹塑形,必须直接对照 E2E-FT 的调度修复、DepthFM 的训练期 OT 和 Lotus 的训练单步。

## ChordEdit/CCF 可迁移内核

ChordEdit 在图像编辑中提出 Chord Control Field,用去噪时间窗口内的速度/漂移场加权平均,把弯曲轨迹压成低能近似直线,从而单步编辑。归档强调两点:

- ChordEdit 的“时间”是去噪步,不是视频帧。
- CCF 是原论文正式术语,不是本库自造命名。

归档记录的简化公式来自其 pipeline:

```text
u_hat = (delta * dv_s + t_s * dv_s0) / (t_s + delta)
```

迁到深度不平凡,因为编辑任务的源端和目标端相对明确,而深度任务的目标端 `d0` 正是未知量。I1 的技术赌注是用 Tweedie/x0 prediction 产生代理目标锚 `d_hat0`,再围绕这个代理锚构造时间窗 CCF。

## ChordEdit 迁移的偏差陷阱

归档中最关键的风险标注是:ChordEdit 的时间平均理论依赖测量误差为零均值,平均能降方差。但深度任务里的 Tweedie/x0 代理是 MMSE 估计,可能存在系统偏差。时间平均压不掉偏差。

因此 I1 不能把“ChordEdit 成功”直接外推为“深度也会成功”。必须通过消融证明净增益来自:

- 几何锚纠偏,而不是单纯降方差。
- 多时间点代理误差的偏差变化能互相抵消。
- CCF 构造优于单点 x0、多噪声平均和普通趋势外推。

这条风险是审稿人最可能攻击的位置。

## 相对深度到米制锚

扩散深度通常输出 affine-invariant 相对深度。归档列出的采样期或免训练米制化近邻如下:

| 工作 | 锚源 | 关键区分 |
|---|---|---|
| GeoDiff | 立体/双视图几何 | 免训练采样期几何引导,但不是单目 |
| Defocus-Marigold | 散焦模糊线索 | 物理锚不同,证明推理期注入成像线索可行 |
| AnchorD | 传感器深度 + 因子图/patch-wise affine | 抬高后处理对照门槛 |
| CoSIGN | 少步 guidance 的软约束/投影 | 说明普通逐步梯度引导在极少步时容易失效 |

I1 的米制贡献必须赢两类后处理:

1. 全局 scale-shift 标定。
2. patch-wise affine 或 AnchorD 式局部后处理。

只赢全局标定不够,因为局部后处理已经能解释很多“采样期注入”的表面收益。

## 视频深度与视频扩散

视频深度的核心问题是逐帧闪烁。归档把方法分成两派:

| 派别 | 代表 | 特点 |
|---|---|---|
| 前馈/回归式 | Video Depth Anything、oVDA | 建在强图像深度骨干上,强调超长视频和效率 |
| 视频扩散先验式 | ChronoDepth、DepthCrafter | 借 Stable Video Diffusion 等视频生成先验,质量高但成本重 |

ChronoDepth 的重要训练洞察是先训空间层,再冻结空间层训时序层。DepthCrafter 强调无需位姿/光流额外输入也能生成时序一致长深度序列。Video Depth Anything 代表更工程化的方向:用基础模型空间先验,轻量适配到时序域。

## 生成式几何:从视频生成到单图 3D

归档中的 B24 和 B33 把扩散深度放进更大的生成式几何图景:

- Aether、FantasyWorld、CVD-STORM 等把深度/几何直接融进视频生成器,不是事后加深度图。
- 单图到 3D 从 3D Photography 的分层深度修复,发展到 VistaDream、A Recipe for 3D Worlds 等扩散 RGB-D 修复和显式几何管线。
- 深度提供可见区几何,扩散负责不可见区修复和补全。

这说明扩散深度不只是一种深度估计器,也是生成式世界模型和 3D 内容生成的几何接口。

## 散焦/聚焦深度作为物理锚

DfD/DfF 方向说明成像物理能提供比单图语义先验更可靠的深度线索。散焦模糊与离焦距离相关,焦栈、双像素、事件焦栈和显式 PSF 模型都能把模糊转成几何约束。

对扩散米制化的启发是:Defocus-Marigold 已经证明把物理线索注入 Marigold 类模型有先例。I1 若改用单目几何尺寸、锚点或场景先验,必须说明锚源比散焦/双视图/传感器锚更通用或更低成本。

## 效率、能耗与 `nfe_real`

扩散深度的效率不能只报表面步数:

- `nfe` 是用户设置的采样步数。
- `nfe_real` 是实际调用 U-Net/denoiser 的次数。

如果方法为了构造 CCF 访问两个时间点或额外 x0 预测,即使最终看起来“一步输出”,也必须诚实报告 `nfe_real`。归档把能耗/绿色 AI 也列为缺口:Marigold 多步采样、Depth Anything 前馈、Mamba 轻量模型、单步扩散之间缺少统一 Wh/帧、延迟和精度帕累托评测。

## 建议的核心消融

| 问题 | 必须对照 |
|---|---|
| 单步是不是只靠调度修复 | E2E-FT trailing timestep 修复 |
| CCF 是否只是单点 x0 的装饰 | 单点 Tweedie/x0、多噪声平均、ETC/趋势外推 |
| 采样期米制锚是否真有用 | 全局 scale-shift、patch-wise affine、AnchorD 式后处理 |
| 近单步是否损失过大 | 多步 guidance、Marigold 原多步 ensemble |
| 速度收益是否真实 | 统一分辨率、同硬件、报告 `nfe_real`、延迟、显存 |

归档的定位很明确:I1 的证据重心不应只是主表涨点,而应是机制消融和协议诚实。
