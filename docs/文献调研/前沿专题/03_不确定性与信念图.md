# 不确定性、标定与 Belief Map

来源: `../综述/03_MDE前沿追踪_全文.md` 的 B7、B17、B23、B29、B30、B31、B40、B52、B54、B58、B60、B63,并吸收 G 部分对“可靠先验组件”的总结。

## 总判断

点估计深度只回答“深度是多少”,不回答“该信多少”。在 SLAM、3DGS、BEV、Occupancy、导航、AR、VLM 空间推理等下游里,错误深度被当作确定事实会污染几何状态。归档把不确定性方向的上位问题改写为:

```text
确定性深度/点图 -> 可校准 belief map -> 下游按信任度消费几何
```

因此 I2 不应只是给 Depth Anything 加一个方差头,而应证明 belief map 能减少错误几何传播。

## 不确定性的两类来源

| 类型 | 含义 | 典型场景 | 常见估计方式 |
|---|---|---|---|
| Aleatoric | 数据固有歧义或传感器噪声 | 透明、反光、远距离、暗光、遮挡边界 | 单次前向可估,如异方差/证据回归 |
| Epistemic | 模型没见过该域或几何 | OOD 场景、罕见结构、敌对成像 | ensemble、MC dropout、测试时扰动或轻量 proxy |

对实时下游系统,多次采样或 ensemble 往往太慢,所以单次前向的 evidential/分布式表示很有吸引力。但单次 UQ 容易“错而自信”,必须做校准。

## 方法谱

| 路线 | 表示 | 优点 | 风险 |
|---|---|---|---|
| 分箱/分类 | 每像素 bins softmax | 天然给分布、熵、方差;承接 AdaBins/Binsformer | 依赖离散化设计 |
| Evidential regression | 输出分布超参数 | 单次前向、轻量 | 容易过度自信,需强校准 |
| 条件生成/扩散采样 | 多样本深度分布 | 表达多模态歧义 | 慢,不适合闭环实时 |
| Gradient/motion cues | 用图像梯度、运动、残差识别歧义 | 可解释 | 与纹理、光流质量混杂 |
| 下游反推 | 根据 3DGS/NeRF/BEV 残差调整置信 | 与任务直接相关 | 协议复杂,不易独立比较 |

分箱方法是隐藏的概率模型:逐像素 softmax 本来就是深度分布,取期望得到点估计,取熵/方差得到不确定性。归档把这条线从 DORN、AdaBins、Binsformer 延伸到概率深度。

## 从 depth uncertainty 到 3D belief map

深度方差只是相机射线方向的一维不确定性。几何基础模型更自然的状态是点图或 3D 坐标的分布:

```math
p(\mathbf{X}(p)\mid I_{\le t})
= \mathcal{N}(\boldsymbol{\mu}_X(p), \boldsymbol{\Sigma}_X(p))
```

其中 `X(p)` 可以是点图坐标、3D Gaussian 均值、体素占用位置或场景流。这个表示比 `d, sigma_d` 更贴近世界模型和下游几何系统。

## 校准指标

| 指标 | 检查内容 |
|---|---|
| ECE / calibration curve | 置信度与真实误差频率是否匹配 |
| NLL / CRPS | 预测分布质量 |
| Sparsification error / AUSE | 按不确定性剔除像素后误差是否下降 |
| Failure-slice uncertainty lift | 透明/反光/暗光/动态/远距离等切片是否更不确定 |
| Downstream error propagation | 使用不确定性后,3DGS/BEV/导航等下游错误是否减少 |

平均 AbsRel 不能证明 uncertainty 有用。必须报告校准曲线、失败切片和至少一个下游消费结果。

## 多任务与几何耦合

深度、语义、法向、光流和场景流不是独立任务:

- 表面法向是深度的空间导数。
- 语义边界常与深度不连续对齐。
- 光流、相机自运动和深度共同解释像素跨帧运动。
- 场景流是 3D 运动的统一表示。

归档中的 B7/B30/B31 把这条线展开为两类方法:

| 类型 | 代表思路 | 与 UQ 的关系 |
|---|---|---|
| 跨任务注意力 | 深度、语义、法向、边缘互相 attend | 任务间不一致可作为不确定性信号 |
| 物理/几何约束 | 法向-深度一致、内在图像分解、刚性/非刚性运动分解 | 约束违背处通常是高风险区域 |

动态物体尤其关键。Monodepth2 的自动遮罩只是早期启发式,2024 方法转向光流解耦、基础模型分割和动态区监督。动态区天然应被视为高不确定区域,否则会污染 SLAM/世界模型状态。

## NeRF/3DGS 对不确定性的需求

NeRF 和 3DGS 都常把单目深度作为稀疏视角重建的几何先验。但单目深度有尺度歧义和局部错误,naive 监督会让错误几何固化。归档总结了演进:

1. 稀疏精确深度监督,如 SfM 点。
2. 稠密补全/单目深度先验。
3. 不确定性感知、相对/排序约束、标准差加权和分布式监督。

这直接支持 I2:可校准 belief map 是比“深度伪 GT”更合理的下游接口。低置信区域应降权,而不是强行监督。

## BEV 和 Occupancy 消费不确定性

相机-only 3D 感知需要把 2D 特征 lift 到 3D。LSS 用离散深度分布 lift-splat 到 BEV,BEVDepth 加显式深度监督。2024-2025 的 GaussianLSS 把深度建模为高斯,说明 BEV 检测可以直接消费深度不确定性。

Occupancy 是 BEV 的稠密 3D 版本。Uncertainty-Aware OCC 说明有限观测、遮挡和体素推断同样需要可靠性估计。对 I2 来说,BEV/Occupancy 是比“深度表格涨点”更有说服力的下游买家。

## 具身导航与风险

机器人导航不在乎 AbsRel,只在乎是否撞上。归档中 Risk-Aware MPC 的设计哲学是:不要硬信噪声深度,而是把深度当上下文,显式预测碰撞风险分布。这是 belief map 的强应用场景。

对闭环系统,不确定性要服务三个问题:

- 哪些几何可用于规划?
- 哪些区域需要保守绕行或主动感知?
- 错误深度是否会在 rollout 中累积?

## 人体、触觉和多感官

人体中心深度说明有强类别先验时,深度歧义可以用 SMPL/人体尺度约束。Probabilistic HMR 把多个合理人体网格作为分布输出,与概率深度思想一致。人体平均高度也可作为米制尺度锚,但只能在含人场景中使用。

触觉-视觉融合则说明在遮挡和接触场景,视觉深度本身会失效。触觉提供局部高置信几何,可以修正视觉深度的遮挡区。归档指出泛化到未见物体仍弱,这也是 belief map 需要表达 epistemic uncertainty 的原因。

## 引导式深度超分的教训

Guided Depth SR 输入低分辨率深度和高分辨率 RGB,用 RGB 边界引导上采样。它的核心陷阱是 texture-copy:把 RGB 纹理错误复制进深度。B52 的教训对所有 RGB 引导深度任务通用:

- RGB 边界有用。
- RGB 纹理不一定是几何。
- 共享特征与模态专属特征要分开。

对 belief map 来说,跨模态不一致区域应提高不确定性,而不是被强行融合。

## I2 的最小可证命题

一个可检验的 I2 命题应形如:

```text
在冻结强深度/点图骨干上,学习一个轻量、可校准的几何 belief map,
使高风险失败切片的不确定性上升,
并在至少一个下游几何系统中减少错误传播。
```

最小证据包括:

- 深度或点图误差与预测不确定性的相关性。
- ECE/NLL/AUSE 等校准指标。
- 透明/反光/动态/暗光/远距离等 failure slice 的 uncertainty lift。
- 下游使用不确定性后性能或风险改善,例如 3DGS 几何伪影减少、BEV/Occupancy lift 更稳、导航 collision/risk 指标改善。
