# 单目深度估计的**基础模型**时代综述

对应 PDF:

- `../单目深度估计的基础模型时代综述_TechRxiv_2025.pdf`

**这篇综述把 foundation model 时代的 MDE 归纳为四个核心挑战:数据扩展、几何细节、尺度/相机映射、视频时序稳定。**

对 A2 最关键的是第三条:主流相对深度基础模型仍然需要外部先验、相机参数、稀疏测量或后处理来获得 metric depth。A2 的价值必须证明“采样期注入稀疏几何锚”不是普通 scale/shift 后处理,而是能在无锚区域传播 metric 修正。

## 原文分类框架

原文的 MDFM taxonomy 按 problem-driven objective 排:

| 挑战 | 原文表述 | 典型技术路线 | 对 A2 的意义 |
|---|---|---|---|
| 数据覆盖不足 | Expanding training horizons | 伪标签生成、合成数据、teacher-student、domain adaptation | Depth Anything/DA V2 的强点来自数据引擎,不是结构魔法。A2 不应声称重训数据优势,应强调免训练/锚定/采样期机制。 |
| 几何细节损失 | Preserving geometric fidelity | 多尺度特征、patch refinement、diffusion refiner、语义/文本/相机辅助 | A2 若只提升全局 AbsRel 不够,需要看边界、细薄物体、远处小物体和 failure slices。 |
| 尺度歧义 | Resolving scale ambiguity | canonical camera、projection transform、scale/shift invariant loss、相机/FOV建模 | A2 的主战场。必须严格区分 relative/affine-invariant/metric,并把 metric 零对齐作为主表。 |
| 时序不稳定 | Ensuring temporal consistency | temporal layer、sliding window、光流/pose约束、视频扩散先验 | 当前 A2 是单图机制,暂不主打视频。可在局限性中说明,不要越界声称时序一致性。 |

这个分类对写作很有用:它避免把所有 foundation models 混成一类,而是让 A2 明确落在“尺度歧义 + 免训练适配 + 扩散采样期约束”交叉点。

## 重点内容拆解

### 1. 数据扩展:Depth Anything 路线的本质

原文把 Depth Anything 作为伪标签范式代表:用 DINOv2 encoder 强化 teacher,再给大规模无标签图生成伪标签训练 student。Depth Anything V2 则进一步引入合成数据增强 teacher,再对真实无标签图产更高质量伪标签。

调研判断:

- 数据扩展路线已经非常拥挤,且门槛在数据规模和 teacher 质量。
- A2 不应去争“更大数据、更强 teacher”,而应使用这些模型作为 frozen backbone 或 reviewer-threat baseline。
- 如果写 introduction,可把 Depth Anything 系列作为“相对深度基础模型已强,但 metric 接地仍不自动解决”的证据。

### 2. 几何细节:patch/refiner/扩散分支是强威胁

原文把几何细节增强分成三类:

- 多尺度特征融合:Depth Pro、PrimeDepth、DAR 等。
- 细粒度校正分支:PatchRefiner、PatchFusion、PRO、MultiDepth、SharpDepth、BetterDepth、GeometryCrafter 等。
- 辅助信息增强:语义分割、文本描述、相机标定。

对 A2 的风险:

- PatchFusion/PatchRefiner 类方法会被审稿人拿来质疑:“你是不是只是一个 patch-wise 后处理/refiner?”
- BetterDepth/SharpDepth 说明 diffusion refiner 已经被用于细节增强,不能把“扩散 + 深度 refinement”当作新意。
- 因此 A2 的 L1 gate 必须保留 `Bp_postproc_patch_ccf` 这种同源后处理对照,否则结论不干净。

### 3. 尺度歧义:metric depth 是核心未闭合问题

原文明确指出,很多主流 MDFMs 属于 relative depth,要输出 metric depth 往往需要 fine-tuning、domain-specific priors、校准数据或 scale 参数估计。代表路径包括:

- Metric3D/Metric3D V2:canonical camera model,学习更稳健的 pixel-depth mapping。
- UniDepth/UniDepth V2:利用或预测相机参数,推动 universal metric depth。
- ZeroDepth、Depth Pro、GRIN、MoGe 等:从不同角度解决 scale-aware/metric depth。
- 后处理/适配类:稀疏 LiDAR/SfM/IMU、SAM 分区 scale-shift、语言描述、人高先验、DEM、depth-from-focus/defocus blur。

对 A2 的直接启示:

- A2 的“几何锚”属于 external metric prior,和 sparse measurement/defocus blur/region-aware scale adaptation 属同一大类威胁。
- A2 必须强调机制差异:不是最后把 relative depth 拟合到锚,而是在扩散采样轨迹中注入锚,让 backbone prior 传播到无锚区域。
- 评测上必须主表 metric 零对齐,另附 affine-invariant 对照。否则很容易被 scale+shift 对齐掩盖真假 metric 能力。

### 4. 适配策略:三类方法都构成 reviewer 问题

原文第 7 节把 foundation model adaptation 分为三类:

| 适配策略 | 优点 | 局限 | 对 A2 的威胁 |
|---|---|---|---|
| 目标域直接微调 | 简单、通用 | 需要目标域标注,计算贵,可能过拟合 | E2E-FT、Surgical Depth Anything、DepthDark 等说明“重训/微调”很自然。A2 需突出免训练。 |
| 轻量可训练组件 | 参数高效,保留 backbone | 依赖模型结构,仍要训练 | LoRA/adapter/calibration token 类方法可作为“训练式适配”对照,但不是 A2 主威胁。 |
| 辅助信息生成 scale 参数 | 不重训,工程实用 | 强依赖外部信息质量 | AnchorD、sparse measurement、defocus、DEM、人高先验等是 A2 最接近威胁。 |

A2 的定位应写成:training-free metric grounding during sampling, not post-hoc scale recovery。

## 原文实验部分的读法

原文实验覆盖四类能力:

- 深度精度:NYU-Depth V2、KITTI,报告 ARel 和 δ1。
- 细节保持:Spring、Sintel、iBims,看 boundary F1。
- 扰动鲁棒性:Infinigen 生成 perturbation,看平均误差与稳定性。
- 下游任务:depth completion、stereo matching、monocular 3DGS、SLAM。

不能直接照抄数字做结论,原因:

- 表 1 同时包含 relative 与 metric 方法,协议容易混淆。
- 不同论文来源的训练数据、模型尺寸、推理开销未完全统一。
- 有些 2025 方法是 arXiv/会议新文,最终状态需回查。

但可以借鉴评测维度:

- A2 主表:metric 协议零对齐。
- A2 对照表:affine-invariant 协议,说明相对深度质量。
- A2 诊断:boundary/thin object/far object/no-anchor region。
- A2 成本:必须报 `nfe_real`,因为扩散类方法的推理成本是审稿高频问题。

## 对 A2 的机会图

### 机会 1:免训练 metric grounding 仍然有空间

综述认为 zero-shot metric depth estimation 仍是 future direction。现有方案要么依赖相机模型,要么依赖外部 calibration/sparse measurements,要么需要 fine-tuning。A2 若能在冻结 Marigold/E2E-FT/Lotus/DepthFM 骨干上用少量锚完成 metric 接地,并优于后处理,就落在真实缺口上。

### 机会 2:采样期传播是可防守的新意

后处理 scale/shift 只能在预测图上做拟合;patch-wise 后处理会受 patch 边界和无锚区域退化影响。A2 的主张应限定为:

> 稀疏 metric 锚在采样期被注入 latent/depth 往返轨迹,经去噪先验传播到无锚区域,因此优于同源 CCF 预测上的 patch-wise metric 后处理。

这句话必须由 L1 gate 支撑,不能只靠视觉例子。

### 机会 3:近单步效率正好回应 foundation model 部署痛点

综述第 9.7 把 limited-resource inference 列为开放问题。A2 的 `nfe_real <= 2` 如果成立,可以把“免训练 + metric + 近单步”作为部署友好卖点。需要严格计数真实 forward,避免被质疑偷换 NFE。

## 对 A2 的高风险点

| 风险 | 为什么危险 | A2 应对 |
|---|---|---|
| 被归类为后处理 scale recovery | 综述已有大量 auxiliary scale 参数方法 | L1 必须证明采样期注入优于 `Bp_postproc_patch_ccf`。 |
| 被 Metric3D/UniDepth/Depth Pro 压制 | 它们是 metric depth 强基线 | 主表必须列这些,但 A2 可强调训练免调、少锚条件、扩散骨干可插拔。 |
| 被 AnchorD/defocus/DepthFM 等撞车 | 都是外部锚或生成式深度相关 | 写 related work 时按“锚来源、注入位置、是否训练、是否采样期传播”区分。 |
| affine 协议误导 | relative 方法对齐后可能很好看 | metric 零对齐主表 + affine 对照,并解释二者语义不同。 |
| NFE 不透明 | 扩散类方法常被质疑慢 | 全表报告 `nfe_real`,不要只写 nominal step。 |

## 可直接写进 A2 related work 的中文骨架

基础模型时代的单目深度估计已经从单数据集监督学习转向混合数据、合成数据和大规模伪标签驱动的泛化模型。Depth Anything 系列证明了大规模无标签图像和强 teacher 伪标签能显著提升相对深度的零样本泛化;Metric3D、UniDepth 和 Depth Pro 则进一步面向米制深度、相机参数和细节边界质量。然而,大量方法仍依赖相机模型、外部校准、稀疏测量、语言/人高/DEM 等先验,或需要目标域微调,才能把相对深度稳定转为 metric depth。与这些方法不同,A2 不重训基础模型,而是:

**在扩散采样期注入稀疏几何锚,检验 metric 修正是否能通过去噪先验传播到无锚区域**,并以同源 patch-wise 后处理作为关键对照。
