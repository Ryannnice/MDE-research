# 失败模式与数据

来源: `../综述/03_MDE前沿追踪_全文.md` 的 B4、B5、B6、B13、B20、B21、B25、B28、B32、B41、B46、B48、B51、B55、B56,并吸收 C 部分数据集全景和 F 部分待核实清单。

## 总判断

基础模型平均指标强,但失败集中在物理成像假设被破坏或相机模型不匹配的区域:

- 透明/反光:折射、反射破坏纹理到几何的对应。
- 暗光/过曝/HDR:信息饱和、噪声或动态范围不足。
- 雨雾雪、水下:散射、衰减、后向散射。
- 动态物体:静态场景假设失效。
- 鱼眼/全景:针孔投影假设失效。
- 高分辨率/细结构:模型训练分辨率与显存瓶颈。
- 专门域:手术、内窥镜、遥感、农业、林业等域差显著。

这些失败模式不是靠普通平均 AbsRel 能暴露的,必须切片评测和定制数据。

## 数据集全景

| 类别 | 数据集/资源 | 用途 |
|---|---|---|
| 大规模合成 | Hypersim、TartanAir、Virtual KITTI 2、Infinigen、Kubric | 提供干净稠密 GT,训教师或做物理切片 |
| 真实混合训练 | Taskonomy、DDAD、DIML | 提供真实外观和跨域覆盖 |
| 零样本评测 | DIODE、ETH3D、iBims-1、vKITTI2 | 测跨室内外、真实/合成、稀疏/稠密泛化 |
| 专门失败模式 | Booster、SCARED、恶劣天气数据、水下基准、事件相机数据 | 针对透明、手术、天气、水下、HDR/高速等失败 |

归档提醒:不要笼统说“稠密 GT”。DIODE 室外、ETH3D 等都有有效像素和稠密度限制。

## 透明/反光/非朗伯

透明与反光破坏单目、立体、ToF 和 LiDAR 的常见假设。镜子显示反射场景深度,玻璃产生透射和折射,模型容易把可见纹理误当物体表面。

关键资源:

- Booster:室内透明/镜面表面稠密 GT,专攻非朗伯和泛化。
- Seeing and Seeing Through the Glass:多层深度,强调透明物体不是单层深度能完整表达。
- 反射检测和机器人建图工作:说明玻璃/镜子会直接破坏导航。

方法趋势是生成式先验、针对性合成数据、多层深度和内在图像/反射分解。对 I2,这些区域应高不确定;对评测基准,这些区域必须单独切片。

## 恶劣天气、暗光和 HDR

md4all 的范式是昼到恶劣天气翻译 + 蒸馏:用干净日间图的可靠预测监督恶劣图输入,绕开恶劣条件 GT 稀缺。后继包括 ACDepth、Structure-Centric、Robust-Depth 和扩散/对比学习路线。

低光和跨光谱方向补充了同一问题:

- Self-Supervised MDE in the Dark 专攻暗光光度一致性失效。
- DepthDark 用炫光/噪声仿真合成真实退化。
- 热成像深度和 RGB 到热模型蒸馏解决夜间/零光照。
- HDR/过曝与暗光是动态范围问题的两端,可以统一为“非理想曝光下的鲁棒深度”。

归档判断:恶劣天气单独作为新架构方向已拥挤,但作为失败切片、数据引擎或物理前端仍有价值。

## 水下与内窥镜/手术

水下特有挑战:

- 不同波长衰减导致色偏。
- 后向散射造成雾状朦胧。
- 真实 GT 极少。

Atlantis 用 Stable Diffusion 生成带深度监督的照片级水下图,体现“真实 GT 难得时用生成/合成数据补”的主线。水下米制深度还需要处理衰减、散射和合成到真实差距。

内窥镜/手术挑战包括湿组织高光、非朗伯表面、GT 缺失、尺度歧义、实时要求。主流策略是:

- DINOv2/Depth Anything 基础模型适配。
- LoRA/adapter 等参数高效微调。
- 合成数据和时序自监督。
- 尺度/米制恢复。

这类专门域是基础模型适配能力的压力测试。

## 事件相机、偏振、散焦和主动信号

非常规传感器/物理线索的共同目的,是用额外物理信息补 RGB 单目的病态性。

| 线索 | 优势 | 代表问题 |
|---|---|---|
| 事件相机 | 高动态范围、低延迟、低功耗、无运动模糊 | 与 RGB/HDR 融合是否已有工作需核实 |
| 散焦/聚焦 | 模糊程度与深度相关,物理可解释 | 需焦栈/双像素/已知光学模型 |
| 偏振 | 偏振度/角与表面法向相关 | 方位角歧义、偏振 GT 稀缺 |
| 主动立体/结构光 | 投射图案提供额外几何信号 | 需硬件或虚拟图案设计 |
| 雷达/热成像 | 雨雾/夜间互补,雷达提供米制稀疏锚 | 标定、稀疏噪声、跨模态融合 |

归档的统一判断:如果目标场景允许加传感器,多模态/主动信号通常比纯 RGB 算法更稳;如果不能加硬件,则需要物理建模、合成数据和不确定性评估。

## 全景、鱼眼和相机模型失配

360 度 ERP 全景有两极畸变和左右边界不连续。策略包括:

- 替代投影:球面、立方体、切平面。
- 几何感知算子:球面 CRF、球面 Transformer、PanoGabor。
- 多表示融合:局部畸变轻视图 + 全局 ERP。

鱼眼/广角相机则要求把真实相机模型嵌进投影/重投影损失,或学习畸变模型,或用 calibration tokens 把针孔数据重标定成鱼眼图适配基础模型。

这两类问题都说明:相机模型错误会让自监督几何损失和米制深度失真。A9 CAM-Convs 的相机感知思想在这些场景中尤其重要。

## 高分辨率与细结构

基础模型多在约 1MP 分辨率训练,直接处理 4K 会爆显存或丢细节。主流解法是分块/多分辨率融合:

- Boosting Monocular Depth:内容自适应多分辨率合并。
- PatchFusion:端到端分块框架,含 Global-to-Local 和一致性感知推理。
- Seamless Patchwise Refinement:处理接缝和低效问题。

高分辨率深度的两大难题是 patch 接缝伪影和推理开销。它是强基础模型的放大器,不是与基础模型竞争的新骨干。

## 合成数据引擎

合成数据是基础设施,不是附属品。归档把 Infinigen 和 Kubric 作为两类代表:

| 引擎 | 特点 |
|---|---|
| Infinigen | 程序化从零生成资产,可控、可无限渲染、精确 GT |
| Kubric | 多物体视频、深度、光流、分割等富标注 |

域随机化、CycleGAN 域迁移、DREDS、BRIDGE 等都在解决 sim-to-real 和数据覆盖问题。Depth Anything V2、Marigold、Atlantis、透明/水下/恶劣天气方向都依赖合成或生成数据来补真实 GT 缺口。

对失败模式研究,最有价值的数据不是“更多自然图”,而是可控改变材质、光照、几何、相机模型和天气的程序化数据。

## 遥感、航拍和高度估计

遥感常估“高度”而不是相机深度,但数学上仍是单图几何回归。HeightFormer 使用图像自适应分类-回归头,与 AdaBins/Binsformer 的分箱思想同源。UAV 米制尺度可由航空三角 tie points 或全球 DEM 锚定。

这说明尺度锚定具有领域特化形式:车载用 LiDAR/IMU,航拍用 DEM/三角测量,室内可用已知物体或人体尺度。

## 失败切片建议

一个完整 MDE 评测不应只按数据集平均,应至少切:

| 切片 | 根因 | 需要检查 |
|---|---|---|
| 透明/反光 | 折射/反射/多层深度 | 深度错误、不确定性是否升高 |
| 暗光/过曝/HDR | 动态范围和噪声 | 是否错误自信 |
| 雨雾雪/水下 | 散射和衰减 | 合成到真实泛化 |
| 动态物体 | 静态假设失效 | 时序一致和动态区误差 |
| 鱼眼/全景 | 相机模型失配 | 投影几何是否正确 |
| 高分辨率细结构 | 分辨率/显存 | 边界、接缝、细线结构 |
| 专门域 | 域差和 GT 缺失 | 适配是否过拟合 |
