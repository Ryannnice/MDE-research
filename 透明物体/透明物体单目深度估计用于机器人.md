# 透明物体单目深度估计用于机器人

日期: 2026-07-01

资料来源: 本仓库 `docs/文献调研/综述/前沿专题/05_评测协议与可比性.md`、`docs/文献调研/综述/前沿专题/06_失败模式与数据.md`、根目录调研包 `literature-search-20260629-robot-transparent-mde/`，以及 2026-07-01 重新联网核验的论文页、项目页和 GitHub README。

本文只服务“透明物体单目/少传感深度估计用于机器人”这个主题。目标是整理最先进工作、经典基线、常见数据集、代码/数据开源状态、复现优先级、机器人使用边界和可投稿选题。所有尚未真实跑出的实验结论保持 `待跑`。

## 一句话结论

透明物体机器人深度不是空白方向。经典主线已经由 [ClearGrasp](https://github.com/Shreeyak/cleargrasp)、[LIDF](https://github.com/NVlabs/implicit_depth)、[TransCG](https://github.com/galaxies99/transcg)、[DREDS/SwinDRNet](https://github.com/PKU-EPIC/DREDS)、[FDCT](https://github.com/Nonmy/FDCT) 和 [ReMake](https://github.com/ChengYaofeng/ReMake) 覆盖了 RGB-D depth completion / restoration；单 RGB 或 foundation MDE 近作由 [MODEST](https://github.com/D-Robotics-AI-Lab/MODEST)、[MOMA](https://arxiv.org/abs/2506.17110)、[SeeClear](https://github.com/YumengHe/SeeClear)、[Depth4ToM](https://github.com/CVLAB-Unibo/Depth4ToM-code)、[AISPO](https://arxiv.org/abs/2606.25503) 形成直接威胁；多层透明深度由 [LayeredDepth](https://github.com/princeton-vl/LayeredDepth) 和 [SeeGroup](https://github.com/princeton-vl/SeeGroup) 重新定义了“透明物体 depth 到底是哪一层”。机器人上界还有 [ASGrasp](https://github.com/jun7-shi/ASGrasp)、[GraspNeRF](https://github.com/PKU-EPIC/GraspNeRF)、[TranSplat](https://github.com/jeongyun0609/TranSplat)、[TRAN-D](https://github.com/jeongyun0609/TRAN-D) 等多视角/主动视觉/NeRF/GS 路线。

对接下来复现最有用的结论是:

- 第一批优先复现: `TransCG/DFNet`、`ReMake`、`MODEST`、`Depth4ToM`、`SeeClear demo`。它们覆盖 RGB-D completion、foundation MDE 适配、单 RGB 透明深度和生成式 opacification。
- 第一批数据集: `TransCG`、`ClearPose`、`Booster`、`LayeredDepth`。`Trans10K` 只有透明 mask；`SeeClear-396k` 截至 2026-07-01 还未发布。
- 不建议第一批硬啃: `AISPO`、`ClearDepth`、`Transparent Object Depth Completion`、`DualTransNet`。本轮未找到可直接复现的官方代码/完整入口。
- 不要把多视角/主动/NeRF/GS 系统和单 RGB 方法混成同输入横比: `ASGrasp`、`GraspNeRF`、`TranSplat`、`TRAN-D` 更适合作为机器人系统上界。

## Idea optimizer: 透明物体主题版

| idea | 核心问题 | 可复现资产 | 投稿风险 | 当前建议 |
|---|---|---|---|---|
| T0: 透明物体深度复现基准 | 现有透明深度方法代码/数据分散，很多论文难以同协议比较 | TransCG、ClearPose、Booster、LayeredDepth；ClearGrasp/LIDF/DFNet/TODE/FDCT/ReMake/MODEST/Depth4ToM/SeeClear demo | 工程味重，需要有清晰协议和 failure slice | 最适合现在做，先产出可复现 leaderboard 和脚本 |
| T1: 机器人可消费的透明 depth completion baseline | mask + MDE + RGB-D completion 在真实透明 mask 内到底能超过经典 DFNet/TODE/FDCT 多少 | ReMake、TransCG、ClearPose、Depth4ToM、MODEST、SeeClear demo | ReMake 已很近，必须用复现质量和协议差异切开 | 第二优先，适合在 T0 基础上形成方法小创新 |
| T2: 单 RGB 透明物体米制深度 | 没有 RGB-D raw depth 时，单 RGB 输出如何接近机器人需要的 metric/contact depth | MODEST、MOMA 论文、Depth4ToM、SeeClear、Booster、LayeredDepth | MOMA/AISPO/SeeClear 威胁很强，数据和真实机器人验证成本高 | 先做调研和离线 proxy，不建议马上主攻 |
| T3: 前表面/contact depth 协议 | 透明物体单层 depth 的 GT 到底应是前表面、后表面、背景还是多层 | LayeredDepth、SeeGroup、ClearPose、Booster | 容易变成协议论文，需要扎实标注/评测设计 | 适合做 failure slice 和论文定义边界 |
| T4: 生成式 opacification 的不确定性 | SeeClear 这类“先变不透明再估深度”在 mask 错误、材质变化、边界处是否稳定 | SeeClear demo、Depth Anything/MoGe、Trans10K、Booster、LayeredDepth | SeeClear 数据未发，完整训练复现受限 | 可做诊断型小题，不应作为第一主线 |

当前最稳路线: 先做 T0，把开源代码、数据下载、环境、指标和失败切片跑通；再基于复现结果选择 T1 或 T3。不要先押一个新模型口号，透明物体方向已有足够多直接威胁，复现表会决定真正的空位。

## 问题定义

透明物体对机器人深度估计的核心困难有四类:

| 失效 | 表现 | 对机器人影响 |
|---|---|---|
| 缺失深度 | ToF/结构光/主动立体返回无效值 | 点云破洞，抓取候选丢失 |
| 背景错深度 | 深度传感器读到透明物体后方背景 | 夹爪/吸盘碰撞，目标点云被拉到背景 |
| 折射/反射错深度 | 局部形状、边界、液体/玻璃导致非物理深度 | 法向、接触点、碰撞检测不稳定 |
| 多层歧义 | 前表面、后表面、背景层都可能可见 | 单层 depth 的任务定义本身不完整 |

机器人真正消费的通常不是“视觉上最像的深度图”，而是可用于抓取、吸取、避障、碰撞检测、6D pose 或 scene update 的物理几何。因此需要区分:

- `sensor depth`: 原始 RealSense / ToF / active stereo 深度，透明区域常失败。
- `completed depth`: 修复后的 RGB-D depth completion 输出。
- `monocular relative depth`: 单 RGB MDE 输出，常只有 affine-invariant 尺度。
- `metric depth`: 不做 scale/shift 对齐即可用米制解释的深度。
- `multi-layer depth`: 每条像素射线上的多层透明表面/背景深度。
- `grasp-relevant depth`: 机器人动作真正需要的接触面、前表面或碰撞包络深度。

## 方法谱系

### 1. RGB-D Depth Completion / Restoration

这是透明物体机器人抓取最成熟的主线。输入通常是 RGB + 原始深度，目标是在透明 mask 内补全或纠正深度。

代表工作:

- [ClearGrasp](https://arxiv.org/abs/1910.02550): 预测 normal、透明 mask、occlusion boundary，再用 global optimization 修复透明区域。公开代码、数据和 checkpoint；是透明 depth correction 的经典起点。
- [LIDF](https://arxiv.org/abs/2104.00622): 用 local implicit function 做透明物体 depth completion，CVPR 2021，官方代码和 checkpoint 公开。
- [TranspareNet / TODD](https://github.com/pairlab/TranspareNet): 联合点云补全和深度补全，并发布约 15k RGB-D 的 Toronto Transparent Object Depth Dataset。
- [TODE-Trans](https://arxiv.org/abs/2209.08455): Transformer + RGB-D 透明深度估计，官方代码和 TransCG checkpoint 公开。
- [TransCG](https://arxiv.org/abs/2202.08471): 大规模真实 RGB-D 透明 depth completion 数据和 DFNet 抓取 baseline。
- [DREDS / SwinDRNet](https://github.com/PKU-EPIC/DREDS): domain-randomized depth simulation + RGB-D restoration，覆盖 specular / transparent / diffuse。
- [FDCT](https://arxiv.org/abs/2307.12274): 强调低功耗机器人平台实时性，官方页面报告约 70 FPS，代码公开。
- [ReMake](https://arxiv.org/abs/2508.02507): instance mask + MDE relative depth + RGB-D completion，RA-L 2026，代码、checkpoint、训练脚本公开。
- [AISPO](https://arxiv.org/abs/2606.25503): RGB-D fusion + affine-invariant shape prior，RA-L 2026，代码当前未找到。

对本主题的意义: 这条线已经很成熟，不能只做 another RGB-D completion network。若做新方法，必须证明它在透明 mask、边界、无效深度洞和机器人接触面上优于同源 post-hoc completion / alignment。

### 2. 单 RGB / Foundation MDE 适配

这条线更接近“单目深度估计用于机器人”，但很多方法仍依赖 mask、伪标签、微调或一次性标定。

代表工作:

- [Depth4ToM](https://arxiv.org/abs/2307.15052): 对透明/镜面区域做 inpainting，再用 MDE/stereo 模型生成 pseudo label 微调，代码/数据/权重公开。
- [MODEST](https://arxiv.org/abs/2502.14616): 单 RGB 同时做透明分割和深度估计，ICRA 2025，代码和权重公开。
- [MOMA](https://arxiv.org/abs/2506.17110): 单 RGB + one-shot sparse metric-depth alignment，用 scale-rotation-shift calibration 做机器人抓取，IROS 2025；GitHub 仓库存在但 README 很短，完整复现脚本需继续核验。
- [SeeClear](https://arxiv.org/abs/2603.19547): 生成式 opacification，把透明区域变成几何一致的不透明外观，再接 Depth Anything 3 / MoGe。2026-06-30 README 显示 demo checkpoints 已上 Hugging Face，SeeClear-396k 数据仍未发布。
- [Towards Robust MDE in Non-Lambertian Surfaces](https://arxiv.org/abs/2408.06083): 面向透明/镜面 ToM 的区域引导训练与光照增强，代码状态未核清。
- [DepthShield](https://github.com/SeracoZ/DepthShield): 2026 demo 级仓库，mask-aware + blur refinement，论文/venue 证据弱，暂作观察。

对本主题的意义: MOMA 是最危险的 post-hoc sparse metric alignment 近邻；SeeClear 压住“先生成不透明外观再跑 MDE”；MODEST 压住“单 RGB 透明分割+深度”。任何多阶段系统都必须把 mask、opacification、额外 MDE backbone、后处理模块计入真实系统成本。

### 3. 多层透明深度

单层深度图对透明物体不总是物理完整。透明杯子可能有前壁、后壁、液体界面和背景层；玻璃门可能需要前表面用于碰撞，而视觉纹理来自后方。

代表工作:

- [LayeredDepth](https://arxiv.org/abs/2503.11633): 提出真实和合成多层透明 depth benchmark。真实 benchmark 有 1,500 张 in-the-wild 图像；synthetic 生成器可产多层 GT。代码和 Hugging Face 数据公开。
- [SeeGroup](https://arxiv.org/abs/2605.28735): CVPR 2026 Oral，多层深度自决定分组，把每像素多层深度建模为 unordered point process；代码和 checkpoint 公开。

对本主题的意义: 如果输出单层 depth，必须限定为 `grasp-relevant/contact/front surface`，不能声称完整透明光学重建。LayeredDepth/SeeGroup 更适合作为边界协议和 failure slice，而不是第一批主实验。

### 4. 多视角、主动视觉、NeRF/GS 系统上界

这些方法通常不是单 RGB 同输入 baseline，但它们构成机器人部署上界。

代表工作:

- [Dex-NeRF](https://proceedings.mlr.press/v164/ichnowski22a.html): 用 NeRF 抓取透明物体，系统成本高。
- [Evo-NeRF](https://sites.google.com/view/evo-nerf): sequential grasping 中复用/演化 NeRF，减少 sensing time。
- [GraspNeRF](https://arxiv.org/abs/2210.06575): sparse multiview RGB + generalizable NeRF 做 6-DoF grasp detection，代码/权重/数据生成器公开。
- [MVTrans / Syn-TODD](https://arxiv.org/abs/2302.11683): multi-view architecture 同时做 depth、segmentation、pose，并发布程序化合成 Syn-TODD。
- [ASGrasp](https://arxiv.org/abs/2405.05648): 用 RGB-D active stereo camera 做透明/反光物体重建与 6-DoF grasp，代码和 checkpoint 公开。
- [Transparent Object Depth Completion](https://arxiv.org/abs/2405.15299): single-view RGB-D + multi-view cost volume + confidence refinement，代码未核清。
- [TranSplat](https://arxiv.org/abs/2502.07840): surface embedding-guided 3D Gaussian Splatting，用 latent diffusion surface embedding 辅助透明 depth completion 和 manipulation，代码/模型/数据脚本公开。
- [TRAN-D](https://arxiv.org/abs/2507.11069): sparse-view 2D Gaussian Splatting + physics simulation 做透明深度重建和 scene update，ICCV 2025，代码、数据、Docker 公开。

对本主题的意义: 如果场景允许主动传感或多视角，通常比纯单目更稳。这些方法应写成“强传感器/多视角上界”，而不是与单图方法做同输入横比。

第一周建议执行顺序:

1. 下载 `TransCG`，跑 DFNet sample inference，确认数据格式、mask、raw depth、GT depth、split。
2. 跑 `ReMake` checkpoint inference，对同一批 TransCG 样本输出 depth。
3. 跑 `MODEST` 官方 inference，确认 single RGB 透明分割和 depth 输出能否对齐 TransCG/ClearPose。
4. 跑 `Depth4ToM` 在 Booster/ToM 上的公开权重，补齐非朗伯单目 MDE 基线。
5. 跑 `SeeClear` demo checkpoints，只记录 inference 可用性、耗时和是否能接 Depth Anything/MoGe 输出。

## 补充观察名单

这些工作没有进入主候选表的原因通常是: 与“单目深度用于机器人”的同输入设置较远、代码/数据状态不够清楚、任务更偏 segmentation/pose，或只适合作为相关工作背景。但写综述和 rebuttal 时仍应知道。

| 工作 | 类型 | 代码/数据状态 | 为什么观察 |
|---|---|---|---|
| [DepthGrasp](https://ieeexplore.ieee.org/document/9636382/) | IROS 2021 RGB-D transparent depth completion | 代码未核清 | ClearGrasp 之后的 self-attentive adversarial depth completion，ReMake/后续论文常在 related work 中提到 |
| [CDGrasp / ClueDepth Grasp](https://www.frontiersin.org/journals/neurorobotics/articles/10.3389/fnbot.2022.1041702/full) | RGB-D depth completion + grasping | 论文公开，代码未核清 | 用几何线索过滤折射/反射点，强调保留可定位 depth clues |

| [Mirror3D](https://github.com/3dlg-hcvc/mirror3d) | mirror depth refinement / dataset | 代码、数据、checkpoint 公开 | 虽是镜面而非透明抓取，但 ToM surface 和 RGB/RGB-D depth refinement 是重要相邻方向 |

## 机器人使用方式

| 使用方式 | 深度需求 | 代表工作 | 可比性 |
|---|---|---|---|
| 抓取候选生成 | 稠密点云、法向、接触面 | ClearGrasp、TransCG、ReMake、AISPO | 可做离线 grasp proxy，真实抓取 `待跑` |
| suction / parallel-jaw grasp | 局部 surface normal 和前表面 metric depth | ClearGrasp、MOMA | MOMA 是 one-shot metric alignment 直接威胁 |
| 6-DoF grasp detection | 物体完整/可见几何，碰撞检测 | ASGrasp、GraspNeRF | 多视角/主动输入，不是同输入 baseline |
| pose estimation | 深度/点云稳定、instance mask、CAD/pose label | ClearPose、FDCT、DREDS、TRansPose | 可作为 downstream proxy |
| scene update / object removal | 多视角一致重建 | TRAN-D、TranSplat | 系统上界，不适合与单图直接混表 |

## 最先进工作如何排序

如果只读 8 个，建议顺序如下:

1. [MOMA](https://arxiv.org/abs/2506.17110): 单 RGB + one-shot sparse metric alignment + UR5 grasp/suction，是单图米制透明深度的最高优先威胁。
2. [ReMake](https://arxiv.org/abs/2508.02507): 透明抓取 + MDE + instance mask + RGB-D completion，代码和训练成本清楚。
3. [SeeClear](https://arxiv.org/abs/2603.19547): 生成式 opacification + frozen/off-the-shelf MDE，压住透明预处理路线。
4. [MODEST](https://arxiv.org/abs/2502.14616): 单 RGB segmentation + depth，压住 mask/semantic 先验路线。
5. [LayeredDepth](https://arxiv.org/abs/2503.11633) + [SeeGroup](https://arxiv.org/abs/2605.28735): 多层深度挑战单层 depth 定义。
6. [TransCG](https://arxiv.org/abs/2202.08471): 最适合作第一批机器人透明离线 failure slice。
7. [Booster](https://arxiv.org/abs/2301.08245): 高分辨率非朗伯 ToM benchmark，用于 metric vs affine 协议。
8. [ClearGrasp](https://arxiv.org/abs/1910.02550) + [LIDF](https://arxiv.org/abs/2104.00622): 经典 RGB-D post-hoc completion 门槛。

需要继续监控:

- AISPO 是否公开代码、数据、checkpoint。
- SeeClear 是否发布 SeeClear-396k。
- MOMA 是否补齐训练/评测/机器人复现脚本。
- NTIRE 2026 Booster metric mono track 的最终报告、leaderboard 和代码。
- DepthShield、Transparent Object Depth Completion、Robust Non-Lambertian MDE、DualTransNet 等是否有稳定 peer-reviewed / official code 状态。

## 透明物体复现实验建议

### 最小可行透明 failure slice

| Stage | 数据 | 目标 | 必须对照 | 状态 |
|---|---|---|---|---|
| S1 | TransCG | 透明 mask 内 metric correction | raw depth、global affine、patch affine、MOMA-style SRS、DFNet/TODE/ReMake if runnable | 待跑 |
| S2 | ClearPose | 遮挡/液体/非平面泛化 | ClearGrasp/LIDF/TransCG/FDCT、MODEST if runnable | 待跑 |
| S3 | Booster | 非朗伯高分辨率 metric vs affine | Depth Anything V2/3、Metric3D、UniDepth、Depth Pro、Marigold/E2E-FT、Depth4ToM | 待跑 |
| S4 | LayeredDepth | 单层 contact/front claim 边界 | first-layer baseline、SeeGroup | 待跑 |
| S5 | Robot/grasp proxy | 修正 depth 是否减少 invalid grasp | MOMA/ReMake/AISPO/ASGrasp 文献或可跑对照 | 待跑 |

### 指标纪律

| 指标 | 为什么要报 | 状态 |
|---|---|---|
| full-image metric AbsRel/RMSE | 不做 scale/shift 对齐的米制主表 | 待跑 |
| affine-invariant AbsRel/RMSE | 诊断是否只是对齐后好看 | 待跑 |
| transparent-mask AbsRel/RMSE | 是否真修透明/反光区域 | 待跑 |
| transparent-boundary RMSE | 是否处理折射/边界不连续 | 待跑 |
| no-anchor transparent AbsRel | 少量锚是否能传播到无锚透明区域 | 待跑 |
| contact/front-surface proxy | 是否贴近机器人消费面 | 待跑 |
| invalid grasp / collision proxy | 应用价值，但不能替代真实机器人日志 | 待跑 |
| `nfe` 与 `nfe_real` | 透明 mask、opacification、MDE backbone、completion 都要计成本 | 待跑 |

### 写作边界

可以写:

- 透明/反光物体是机器人深度估计的高压 failure slice。
- 现有路线多依赖 RGB-D post-hoc restoration、mask-guided completion、训练式 ToM MDE、one-shot output calibration、生成式 opacification 或强传感器/多视角系统。
- 新方法要检验自己是否优于同源输出端校准/补全，尤其在透明 mask、边界、无效深度洞和无锚区域。

不应写:

- “透明物体深度估计仍无人解决”。
- “本文解决完整透明物体重建”。
- “本文超过 active stereo / NeRF / 3DGS / multi-layer depth 系统”。
- “本文提高真实抓取成功率”，除非仓库中已有真实 CSV 或 robot log。

必须写:

- metric 主表和 affine-invariant 辅表分开。
- `nfe` 和 `nfe_real` 分开。
- MOMA、ReMake、MODEST、SeeClear、LayeredDepth/SeeGroup、AISPO 是近期 reviewer threat。
- 所有透明 slice 和机器人结果当前为 `待跑`。
